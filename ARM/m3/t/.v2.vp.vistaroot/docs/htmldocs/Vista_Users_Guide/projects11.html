Defining Link Dependencies

A project may depend on several other projects, collectively referred to as its “dependencies.” This is important because the functionality compiled into the dependencies' design executable, pre-compiled objects, and shared libraries can be used by the dependent project.

The Link tab is used to link to the current project to one or more dependencies. Note that these links must be established before building the project.

To define link dependencies:

  1. Click the Link tab. The Link dialog box is now displayed.
  1. At the top of the dialog box, select the checkbox next to the name of any of the projects for which an object is to be defined as a dependency.
  1. Select the appropriate checkboxes to specify which of the available SystemC Verification (SCV) libraries can be used by the current project, or any of the projects defining it as a dependency.
  1. In the Link Object Name list, click the browse button (...) in the right-hand column.
  1. Using the Select Directory dialog box now displayed, navigate to a desired object and click OK. The specified object will be added to the Linked Object List. The linked objects listed for the current project will be accessible to any other project dependent on the current one. Conversely, the linked objects listed in any project selected in step 2 (above) will be available to the current project.
  1. Carry on defining additional linked objects as required.
  1. Click OK to save your definitions.

Once you have defined the dependencies for a project, you can build it together with these dependencies by choosing the Build > Build Tree command.
