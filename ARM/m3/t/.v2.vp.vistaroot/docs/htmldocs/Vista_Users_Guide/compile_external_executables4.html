Response File Mechanism

Vista's compiler script wrappers, vista_cc and vistac++, support entering one or more response files at the command line. Response files are plain text files that contain valid compilation options.

Ordinarily, compilation options can, of course, be individually specified on the command line manually entered in your system or Vista shell; but at times this can be tedious, time-consuming, and error-prone.

By utilizing response files, you can save time and ensure consistency of compilation option entries, configuring as many different compilation options as necessary, say for different projects, each as a separate response file. You can also invoke several response file calls on the command line, as well as enter them together with regular command line options. When you do so, the options listed in each response file are inserted sequentially at the point at which the response file is referenced.

In the command line, each response file must be preceded by an at (@) character.

You might then prepare a response file like the one depicted below: 

     -Xlinker
     -E
     -fpermissive
     -/home/foo/bar.c
     -/home/bar/01.0
     -/home/bar/02.0
     -/home/bar/03.0
     .
     .
     .
     /home/bar/0n.0

Having named this file response1.txt, you could then implement it on your vista_cc command line as follows: 

     prompt> vista_cc @./customization/response1.txt
