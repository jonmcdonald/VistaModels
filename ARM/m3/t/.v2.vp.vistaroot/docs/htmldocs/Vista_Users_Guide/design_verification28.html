Monitoring and Analyzing TLM2.0 based designs

Vista enables you to monitor and analyze transactions of any SystemC design as long as the transactions are based on the TLM2.0 standard.

You must add Vista monitors to the design. Vista monitors replace the design initiator/target socket bind functions, then the monitor objects can be added to the Vista waveform and/or Vista Analysis viewer.

To add a monitor:

  1. Activate the Vista TLM Tracer API.

In the Project Settings link tab sheet, click the vista_tlm_tracer package checkbox, or use the `+vista_tlm_tracer flag for the `vista_c++ command.

  1. For any initiator socket that binds to a target socket, as specified in an sc_module elaboration code (usually, constructor), add a Vista TLM Monitor object instead:
     VISTA_BIND_VIA_MONITOR(<Vista TLM Monitor sc_object name string>, <initiator_socket>, <target_socket>);
  1. Once simulation begins, trace and acquire the Vista TLM Monitor object to the waveform and/or analysis viewer. The Vista TLM Monitor object is viewed like any other Vista model socket.
