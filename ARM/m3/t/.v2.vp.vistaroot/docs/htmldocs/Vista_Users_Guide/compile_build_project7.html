Elaborating the Design

Once your design is built, elaborating it will allow you to

  • simulate and debug it
  • view its hierarchical structure

Simulation and debugging are discussed in the next chapter. Presently, we'll deal only with the hierarchical structure.

The OSCI kernel used by Vista was modified by Mentor Graphics so that, following elaboration, the structure of the design is discernible.

It is this structure that is reflected in the Vista browser when, following elaboration, a design is selected. In order to facilitate the utilization of this information, the concept “project” is central to Vista, and is expressed in the product's GUI. A project contains the following elements grouped into folders:

  • C++ Sources, some of which include the sc_main function
  • C++ Headers, some of which are public
  • Other files (documentation, for example)

  Note

Always keep in mind that the display of files in the Vista's virtual folders is the result of referencing these files only. Thus, copying, moving, and deleting of files in the Vista folders is in fact copying, moving, and deleting the appropriate references.

To elaborate the design:

  1. In the browser, select sc_main of the project.
  1. Choose the Simulation > Simulate command, or click the Simulate icon
  1. In the Simulation dialog box now displayed, select the “Stop after elaboration” checkbox; then click OK.
  1. Wait until the design is fully elaborated. When this happens, all debugging features become enabled, and a [+] symbol appears next to sc_main. (Note that the [+] symbol next to sc_main allows you to view the design structure at any time. This structure accurately reflects that of the design unless you modify it; in this case you must build and elaborate the design again.)
  1. At this stage quit simulation by choosing Simulation > Quit Simulate command or by clicking the Quit Simulation icon

Do not save your simulation results.
