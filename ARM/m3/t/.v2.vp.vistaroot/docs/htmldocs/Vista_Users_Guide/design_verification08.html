Tracing and Untracing Objects

Use the Trace tool button to turn on tracing for an object, and Untrace to cancel tracing for the object. (For transactions, this option is available in addition to Trace/Untrace Transactions).

TLM sockets can also be traced and acquired, and their transactions can be viewed in the Waveform. The beginning and end times of Read and Write transactions are shown, as well as Interrupt times.

Acquiring and/or Tracing Via the Trace/Trace Transactions/Acquire Dialogs

The Trace and Trace Transactions dialogs allows you to easily specify what objects to trace/untrace. Access these dialogs via the Simulation > Trace menu commands.

  • Trace/Untrace option to select the basic command
  • Command. This field reflects the parameters entered in the remaining fields of this dialog. You can also enter the command here directly.
  • In Trace Path, specify the full path of the object to be traced (for example, a.b.obj).
  • Use the Name Pattern and Kind fields to narrow the set of objects traced; regular expressions can be used (for example, top.a.* or top.x.port_*).
  • Mark the Trace tree checkbox to trace an object's sub-elements as well as the object itself.
  • Use the Trace at time: field and the until: field to specify a time range.
  • Limit the number of traced objects via the Trace Limit option and field.

To acquire the object, or acquire and trace (via the Simulation > Trace > Acquire dialog):

  • Specify command parameters (similar to Trace dialog parameters above, with commands Acquire and Trace and Acquire Only)
  • Specify a wave in the Acquire to wave field, either by entering the path or by using the pull-down list to select the location.
