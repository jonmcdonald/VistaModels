Using Vista Learning Power

Vista power modeling uses a state based power approach that maps power values to the state of the model or Signal Toggle-Rates indicating the level of consumed power of the model based on the most active signal toggle-rates. Power learning is based on one or more HDL simulation run results, each run result relies on two inputs: a power graph for each simulation run and a vcd file that includes chosen (state) register and signal value changes.

The power graph can be produced by either a commercial power extraction tool or your in-house technology. The power graph should be converted to the Vista power graph format described in the prerequisite procedure of “Extracting Power — State Based Method”.

The Vista Learning Power feature produces the 3 power components that Vista declares:

  • Leakage power — the static power; leakage power is considered as a constant that the power extraction tool provides
  • Clock Tree power — the part of the dynamic power that is always consumed when a clock changes; Vista calculates the clock tree power as the minimal power consumption of the clock changes
  • Dynamic Power — the part of the dynamic power that is consumed based on different model functional states; dynamic power is calculated as state based power using the values of the state variables and the power graph

Vista make use of 3 different methods to measure the model power:

  • Power policy — For a Vista TLM Model, it is possible to provide power estimation in a Top-down approach using Vista Power policies associated with the model states and/or the model transactions.
  • SystemC State-based power model — This power model can be attached to every kind of timed functional SystemC model. In this Bottom-up approach, Vista learns power depending on the state variable values extracted from the RTL VCD files and the corresponding Power-Waveform input.
  • SystemC Toggle-rates based power model — can be attached to every kind of timed functional SystemC model, and learns power depending on the activity (toggle rates) of the most active RTL signals. Overall power is calculated (not divided by leakage, clock tree, and dynamic components).

The differences in the flow for the first and second models are described in “Extracting Power — State Based Method”. The SystemC toggle-rate based power model functionality is described in “Extracting Power - toggle rates based method”.
