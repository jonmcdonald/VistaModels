Manipulating Files Between Target OS and Host


  Note

This section applies only for target platforms that boot an Operating System (OS). You must perform specific setup steps during the OS (Linux) build and configuration process for this feature to operate properly. For a Linux build example, refer to: VISTAINSTALL/generic/doc/fuse_host_access/README.txt.

When the OS console prompt is available (for example, after boot of Linux OS image) during Vista or Vista Virtual Prototype simulation, you can access the Host file system to perform read and write operations. This enables you to develop, build and debug software packages on the host then access them from the target OS console without the need to re-simulate and re-boot the OS on the virtual prototype.

Once the OS has been properly setup during the build and configuration process, then you can issue the following commands at the target prompt: 

     mdev  -s
     mkdir   /mnt/
     ./fuse_host_access.exe  /mnt/

The command mounts target /mnt/ to the host root file system. All file-system operations under /mnt/ are performed on the host file-system. For example, performing the following command, copies a file located at /some/directory/some/file on the host to the local directory on the target. 

     cp /mnt/some/directory/some/file .

The following command copies a file from the target to the root of the host: 

     cp /some/directory/some/file /mnt/.

  Note

Link-commands (ln) are not supported.
