Environment Variables for Controlling Behavior

The table below details those Vista environment variables which you can set to control Vista's behavior, as is described above.

Note that some additional environment variables are automatically set when Vista is invoked (and must not be changed by you):

  • VISTA_ROOT, which points to the root of Vista's installation
  • VISTA_VERSION, whose value is set to the installed version of Vista that you are using

Because these environment variables are set before Vista sources the .vistarc and /home/$LOGNAME/.vistarc files, you can utilize these variables to define settings within either of these files.

The right-hand column of this table details the command line switch that you can use, when invoking any Vista script, to override the value currently specified by an environment variable.

Settings for Modeling

The various settings available for Vista modeling tasks are summarized in the Table 2‑2, and are also noted in the relevant procedural sections of this document.

To change the Vista modeling settings:

  1. Locate the settings file called .papoulis_settings.tcl in your home directory.
  1. Make the required changes to the variable defaults, log options and/or procedures in the file, using the format:
           set ::papoulis_variables(variable_name) new_value
  1. Save your changes.

The model setting changes you made in the settings file take effect the next time you invoke Vista.

Using VISTA_INVOCATION_OPTIONS

The variable VISTA_INVOCATION_OPTIONS is used to specify the default Vista invocation parameters.

Values specified via command-line switches have the highest priority, followed by the values specified in VISTA_INVOCATION_OPTIONS, then by the values specified in the file /home/$LOGNAME/.vistarc, then by the values specified in this file, and finally, by the values specified in the environment variables of the shell which started Vista.

You can disable use of VISTA_INVOCATION_OPTIONS by invoking Vista with -no-vista-invocation-options.

The value of VISTA_INVOCATION_OPTIONS is read and evaluated by shell (sh). Therefore, any shell expansions and substitutions will be applied, unless prevented by quoting.

Example 2‑1. Batch-mode invocation 

     VISTA_INVOCATION_OPTIONS='-batch -tsv -simscript script.tcl -exec a.exe arg1 arg2'

This setting instructs Vista to be invoked in batch mode with subsequent simulation of the a.exe executable. It also switches on the tsv flag and specifies the simulation script to be used for the simulation. Execution parameters arg1 and arg2 will be passed to the simulation process.

Example 2‑2. Shell environment expansion

This example shows how the shell environment expansion may be used. 

      VISTA_INVOCATION_OPTIONS='-simscript $HOME/script.tcl -exec a.exe arg1 arg2'

Example 2‑3. Preventing expansion of parameters

This example shows how to prevent parameters from expansions.

A parameter "The price is $5" is passed "as is". The following assignment quotes the parameter in single quotes ('): 

     VISTA_COMPILER_OPTIONS='-exec $HOME/a.exe '"'"'The price is $5'"'"

As a result, VISTA_COMPILER_OPTIONS will be equal to 

     -exec $HOME/a.exe 'The price is $5'

It is recommended to check the value of VISTA_INVOCATION_OPTIONS using the echo command: 

     echo $VISTA_INVOCATION_OPTIONS

The output is as follows: 

     -exec $HOME/a.exe 'The price is $5'
