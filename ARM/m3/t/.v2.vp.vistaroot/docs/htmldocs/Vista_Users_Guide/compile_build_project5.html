Design Linking

The following elements are brought together during the link process:

  • Linked objects, specified by the user
  • Archive of the non-sc_main sources
  • sc_main
  • SystemC kernel
  • Dependencies of the project, whose archives (i.e., non-sc_main sources) are taken by Vista and are used in building the executable

The result of the link process is the executable that can be simulated.


  Note

The Relink Executable command is similar in function to Build Executable, but different in that it always relinks the executable. (Note that Build Executable will relink the executable after an analysis of project dependencies reveals that the executable must be relinked.)

The Relink Executable command (on the Build) can be used after you have modified some libraries or objects linked to an executable.
