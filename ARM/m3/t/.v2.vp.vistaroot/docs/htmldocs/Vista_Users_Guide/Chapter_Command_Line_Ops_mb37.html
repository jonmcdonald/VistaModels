read_vcd

Extracts a TLM database from a VCD file. A warning is issued if ports are not properly associated.

Syntax

read_vcd [-help] [-usage] [-interactive-mode] [-no-vista_sim] [-rand-seed seed]
[-energy_dump_file dump_file] [-format_energy_dump dump] [-tcl_script script]
[-vsim-options options] library_name model_name instance_path db_name vcd_file_path

Arguments
Common
  • -help
Provides help.
  • -usage
Displays the syntax for this command.
Optional
  • -interactive-mode
Open the HDL simulator in GUI mode (default is batch).
Note: The full argument name must be used.
  • -no-vista_sim
Do not dump transactions to the Vista standard simulation database.
Note: The full argument name must be used.
  • -rand-seed seed
Select seed for random sequence.
  • -energy_dump_file dump_file
Path for energy data file.
  • -format_energy_dump format
Format of energy data; default is internal format.
  • -tcl_script script
You can specify a path for a file which contains simulation parameters or commands for ModelSim (or another external simulator). The commands in the script that can be passed to read_vcd (by the -tcl_script option) are as shown in Table 18‑3.

Table 18‑3. tcl Commands for read_vcd

Command

Description

set_parameter_value

port_name parameter_name value

Assigns a value to a protocol parameter.

Example: The USB protocol contains the following parameter:

module usb #(parameter endpoint_type_param = `INTERRUPT_CONTROL)

The following command changes this mode from `INTERRUPT_CONTROL (=0) to another mode, `ISOCHRONOUS (=1):

set_parameter_value usb_port endpoint_type_param 1

associate_signal_to_pv_variable

signal_heir_name mb_variable_name

Values of signal_hier_name that appear in .vcd file are sent to learning process. An mb_variable mb_variable_name must be defined to provide the timed values of signals to Learning algorithm.

The mb_variable should be assigned by PV model in an identical way to signal in HDL process.

subtract_base_address

port_name base_address

For a specific port, subtracts base address from each address and writes offsets as addresses in message file.

Example:

You have a DMA at address 0x30000 in the system, and you write to a register located at 0x4 in the DMA itself. (0x30004 - 0x30000)

When using the same DMA in another design, you can use another base address (for example, 0x50000).
