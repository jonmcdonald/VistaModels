Command Reference - Modeling

The following table provides a brief description of each modeling command. Command details, arguments, and examples are available by selecting the links in the Command name column.

Table 18‑2. Vista Model Builder Commands

Command name

Description

add_generated_files_to_vista_project

Adds model generated files to a Vista project.

add_template

Add a user-defined model template.

associate_hdl_unit_to_model

Associates HDL unit and signals with a TLM model and Ports.

clean_tlm_library

Removes all models from a TLM library.

clear_causality_ban

Removes the causality ban.

configure_model_from_ipxact

Create a TLM2.0 component based on an IP-XACT description and saves the created component.

create_protocol

Creates a protocol in Vista model builder using a .tcl script.

create_tlm_library

Creates a TLM library.

create_tlm_model

Creates a new TLM Model Interface with the defined ports and parameters.

delete_objects

Remove objects from current project database.

does_tlm_library_exist

Confirms the existence of a TLM library.

dump_simulation_data

Dumps an ASCII list of all transactions in the TLM database into a file.

If an instance is specified, only transactions of that instance are dumped.

extract_model_interface

Creates a Model Interface where all Ports and Parameters are automatically extracted from the HDL interface signals.

generate_power_model_code

Creates a power model based on a state power policy defined in a TLM module.

generate_testbench

Extracts a Transaction sequence for selected ports from a TLM database.

get_causality_ban

Gets a banned event name for a port in a model.

get_most_active_signals

Extracts most active signals from VCD inputs of the model.

instantiate_model_from_ipxact

Creates a new TLM model according to the data extracted from the IP-XACT RTL description without saving the model.

instantiate_template

Creates an instance of a template model.

learn_model

Learns an HDL model from a set of TLM databases.

learn_power_hdl

Learns the power behavior by activity of HDL signals of the model.

list_commands

Lists Vista model builder TCL commands.

list_machine_architecture_commands

Lists Vista model builder machine architecture TCL commands.

list_read_vcd_commands

Lists Vista Power commands for the -tcl option in the read_vcd command.

load_model

Loads model data from a model (.mb) file.

load_tcl_machine_arch

Reads a tcl script for timing/power learning

parse_protocol_source

Parses protocol source files into Vista model builder.

parse_regular_expression

Parses the protocol internal regular expression file or an external source file if specified.

parse_verilog_source

Parses the Verilog source files into the model builder HDL with library_name.

parse_vhdl_source

Parses the VHDL source files into the model builder.

put_causality_ban

Allow to remove incorrect causality between input and output event.

read_vcd

Extracts a TLM database from a VCD file.

reload_tlm_library

Loads all models, by .mb file, into a library.

remove_tlm_library

Removes a TLM library.

report_instance_activity

Reports transaction activity statistics for the specified instance in the specified database.

run_external_command

Execute shell commands in a Vista Power script or prompt.

save_model

Saves a model.

save_tlm_library

Saves all models in a TLM library.

set_tlm_library_physical_path

Sets the physical path of a library.

start_output_log

Enables dump of output messages to a ModelBuilder_output.log (papoulis_output.log) file.

stop_output_log

Disables dump output messages to ModelBuilder_output.log (papoulis_output.log) file.
