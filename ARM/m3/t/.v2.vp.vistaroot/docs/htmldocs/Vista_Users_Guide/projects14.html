Specifying the Top of the Design

In SystemC, the entry point for executing a design is the sc_main routine. Therefore, in the context of Vista, in order to generate an executable a project must reference at least one source file that defines this routine. Only such a source file may serve as the top of a design.

In general, Vista differentiates between sources that define the sc_main() routine and those that do not. In the browser, the source files that define the sc_main routine for a project must be stored in (i.e., referenced by) the Designs folder for this project. (Vista in facts creates this reference after the project has been built.) All other source files for the project are stored in the project's Sources folder.

This is in keeping with the Vista methodology of clearly distinguishing between the class structures of a project and the design hierarchy decomposition, in order that they can be correlated later on.

During compilation, Vista uses code introspection to determine the following information about a design:

  • the classes used in the design
  • the structure of the classes (methods, fields, inheritance graphs)

Note, however, that information about any source defining the sc_main routine is displayed only after elaboration.
