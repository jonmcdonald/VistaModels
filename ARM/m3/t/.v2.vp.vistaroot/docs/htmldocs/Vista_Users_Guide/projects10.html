Setting Compilation-Related Options

Use the Compilation tab of the Project Settings dialog box to make the following types of compilation related settings: pre-processor definitions, makefile invocation options, and compilation options.

Definitions

Project settings can specify pre-processor definitions to enable conditional compilation. When you specify the -D compilation option (see "Running Compilation"), these definitions are added to the compilation command line for each of the current project sources.

  1. For each such definition, specify its name in the left-hand column.
  1. Optionally, specify a value in the right-hand column.
  1. Carry on defining additional pre-processor definitions as required.
Makefile Invocation Options

Any options you enter here (delimited by a space) are passed to the project's makefile whenever it is invoked during the build or compilation of a project. By default, “-k” is the makefile invocation option for a new project. However, you can control the value of the default using the VISTA_DEFAULT_MAKEFILE_INVOCATION_OPTIONS environment variable in the .vistarc file. (See "Environment Variables for Controlling Behavior".)

Note that, in order to improve performance of compilation and builds, if you do not explicitly enter the “-j” option here together with the number of CPUs to be used, if possible, Vista will try to determine the number of available CPUs and then pass this value, appended to the “-j” option, when the makefile is invoked.

Compilation Options

Any options you enter here (delimited by a space) are passed to compiler. By default the -Wall option is passed. However, you can overwrite this default by setting VISTA_DEFAULT_COMPILATION_OPTIONS environment variable in the .vistarc file. ("Environment Variables for Controlling Behavior")
