Examples structure

examples/test_modules/*.h - Test modules.h

examples/tops/*.h - top files, instantiating the test modules

examples/*.cpp - sc_main files.

examples/tops/schematics - graphical top units.

examples/parameters.txt - parameters file

examples/*.v2p - project files

examples/vista.ini - ini file of examples package.

Examples are organized as Vista projects. Following is a proposed flow to work with the examples:

  • Copy 'examples' directory to some writable place.
  • CD examples
  • Run Vista. The browser should show the list of the libraries and projects.
  • Open any graphical object 'tops' library.
  • Select any component.

Right-click and choose Open to open the source file, where the component is defined.

Right-click and choose Go To Instantiation to show the top source file, where the component is instantiated.

  • Select an appropriate example project, build and simulate it.
  • Consider the messages printed out by the example during elaboration.
  • Expand the design (and sockets/ports) and view the design structure
  • Continue simulation and consider the output.
  • You may change the parameter values in the parameter file (parameter.txt) and re-simulate, after elaboration expand the design and view the new module and connectivity structure
