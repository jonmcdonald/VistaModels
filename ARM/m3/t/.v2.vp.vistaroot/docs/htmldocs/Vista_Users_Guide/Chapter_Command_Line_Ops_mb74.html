vista_create_project

This script performs batch creation of a new project incorporating the specified source files and settings. Upon completion, the new project file is printed to the standard output.

Syntax

vista_create_project [-name project-name] [definitions-file]

Arguments
  • -name project-name
The project name is the name assigned to the project.
  • definitions-file
A textual file that lists the source files and settings to be incorporated into the new project. If not designated, Vista will use the contents of a file named “sources.sc” that should be stored in the directory from which you invoke this script.
Definitions File Format

Each line entry of this file should enumerate one of the following:

  • A path to a file to be included in the project. This path may contain references to path variables. If the path does not begin with either a path variable or a / (indicating that it is a full path), $PRJDIR/$PRJFILE_BASENAME using the path variable $PRJDIR (pointing to the location of the project file) and $PRJFILE_BASENAME will be pre‑pended to the path.
    The source files will be added to the appropriate Vista folder according to their respective suffixes:

    Table 18‑6. Vista Folders 

    Vista Folder

    File Suffix

    Sources

    *.c, *.cpp

    Public Headers

    *.h, *.hpp

    Documentation

    *.txt, *.doc


  • -I followed by a directory path that is to be added to the project's include path. Note that the rules defined above are also applicable here.
  • -D followed by a pre-processor definition either in the format -D<name> or
    -D<name>=<value>.
  • A line starting with -L or -l (lower-case L), which will be added to the project's link options.
  • A file having the suffix *.o, *.a, or *.so, which will also be sent to the compiler as a link option.
  • A path variable entry whose definition conforms to the format <path_variable>=<value>.
  • The name of a Vista package to be linked with the project. This may be any of the following:
  • SCV
  • SystemArchitect
  • SAArbBusLib
  • SABusLib
  • SAGenericLib
  • SAGenericProcessorLib
  • SA_IEEE394Lib
  • SA_PCILib
  • SASchedulerLib
