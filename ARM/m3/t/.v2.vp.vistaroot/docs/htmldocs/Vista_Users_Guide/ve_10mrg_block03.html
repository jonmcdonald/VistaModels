Vista Workflow

The following steps summarize the design and verification process and clarify where the schematic is used in this process:

  1. A Vista library is created, using the File > Open Library Manager command.
  1. A set of models are built, using the modeling dialogs. As a result the h, cpp and mb files are added to the Vista library.
  1. For the assembly phase, a new schematic (block diagram) is created, and components (models) are instantiated using copy & paste from the libraries to the block diagram (or the Get Component command from the block diagram).
  1. For simulation, a Vista project is created, enabling adding models and block diagrams to it using the Add to Project command. Building the project (compiling) enables simulating the design.
  1. Simulations are run and analyzed (see "Simulation and Verification" and "Analysis Graphs and Reports").

Typically, in most TLM models only binders are used to bind initiator sockets to target sockets. In some cases, primitive channels connected to ports may be used as well. It is also possible to create ports or sockets that are hierarchically bound to instances' socket pins.
