Linking with Purify

Vista supports working with IBM Rational PurifyTM, thus enabling you to examine memory utilization in your designs and to debug any related problems. Note, however, that this support does not include detection of stack overflow or stack bounds read/write errors on SystemC thread stacks.

As is described below, you can run Purify for your Vista designs in one of two possible ways:

  • By calling the “vista_purify” wrapper script
  • By specifying Purify when invoking compilation

In both cases, by default the installation of Purify designated by your PATH environment variable will be used. However, you can specify a different installation of Purify using the --purify=<path> flag (see below).

Calling the vista_purify Wrapper

When linking your design, specify the following arguments in the link command line: 

     vista_purify [<purify_options>] vista_cc [<vista_cc_options>]

where:

  • vista_purify is the name of Vista's Purify wrapper script provided in the $VISTA_ROOT/bin directory.
  • purify_options can be any of the available Purify options. (For details, see your Purify documentation.)
  • vista_cc is the wrapper script used to compile the design with gcc. Note that this argument may be replaced by vista_c++ when g++ is used. Both of these, in turn, may invoke any of the supported versions of the gcc or g++ compiler. For details, see "Controlling Vista Behavior", and "Invoking Vista".
  • vista_cc_options can be any of the available options for vista_cc or vista_g++. Among these options is --purify which can be used to designate that a non-default installation of Purify is to be used, and is followed by the appropriate path, i.e., the full path to the desired installation; for example,
        vista_purify -always-use-cache-dir=yes \ 
          vista_cc --purify=/usr/local/purify/purify \ 
          simple_bus.o peripheral1.o ... -o sim

(For details about the various other available vista_cc and vista_c++ options, see the discussion of the command-line arguments that are available for these two wrapper scripts in "Compiling Externally Built Executables")

Specifying Purify when Invoking Compilation

As an alternative to specifying the vista_purify wrapper as an argument in the link command line, you can use Purify by specifying the --purify option when running one of Vista's compilation wrapper scripts as follows: 

     vista_cc --purify[=<path>] [--purifyargs=<arg1>,...,<argn>] [<vista_cc_options>]

where:

  • vista_cc (or alternatively vista_c++) is the name of the wrapper script used to compile the design.
  • --purifyargs= specifies one or more of the available Purify options to be used.
  • vista_cc_options can be any of the available options for vista_cc or vista_g++. Among these, once again, is --purify which can be used to designate that a non-default installation of Purify.
Integration with Visual Elite

One of the most important features of Vista is that, after you build a project, a library that encompasses the entire project can be created. This library can be read directly by Mentor Graphics's Visual Elite tool.

In this way designers can develop their SystemC IP's in Vista, and immediately integrate every component in Visual Elite's graphical environment. The benefits from this robust Visual Elite system integration include:

  • Integrating the design with software using Visual ESC
  • Generating a complete Virtual Prototype for the software and firmware engineers
  • Co-simulating SystemC designs with HDL using any commercial HDL simulator

For details, see "Vista and Visual Elite".
