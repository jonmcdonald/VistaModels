mb_event Class

mb_event objects serve to synchronize the PV and T models when there are threads in a component other than a CPU.

The header file contain the following definitions: 

     mb_event Data_event, Configuration_event, Output_event;
     bool valid_Destination, valid_Size, valid_Trigger;

The implementation uses the mb_event as follows: 

     void CU_pv::thread() {
       cout << name () << " Waiting for Data" << endl;
       wait (Data_event);
       cout << name () << " Waiting for Configuration" << endl;
       wait (Configuration_event);
       if (irq_in.read () == false) {
         cout << name () << " Waiting for Listener" << endl;
         wait (Output_event);
       }
       cout << name () << " Transfer" << endl;
       master_write (Destination, memory, Size);
       valid_Destination = valid_Size = valid_Trigger = false;
       cout << name () << " Assert Interrupt" << endl;
       irq_out.write (true);
     }
