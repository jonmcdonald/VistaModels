Editing Symbols within the Block Diagram Editor

You can edit component symbols either directly within the Block Diagram Editor or in the Symbol Editor. (For the latter method, see "Symbol Editor".)

New Component Editing Capabilities

You can edit the component's frame directly in the block diagram editor as well as move the component's pins around its frame. Those graphical changes can be saved to the underlying component symbol graphics using the Save Symbol command.

Editing the component's shape

Follow this procedure in order to edit a component's frame directly in the block diagram editor:

  1. Open a new block diagram.
  1. Drag and drop a component into the block diagram you've just opened.
  1. You may (or may not) select the component and from the RMB popup menu, choose the Add-stub command.

You'll notice that a binder and an external pin have been added to the component's pins.

  1. Select the component and view the component's frame handles. Select a handle, then drag the frame and drop it.

The component changes its shape accordingly and (when applicable), the binders are correctly routed.

Editing the pins' locations

Follow this procedure in order to edit a pin's location directly in the block diagram editor:

  1. Select a component with routed pins.
  1. Choose the Enable - Move Pin option from the RMB popup menu.
  1. Select a pin and move it around the frame, then drop it. The pin and its name is correctly positioned based on the frame they were dropped on.
  1. From the RMB popup menu, choose the Disable - Move Pin option. The pin's position is locked again.
Save Symbol

Graphical changes to components in the block diagram are made to the component's representation in the block diagram, and not to the underlying component graphical symbol.

To update the underlying component symbol, use the Save Symbol command by selecting the edited component and choosing the Save Symbol command from the RMB popup menu.

If there is no user defined symbol for the edited component, a “Save as” prompt will open, enabling you to specify the symbol name. Further issues of the Save Symbol command will update that (user defined) symbol.


  Note

While the Save Symbol command updates the underlying component symbol from the block diagram component graphics, the Update Component command does the opposite; it updates the component graphics based on the underlying component symbol. Hence, unless the Save Symbol command was previously used, the Update Component command may revert the component graphics back to the original symbol graphics.

Edit Symbol in Place

For more profound changes in the component graphics, the Edit Symbol in Place command may be used. To edit a component symbol in the Block Diagram Editor, select the symbol and, holding down the right mouse button, choose the Edit Symbol in Place command.

When you choose this command, the elements of the symbol are un-grouped and the fill color for the symbol is automatically removed. This means that (1) you do not have to manually unfill the symbol before editing and that (2) it is always clear which component symbol is currently being edited. When you finish editing the symbol, but before saving your changes, if you want to refill the symbol with color, you must do so manually.

During an Edit Symbol session in the Block Diagram Editor, you can edit the symbol graphics using the Draw icon group in the left-hand toolbar. (These icons are otherwise used for editing free graphics in a block diagram.) You can also perform the following operations on pins: Delete, Move, and Rotate (as is done in the Symbol Editor).

Once you have begun an Edit Symbol session in the Block Diagram Editor, the right mouse button displays the following two commands:

  • Update Pin, which updates the symbol consistent with the current interface of the instantiated unit.
  • Conclude Edit Symbol in Place, which ends the Symbol Editing session and displays a dialog box offering three possibilities:
  • Locally save edited symbol
  • Save changes to original symbol
  • Save and update symbol in design tree

  Note

While editing a symbol within the Block Diagram Editor, you can also edit the block diagram unit itself, performing any of the editing operations that are normally available for a unit.

Rotating Graphic Elements

Channel names, components, connectors, connector names, block names, and taps can be rotated.

To rotate a graphic element —

  1. Select the element to be rotated.
  1. Click the Rotate icon: .

With each click, the element rotates 90 degrees counterclockwise.

In block diagrams, after you rotate a component, autorouting may adjust the paths of channels in order to keep them as clear and separate as possible, unless you disable autorouting. To disable autorouting, click Tools > Options > Setup in the main bar and deselect the “Perform auto route” check box.

Deleting an Element

To delete an element, such as a block or channel in the Block Diagram Editor, use whichever of these methods you prefer:

  • Select the element, and then click the Delete icon:
  • Select the element, and then select the Delete or Cut command in either the Edit menu or popdown. (If you use the Cut command, the deleted element is retained in the clipboard.)
  • Select the element, and then use the Del key.
Duplicating Graphic Elements with the Array Command

As an alternative to copying and pasting a graphic element a number of times, the Array command provides a quicker way to duplicate a graphic element several times

  1. Select the element.
  1. Click Edit > Array.

Or, select the Array command in the popdown that appears when you click the right mouse button.

  1. In the dialog box that appears, select any of the following options:
  • Horizontal. Specify the number of elements to be created in a row.
  • Vertical. Specify the number of elements to be created in a column.
  • Diagonal. Specify the number of elements to be created diagonally.

You can enter numeric values for both Horizontal and Vertical, and thereby
create a two-dimensional array of elements. However, if you enter a value for Diagonal, no additional value may be specified.

  1. Move the pointer to position the elements, and click.

The elements are spaced evenly from one another.
