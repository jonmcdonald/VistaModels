Memory Storage and Print command

Vista provides an external utility called “Memory Storage and Printer” that is operated in batch mode and enables you to associate files under a host with memory models and print the file content (memory) at any time during simulation or post-simulation. The memory content print-out shows updated values using different formats as specified in the print command arguments.

Memory Storage

The memory storage functionality supports paged memory. You can choose to base any memory model on a host file storage, which helps you load initial values into memory and print the memory using the Memory Printer command. To associate a file with a memory model, a memory parameter should be associated with the full-pathname of the file. Currently only Vista generic memory models can be used.

Memory Storage Modes

You must specify the mode you need to work with by adding a mode-specific parameter in the Vista parameters file (parameters.txt).

Memory Storage Parameters

The following parameters can be used:

  • mb_memory_file_storage — path to a memory file to use as the initial memory and memory storage. File must be set for read/write access.
  • mb_memory_file_storage_load — path to a memory file to use as initial memory only. Memory is stored in a temporary file and deleted when simulation is complete.
  • mb_memory_paged — If true, memory is based on a temporary file. An error is issued if both mb_memory_file_storage and mb_memory_paged are defined.
  • mb_memory_max_loaded_num_chunks - Specifies the maximum total number of chunks to load in memory. The number specified must be greater than 10. If not specified, mb_memory_max_loaded_num_chunks is 10.
Memory Storage Examples

The following example shows how to set the memory file storage and the maximum number of data chunks allowed to be loaded. 

     top.rom0.mb_memory_file_storage = storage.memory
     top.rom0.mb_memory_max_loaded_num_chunks = 20
Memory Printer

The vista_memory_printer command enables you to print a memory file. You can specify a path to a file to print, or a path to a simulation directory and memory module to print, if the memory model is associated with a storage file.

Command Arguments
  • -file — file path
  • -address — starting address
  • -size — size of element
  • -count — count of elements
  • -radix — the printing format; specify as a string, hex, dec, oct, or assembler

  Note

When specifying an assembler radix, the VISTA_MEMORY_PRINTER_OBJDUMP environment variable is mandatory.

  • -outformat — number of columns (default 8)
  • -arch — processor architecture
  • -objdumpargs — any extra arguments to pass directly to objdump; must specify as quoted (“”) strings
  • -simdir — simulation directory path
  • -module — full-path-name of memory module from your simulation to print
Memory Storage Print Examples

The following are some examples of how to use the vista_memory_printer command: 

     vista_memory_printer  -file /home/asmair/vova/ObjDump/a.out 
     
     Output:
     0x0 :   0x7f   0x450x4c   0x46   0x020x01   0x01   0x00
     0x8 :   0x00   0x000x00   0x00   0x000x00   0x00   0x00
        
     
     vista_memory_printer -simdir /home/tutorial/ -module top.mem_inst -size 4 
        -count 4 -radix dec 
     
     Output:
     0x0 :       117940364765794      0   0
         
     
     vista_memory_printer -file /home/asmair/a.out - size 4 - count 11 
        -radix oct -outformat 3
