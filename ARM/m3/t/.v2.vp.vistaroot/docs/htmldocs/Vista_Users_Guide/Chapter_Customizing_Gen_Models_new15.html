Timer

This component raises an Interrupt in a periodic manner.

Ports provided for this model type:

  • One bus (ahb_slave) port through which the Timer is programmed
  • One interrupt (signal) port for the generated interrupt.

Parameters include:

  • Clock period: the number of picosec per the period of the clock used by the timer to compute the timer.

Via the registers you can set:

  • clock_count: the number of clocks between the interrupts
  • int_count: number of interrupts to raise
  • clear_irq: The interrupt is cleared when this register is written (the interrupt is not automatically cleared after a time period)
  • restart: this triggers a restart of the complete sequence of interrupts

The following is an example of a code segment that programs a timer: 

               //Initialize the timer
               //Generate an interrupt each 100 clocks
               cpu_master_write ( timerAddress+clockCountOffset, 100 ) ; 
               //Generate 1000 interrupts
               cpu_master_write ( timerAddress+intCountOffset, 1000 ) ;
               //and start it - 1000 interrupts will be sent
               //the time between the interrupts will be 100*Clock_period
           
               cpu_master_write ( timerAddress+restartOffset, 1 ) ;
               ......
               //When an interrupt is recievd it should be cleared by:
               cpu_master_write ( timerAddress+ clearOffset, 1 ) ;
