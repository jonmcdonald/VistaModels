Object Attributes

Each graphical object can be further refined using the “Attribute” dialog. As described below, each object type has its own attributes that you can configure in order to define detailed design specifications.

Component Attributes

Component attributes are general attributes for all components.

  • Name: Instance specific name of the design unit
  • Visible: Mark this check box to make the Instance name visible.
  • Create Instantiation as: Select either “Class pointer” creating a pointer to the instance or “Class object” to create an object of the instance.
Component Pin Attributes

The Attributes dialog also displays the attributes of the selected pin (port) of a component.

  • Name: The name assigned to the pin.
  • Port Class: The name of the class, as defined for the port/socket
  • Data Type: The data type class name of the channel
  • Expression fields: These fields allow you to associate a valid expression that is applicable to the instantiating unit with a selected pin.

Note that a pin associated with an expression cannot be connected to a signal in the block diagram. Conversely, a pin already connected to a signal cannot be associated with an expression.

  • Associate expression: Mark this check box if you want to associate an expression with the selected pin.
  • Expression: Enter the expression to be associated with the pin.
  • Visible: Mark this check box if you want the associated expression to appear in the block diagram.
