Multi-Thread Simulation

The separately licensed multi-thread simulation feature of the Vista product supports QEMU based ISS's and enables you to utilize multi-core host computers to speed up multi-core VPs.

The multi-thread simulation feature utilizes the LT sync quantum concept of TLM-2.0, so that the ISSs run asynchronously on different native threads during a single sync quantum period. When the quantum is expired, the thread is forced to synchronize with the main SystemC thread. Synchronization also happens when an IO (a non-DMI transaction) is performed (and in some other cases).

To use this feature effectively, you should use high values of LT sync quantum.

To activate the multi-thread simulation feature:

  1. Link your project with -lpthread.
  1. Assign a thread_mode parameter to one of the following:
  • SYSTEMC - regular mode
  • ASYNC_STABLE - Use pthreads stable mode
  • ASYNC_FAST - Use pthreads fast mode

The difference between ASYNC_STABLE and ASYNC_FAST is related to how the atomic instructions (for example: exclusive load/store or atomics swaps) are modeled. In ASYNC_STABLE mode, each atomic instruction causes synchronization. In ASYNC_FAST mode, atomic instructions are implemented using x86 atomic instructions. Correct algorithms must work in both modes; ASYNC_FAST mode is non- deterministic but faster when many atomic instructions are executed. Lock-less algorithms on the target platform are non‑deterministic in any mode.
