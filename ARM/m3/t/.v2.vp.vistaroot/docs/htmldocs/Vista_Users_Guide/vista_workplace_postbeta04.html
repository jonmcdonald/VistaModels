Libraries and Projects

Vista Project is a container which allows you to reference files associated with your design though various folders:

  • Public Headers
  • Sources
  • External Headers
  • Linked Binaries (.o, .a, .so) - objects, archive (static) libraries or dynamic libraries
  • Other Binaries - (.exe, .x) - manually-added executable allowing you to list and instantiate models in a Block Diagram whose objects are included in the executable. The executable residing under “Other Binaries” is not part of the link upon project build. (You add these by Add Files in the Project Settings > Files tab.)
  • Documentation
  • Schematics (created automatically when a Block Diagram is first saved) - .h file per schematic
  • Simulation - created automatically when a successful build generates an executable ready for simulation.

When a build of the project is done, all objects included in the executable are found by Vista (graphical symbols are generated); in the Block Diagram window they are listed in the Modules list, allowing you to instantiate them and create a new schematic.

As you design new models (or configure generic models) they get saved in your own libraries. For instructions on creating a new library, see "Libraries and Projects".

To build a schematic, compiled modules are needed: they can be compiled in Vista via the project, or you can include precompiled objects or executables in the project (see "Libraries and Projects" for details).

The Vista workplace consists of various windows, dialogs and viewers corresponding to Vista's main workflow (modeling, platform assembly, verification, analysis and virtual prototype creation). The main windows are:

  • Vista Window
  • Vista Modeling Window
  • Block Diagram (Schematic) Editor Window
  • Verification Viewers (Waveform, Simulation Control, TSV, Memory Profiler)
  • Analysis Window

Each window contains its own s, toolbars, and panes.

This chapter describes how to work with the various elements belonging to Vista's graphical user interface.
