Models

Users can select among the following models for customization:

  • Bus
  • BusRemapper
  • CPU
  • Cache
  • DMA Controller (DMAC)
  • Driver
  • Double Data Rate Memory Controller (DDRMC)
  • Fileinput
  • Generic Interrupt Controller (PL390)
  • INTC (Interrupt Controller)
  • Interconnect
  • Level 2 Cache Controller (L2C-310)
  • Memory
  • NIC301
  • Snoop
  • Sparsemem
  • Splitter
  • SystemC Model for UART Visualization Object - Middleware model
  • Timer
  • Transactor

These can be incorporated by double-clicking on a model in the Vista browser (Generic_Library). This opens the modeling dialog (see "Defining a Model").

After the specifications (ports, parameters, registers etc) are customized as needed and saved, the model can be generated. The resulting (generated) code characterizing the generic model is ready for compilation. The code includes not simply “template” definitions, rather the complete implementation of the model which can be used either as-is or as a basis for a new model.

The customization that can be done for a model (without changing its code) includes the following:

  1. Adding ports where it is allowed.
  1. Adding parameters to be used afterwards (e.g., in the policies).
  1. Changing the parameter default values.
  1. Modifying the values of the addresses and the widths of the registers in the register table.
  1. Adding or changing policies.
