ARM ADR Tool Directory Structure

The directory structure created by the ARM ADR tool as a result of generating IP-XACT description for a configured component, is similar to that shown in Figure D‑3 for the NIC301.

For the NIC301:

  • The top level component description is located at:

…/project_name_RTL/logical/component_name/spirit/component_name.xml

and

…/project_name_RTL/spirit_stitching/currcomponentdef/component_name.xml

  • The top level design description is located at:

…/project_name_RTL/logical/component_name/PL301/stitching/component_name.xml

  • Design descriptions created for intermediate clock domain hierarchies are located at:

…/project_name_RTL/logical/component_name/PL301/stitching

  • Bus definitions and abstraction definitions are located at:

…/project_name_RTL_RTL/logical/component_name/shared/busdefs


  Note

There are no hierarchical components created for intermediate clock domain nodes. The design objects of these nodes are referenced directly from the top level design description as component instances.
