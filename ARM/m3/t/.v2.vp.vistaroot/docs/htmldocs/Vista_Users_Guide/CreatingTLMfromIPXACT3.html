Creating a TLM Model Using Batch Flow

If you are running Vista in batch mode, use the following commands to create a fully configured TLM2.0 model out of the IP-XACT RTL description created by the ARM AMBA Designer (ADR) tool: configure_model_from_ipxact and instantiate_model_from_ipxact. See the following sections for a description of each command:

  • configure_model_from_ipxact
  • instantiate_model_from_ipxact

configure_model_from_ipxact

Use the following command to:

  • Import the IP-XACT description from a specified root path.
  • Extract the configuration information from the IP-XACT description.
  • Create the TLM2.0 component based on the configuration data.
  • Save the created component (.mb file) into a specified Vista library.
  • Optionally add the component to a specified Vista Project.
  • Optionally generate an IP-XACT description (TLM2.0 level) for the created component.
Command

configure_model_from_ipxact

Mandatory Arguments
  • template_path — path to the generic model

Currently only Nic301 is available: /TLM/Generic_Library/Nic301

  • model_name — name of the ARM top component created by the ARM ADR tool
  • ipxact_root_path — full path to the root of the directory tree created by the ARM ADR tool that contains the IP-XACT description for the created component:

…/<project_name>_RTL/logical/component_name

or

…/project_name_RTL/logical

  • output_library — name of Vista library in which to create the TLM model
Optional Arguments

All optional arguments accepted by the generate_model_code command, in particular:

  • generate_kind -pvt | -pv
  • overwrite 1 | 0
  • vista_project — name of Vista project in which to add the model
  • generate_ipxact 1 | 0
  • ipxact_vendor — vendor name to use in generating the IP-XACT description
  • ipxact_version — version string to be use in generated IP-XACT description

instantiate_model_from_ipxact

Use this command to create a new TLM model according to the data extracted from the IP-XACT RTL description without saving the model.

This command is useful when additional batch processing is required during the time between creating the model and saving it to a Vista project library.

Command

instantiate_model_from_ipxact

Mandatory Arguments
  • template_path — path to the generic model

Currently only Nic301 is available: /TLM/Generic_Library/Nic301

  • model_name — name of the ARM top component
  • ipxact_root_path — full path to the root of the directory tree created by the ARM ADR tool that contains the IP-XACT description for the created component:

…/<project_name>_RTL/logical/component_name

or

…/project_name_RTL/logical

  • output_library — name of the Vista library in which to create the TLM model
Optional Arguments

All optional arguments accepted by the generate_model_code command, in particular:

  • generate_kind -pvt | -pv
  • overwrite 1 | 0
  • vista_project — name of Vista project in which to add the model
  • generate_ipxact 1 | 0
  • ipxact_vendor — vendor name to use in generating the IP-XACT description
  • ipxact_version — version string to be use in generated IP-XACT description
Return Value

The command returns the Vista model builder path (model_path) of the created model. The path can be used in a subsequent save command; for example:

save_model -generate -vista_project $vista_project $model_path
