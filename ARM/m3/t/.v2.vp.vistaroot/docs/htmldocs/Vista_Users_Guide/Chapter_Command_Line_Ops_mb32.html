parse_protocol_source

Parses protocol source files into Vista model builder.

Syntax

parse_protocol_source [-h | -help] [-usage] [{-i | -include-dir} include_dir]
[-s | -suppress-warnings] [-generate_ipxact] [-dest_library string] [-vendor string] [-version string] source_file

Arguments
Common (Optional)
  • [-h | -help]
Provides help.
  • [-usage]
Displays the syntax for this command.
Optional
  • [-i | -include-dir include_dir]
Path to the location of includes.
  • [-s/-suppress-warnings]
Causes warnings not to be displayed during the parsing process.
  • [-generate_ipxact]
If present, this will cause the generation of an IP-XACT Bus Definition named <protocol>.xml and an IP-XACT Abstraction Definition named <protocol>_tlm20.xml, describing the protocol. These definitions are at TLM 2.0 abstraction level, using transactional ports.

  Note

Both ahb_master and ahb_slave protocols are represented by the same AMBA AHB bus definition (and AHB_tlm abstraction definition). These AMBA AHB definitions refer to a pair of AHB Lite IP-XACT bus definition and abstraction definition, which are created automatically.

  • [-dest_library string]
Defines the library name for the generated IP-XACT descriptions. If not present, the name “Library” will be used as the default. This is only considered if the -generate_ipxact option is present.
  • [-vendor string]
Specifies the vendor name for the generated IP-XACT descriptions. If not present, the vendor name defaults to “Mentor.com”. This is only considered if the -generate_ipxact option is present.
  • [-version string]
Specifies the version string for the generated IP-XACT descriptions. If not present, the version string defaults to “1.0”. This is only considered if the -generate_ipxact option is present.
Required
  • source_file
Name of the source file to be parsed.
