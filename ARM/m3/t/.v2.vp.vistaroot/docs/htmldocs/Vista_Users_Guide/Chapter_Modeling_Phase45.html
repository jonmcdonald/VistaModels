Model Representation in the Vista Browser

Model interface and attributes (.mb) files are saved under the library folder. The .mb file is a tcl script containing all settings made using the Model Editor dialog: ports, registers, policies, parameters and other model specifications -- i.e., all specifications except code that might be added to the model manually. It is represented as a text file in the design tree. Options for the Generate commands set_generate_kind and set_overwrite are also included here. (Note that this file is distinct from the model header and c++ generated files.)

The .mb file makes it possible to recreate the model by selecting the associated tcl script file object, using the menu command Modeling > Load Model Files (or Load Model Files in the right-mouse-button context) in cases where it is necessary to revert to an different version of a model.

The following Browser objects relate to the task of building models:

  • Files - for each saved model, .mb (text) file representing the most recent configuration of the model
  • Libraries - user-created libraries (both based on Generic models and from scratch)
  • Protocols provided by Vista
  • Generic Models (in the Generic Library) provided by Vista

Note that if the .mb file is modified after a model object has been created, a graphical indication lets you know that the modified .mb file is not in synch with current model.
