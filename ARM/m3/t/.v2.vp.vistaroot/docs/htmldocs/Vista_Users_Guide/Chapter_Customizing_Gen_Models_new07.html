DMA Controller

The purpose of this component is transfer of data from one memory location to another.

Associated ports:

  • One slave port (bus slave protocol), through which the DMAC is programmed
  • One master port (bus master protocol)
  • One interrupt port (signal protocol)

The following set of registers is provided:

  • Source - source address to read from
  • Destination - destination address to write to
  • Size - size of the transfer (in bytes)
  • Trigger - when written, the specified number of bytes are transferred from source to destination and an interrupt is sent to the processor
  • Acknowledge - should be written when the interrupt has been processed. A write to this register clears the interrupt.

By default, the policy used for the transfer is pipelined (the DMA implements a block transfer by interleaving reads and writes); You can define your own policy for the transfer.

The following is an example of code that generates a DMA transfer: 

              //Set the source address
               cpu_master_write ( DMACAddress+ SourceOffset, 0x10000 ) ; 
               // Set the destination address
               cpu_master_write ( DMACAddress+ DestinationOffset, 0x20000 ) ;
               // Set the transfer size
               cpu_master_write ( DMACAddress+ SizeOffset, 0x20000 ) ;
               
               //and start the transfer
               cpu_master_write ( DMACAddress+ TriggerOffset, 1 ) ;
