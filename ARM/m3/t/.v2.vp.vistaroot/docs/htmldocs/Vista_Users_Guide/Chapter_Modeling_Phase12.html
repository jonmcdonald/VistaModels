Defining a Model

Interface

For the task of defining an interface for a TLM model, all the ports used by the model to send and receive messages and transactions should be specified. Each port is bound to a specific protocol type. Once a protocol is defined for a port, the protocol communication functions (transactions) are automatically created.

For generic models, most ports are pre-defined but, depending on the model type, some must be created manually (see "The Generic Models and Models Catalogue Libraries" for details).


  Note

A Model Interface can also be extracted automatically from an HDL model or by reading a Verilog module or a VHDL entity. To use an HDL unit, see Specifying or Modifying HDL Signal Association in Extracting Power from HDL Models.)
