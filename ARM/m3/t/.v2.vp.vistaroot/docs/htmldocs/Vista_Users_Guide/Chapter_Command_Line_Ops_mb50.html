add_sequential_policy

Adds a 'sequential' policy. (See "Sequential Policy".)

Syntax

add_sequential_policy [-h | -help] [-usage] [{-c | -cause} cause] [-d | -disabled]
[{-l | -latency} latency] [{-s | -sync} sync] [{-p | -power} power] port_or_transaction

Arguments
Common (Optional)
  • [-h | -help]
Provides help.
  • [-usage]
Displays the syntax for this command.
Optional
  • {-c | -cause} cause
Cause port or port.transaction
  • -d | -disabled
A flag for not using this policy
  • {-l | -latency} latency
Latency in clocks.
  • {-s | -sync} sync
Specifies that waits are to be used to synchronize PV and T.
  • {-p | -power} power
Specifies the power value in mW.
Required
  • port_or_transaction
Effect port or port.transaction
