Vista Design Element Path Format

Name

Vista design element path format.

Vista simulation command-line commands recognize the following format for design elements.

Syntax

DesignPath := PathElement (PathElementSeparator PathElement)*

PathElement := [`(' PathElementKind `)'] PathElementName

Description

Path of a design element (`DesignPath') consists of a number of `PathElements', separated by `PathElementSeparators'.

`PathElementSeparator' is either a dot `.', or the current SystemC design path separator.

`PathElementKind' is one or more of the following supported kinds:

  • any — any elements of the design;
  • module — SystemC modules;
  • channel — SystemC channels;
  • primchannel — SystemC primitive channels;
  • hierchannel — SystemC hierarchical channels;
  • port — SystemC ports;
  • export — SystemC exports;
  • process — SystemC processes;
  • binding — port/export bindings;
  • field — fields in classes;
  • base — base classes;
  • deref — dereferenced pointers and references;
  • index — indexes of arrays and pointers;
  • variable — variables in SystemC threads;

To specify more than one kind, the pipe separator (`|') is used.

Each `PathElement' consists of an optional parenthesized `PathElementKind', followed by a possibly empty `PathElementName'. The name cannot be empty if `PathElementKind' is omitted. The name may contain multiple wildcards (asterisk `*' and question mark '?').

Naming

SystemC elements are named by their SystemC design names. Those SystemC elements which are fields in SystemC modules, and whose SystemC design names are automatically generated, are alternatively named by their field names.

Port and export bindings are named by the word `binding', followed by their binding number (thus export bindings are always named `binding0').

If a name of a port or export binding is omitted, the default binding (`0') is taken.

De-referenced pointers and references are named `deref`.

Indexes of arrays and pointers are named by the word `index` followed by their index number.

Class fields are named by their field names.

Base classes are named by their class names.

Aliases

The following aliases are supported:

`[' <number> `]' translates into `.(index)index' <number>

`(' <number> `)' translates into `.(binding)binding' <number>

'->' translates into `.(deref).deref'

Examples
  • top.(channel)*bus.(field)buffer[10]->command.name
  • *.(module)m*.(process)*
  • *.fast*.(port)*.binding0
