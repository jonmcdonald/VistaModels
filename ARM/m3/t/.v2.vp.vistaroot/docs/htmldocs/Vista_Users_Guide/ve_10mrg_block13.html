Creating a New Schematic (Block Diagram)

Models created in the Vista modeling dialog, as well as any SystemC class that inherits from sc_module and is compiled under a specific Vista project, can be instantiated in the Vista Block-Diagram editor.

In order to use models in a block diagram:

  1. Create a model, either from a generic model or from scratch (File > New Model) and save it to a previously created library. For further details on model creation (specifying Ports, Registers, Policies and more) refer to Creating TLM Models.
  1. Create a new schematic by choosing File > New > Block Diagram or clicking the Block Diagram icon.
  1. Copy & Paste models from a library to the new block diagram.
  • Open a library, select a model and with RMB, click the Copy command. Move cursor to the block diagram canvas and with RMB, click the Paste command.

You may also use the Get Component command from the block diagram. This is done as follows:

  • In the Block Diagram window's left side bar, click Get Component to show the panel of component names on the right side of the window.
  • Select the name of the desired component. (A thumbnail image of a component appears at the bottom of the pane when its name is highlighted.)
  • Click Get.
  1. Instantiate the component as desired. The Component and its sockets are displayed in the schematic-editor workspace. Target and initiator sockets are included where relevant.
  1. In the top-level design, you can also include design units that were built outside of Vista by adding their sources to the project. Once compiled, it is possible to create graphical model interface for each of the classes derived from `sc_model' class. To do that:
  • Select the project, or specific source file(s) and with RMB, choose the command: Create Model Interfaces. A dialog box is displayed enabling you to narrow or widen the selection and choose a target library or a directory to save the graphical symbols.
  • Select the target library from the library list (or a directory) and either all possible classes or a limited list and click Ok. The new graphical symbols will be listed in the selected library and can be instantiated in the block diagram.

  Note

Each model interface is defined in a specific file with the extension *.mifc (model-interface). This file specifies the class's header file, model type and ports and it can be further modified.

  1. Click Save (File > Save) to generate the code for the assembly and save it. The first time the assembly is saved, you will be prompted to specify a name for the model (schematic). You may select a project in order to be able to reference the generated code by project.

  Note

The instantiation can be represented during compilation and in the generated code as either a class pointer (default setting) or a class object. The default setting is available in Tools > Options > Setup (Create instantiation as field) and it can be set per instance in the instance's Attributes dialog box.
