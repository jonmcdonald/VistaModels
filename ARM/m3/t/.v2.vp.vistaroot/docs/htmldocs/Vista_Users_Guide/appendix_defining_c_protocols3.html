1 - Declaration of Protocol Parameters

Each protocol includes parameters for three types of HDL signal:

  • master signals- signals driven by the master side of the protocol.
  • slave signals- signals driven by the slave side of the protocol.
  • clock signal- a Clock signal; there should be one and only one such signal for the protocol.

Any value can be parameterized, most notably the width of signals. Default values can be specified for optional signals.

The signal carrying the address of the transactions should be indicated as shown below. 

     module generic #(parameter WIDTH = 32)
       (
         (* clock  *)                         input CLK,
         (* master *)                         input CS, // Chip Select
         (* master *)                         input CMD, // Command
         (* master *) (* default_value = 0 *) input [7 : 0] PRIORITY,
         (* master *) (* default_value = 1 *) input [7 : 0] BURST,
         (* master *) (* default_value = 2 *) input [7 : 0] SIZE,
         (* master *) (* address *)           input [WIDTH-1 : 0] ADDR,
         (* master *)                         input [WIDTH-1 : 0] wDATA,
         (* slave *)                          input [WIDTH-1 : 0] rDATA,
         (* slave *)  (* default_value = 1 *) input STATUS
     );
