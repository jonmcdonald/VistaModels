Creating TLM Models

Traditional SystemC models are created by describing the model's interface and combining its behavioral (functional) and timing attributes in a single model. Timing is typically estimated, and this results in inaccurate performance.

Figure 7‑1. Traditional SystemC Modeling vs. Modeling in Vista

In contrast, Vista provides an environment for creating SystemC models with separate functionality and timing/power layers. Accurate power and timing attributes can be manually defined using a set of Policies or (for advanced users) directly generated from the HDL model (Learning: see "Extracting Power from HDL Models"). The separate timing/power model generated by Vista allows designers to quickly explore performance alternatives without the need to change the core functional model.

For further details on Policies, see Modeling TLM Timing/Power later in this chapter. For further details on Learning, see "Extracting Power from HDL Models".

The SystemC models can be simulated within the Vista environment or exported and plugged into external TLM platforms.

Vista provides both a command-line interface (CLI) and an object-oriented graphical user interface (GUI) that provides quick access to all elements and operations involved in the design/architecture/analysis cycle.

For a CLI command summary, see "Batch and Command-Line Options".
