GDBSTUB Port Spec Parameter

A new parameter has been added to all QEMU based ISS models: gdbstub_port_spec. This parameter should be used as a replacement for the existing gdbstub_port parameter. Although gdbstub_port parameter will be supported for now, it should be considered as deprecated.

The gdbstub_port_spec parameter can have the following values:

  • An integer — functionality is similar to the existing gdbstub_port parameter.
  • An integer range (in the form of 2000..3000)— GDB server port is automatically selected from within the given range; selected value is printed on terminal and also written into Red Sea registry for discovery.
  • auto (case is ignored) — GDB server port is automatically selected from a predefined range; selected value is printed on terminal and also is written into the software debugger (for example, Sourcery CodeBench) registry for discovery.

The parameter can be specified either globally or per each simulated ISS core.
