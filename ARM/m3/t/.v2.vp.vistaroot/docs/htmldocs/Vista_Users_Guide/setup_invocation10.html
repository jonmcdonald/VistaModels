Vista Working Directory

Vista utilizes a specific directory designated as the Working Directory. It is here that Vista stores the makefile generated for each project in the current environment. Whenever a project is somehow changed in Vista, its makefile is updated in the Working Directory.

On Linux/Unix, the working directory is automatically created as needed when Vista is invoked.

Note the following points regarding the Working Directory:

  • You must have write permission to the Working Directory.
  • Each Vista process (GUI) should have its own dedicated working directory. In other words, two or more separate Vista processes must not share the same Working Directory.
  • By default, the Working Directory is the directory named “vista_work” under the directory from which Vista was invoked (Linux/Unix).
  • In the command line used to invoke Vista, you can specify a different Working Directory for the current session by means of the -wd command line option. In this case the command line to invoke Vista would have the format:

Vista-installation-dir/vista -wd path_to_Working Directory &

Note that the path to the Working Directory is relative to the current directory: i.e., the directory from which you invoke Vista.
