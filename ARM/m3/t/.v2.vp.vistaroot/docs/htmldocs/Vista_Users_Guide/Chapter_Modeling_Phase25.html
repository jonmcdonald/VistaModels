PV Variables Propagated to T

Variables that can be read at actual (synchronized) time in the T model can be declared as part of the PV (untimed) model. This is useful, for example, to enable monitoring the progress of a particular algorithm.

The variables must be declared in the PV > Variables tab sheet (see "Specifying PV Variables") or as part of the tcl script for the create_tlm_model command (-m option). Use the command: declare_variable. If you ned to change the kind of one or more state variables from the default value, access them through the Attributes tab (see "Attribute Specifications for Variables and Registers").

The command declare_variable also declares a state variable with the same name in the machine architecture of the T part (see "Machine Architecture Specification"). Each assignment to the mb_variable object is propagated at the right time, specified by a sequential policy (see "Sequential Policy"). The timing code in the policy callbacks (see "State Based Power Policy") can use the state-variable values.
