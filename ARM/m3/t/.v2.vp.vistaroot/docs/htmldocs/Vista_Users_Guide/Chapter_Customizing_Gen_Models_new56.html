Debug Flow Using GDB

  1. Specify the GDB stub port (i.e. 1234) through the model parameters or through an external parameters file (refer to GDB Stub Port (gdbstub_port) for the parameter descriptions).
  1. Start simulation in Vista.
  1. Invoke “xemacs” -load arm_gdb.el (refer to ARM GDB Cross Debugger Configurations).
  1. Press Alt + x.
  1. Type arm-gdb to start the arm cross debugger.

Figure 8‑12. ARM Cross Debugger

  1. At the GDB prompt specify the path to the software ELF image files.

Figure 8‑13. SW Executable

  1. Start the simulation and wait for the following prompt:

“qemu: gdbstub is waiting for SW Debugger connection on port '1234' “

  1. At the GDB prompt, type target remote: 1234.

Figure 8‑14. Connect To Target

  1. Type the following commands
  • <gdb prompt>b main.
  • <gdb prompt>c + enter.
