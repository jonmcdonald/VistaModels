Dynamic Design Configuration

The need for Dynamic Design Configuration

Sometimes it is needed to declare VECTOR of PORTS or MODULES rather than declare them separately.

Currently, you must instantiate MODULES N-times and/or declare each PORT separately which makes the graphical unit more complicated and less intuitive. You must also change the SCHEMATICS each time when the number of instantiated MODULES or PORTS is changed. These limitations avoid dynamic design configurations.

A set of packages are provided to overcome these limitations by allowing:

  • Instantiation of a vector of modules.
  • Creation a vector of ports/sockets/exports allowing to connect it to yet another 'vector'.
  • Definition of a vector size at elaboration-time (from external parameter file).
