Transactor

This is a template component that enables you to configure several transactor components by specifying values for the following template variables:

  • PROTOCOL — can be specified as AHBLite, APB3 or Signal; default is AHBLite
  • SIDE — can be specified as master or slave; default is slave
  • BUS_WIDTH — width of the bus read/write port in bits; can be 8, 16, 32, 64 (default is 32)
  • ADDRESS_WIDTH — width of the address port in bits; can be any number up to and including 64 (default is 32)
  • SIGNAL_TYPE — relevant only for signal protocol; can be specified as bool or sc_logic (default is bool)
  • SIGNAL_WIDTH — relevant only for signal protocol; can be any number up to and including 64 (default is 1)

The supported combinations and resulting transactor components are:

  • AHBLite slave transactor component (protocol=AHBLite, side=slave) — translates the functionality of an AHB slave TLM2.0 port to an RTL level AHBLite slave interface.
  • AHBLite master transactor component (protocol=AHBLite, side=master) — translates the functionality of an RTL level AHBLite master interface to an AHB master TLM2.0 port.
  • APB3 slave transactor component (protocol=APB3, side=slave) — translates the functionality of an APB slave TLM2.0 port to an RTL level APB3 slave interface (conforming to the AMBA 3 APB Protocol Specification v1.0).
  • APB4 slave transactor component (protocol=APB4, side=slave) — translates the functionality of an APB slave TLM2.0 port to an RTL level APB4 slave interface (conforming to the AMBA APB Protocol Specification v2.0).
  • Signal master transactor component (protocol=signal, side=master) — translates an RTL signal of type specified by the SIGNAL_TYPE template variable, to a TLM2.0 signal port.
  • Signal slave transactor component (protocol=signal, side=slave) — translates a TLM2.0 signal port to an RTL signal of type specified by the SIGNAL_TYPE template variable.

For both Signal transactors (master and slave), if the SIGNAL_WIDTH template variable specifies a value greater then 1, then the RTL signal is defined as a vector:

  • for signal_type=bool, an sc_uint signal_width vector is created,
  • for signal_type=sc_logic, an sc_lv signal_width vector is created.

  Note

For AHBLite, APB3, and APB4 transactors, a parameter named rtl_address_shift is declared with a default value of 0. The parameter enables the connection of the TLM2.0 port to the RTL components that define the address signal incompletely.
For example to connect an APB TLM2.0 port to an RTL component that defines the PADDR signal with range [31:2], you should use rtl_address_shift=2.
