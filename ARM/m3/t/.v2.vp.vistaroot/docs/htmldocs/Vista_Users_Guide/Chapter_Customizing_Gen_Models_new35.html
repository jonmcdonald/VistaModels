ARM Cortex A8

Feature List
  • V7A ARM Instruction-Set
  • Thumb-2 Instruction-Set
  • Integrated level 1 D/I Caches
  • Integrated level 2 U Cache
  • V7A Memory Management Unit (MMU)
  • VFPv3 Floating Point Unit
  • PMU - Performance monitor unit (cycles and bus cycles counters are implemented)
  • NEON (A feature combining SIMD and VFP committed in co-processors)
Model Structure

Under cpu.PV:

  • “core” : mb::models::arm::core
  • “l1_dcache” : mb::models::cache
  • “l1_icache” : mb::models::cache
  • “l2_cache” : mb::models::cache
Interface Ports
  • master - Master port
  • n_irq - IRQ input; active low
  • n_fiq - FIQ input; active low
  • n_reset- CPU reset input; active low
  • n_pmu_irq - IRQ output signal for PMU; active high
CPU Specific Parameters

Parameters paths shall be provided as Inst.PV.<PATH>.

  • l1_dcache_size — Specifies L1 data cache size. Default is 32K
  • l1_icache_size — Specifies L1 instruction cache size. Default is 32K.
  • l2_cache_size — Specifies L2 cache size. Default is 128K.
  • endiannes — Propagated for all components. Default is “little”.
  • high_vec — Specifies location of interrupt handle routines: 0 - low addresses; 1 - high. Default is 0.
  • All common core parameters are available under “core.*”.
  • All common cache parameters are available under “l1_dcache.*”, “l1_icache.*” and “l2_cache.*”.
  • pmu_implemented_counters — Indicates which counters are implemented in the PMU. Each bit stands for a different counter. The pmu_implemented_counters value is assigned an AND operator with an internal PMU register. The results show which counters the PMU knows are implemented. Value range varies from 0 to 0xFFFFFFFF.
  • arm_v7_poc — Contains the cache level defined as Point of Coherency (PoC) by the ARM V7 specification. Used, in most cases, by the MVA untility as a final level in which to make cache commands (invalidate and clean). Default value is 2.
  • arm_v7_pou — Contains the cache level defined as Point of Unification (PoU) by the ARM V7 specification. Used as the cache level starting point for operation commands to consider the given cache as unified. Default value is 1.
  • teinit — Specifies whether the core should start working in thumb mode from reset or not. Range is Boolean (0 - false, 1- true). Default value is 0 (false).

NEON

Neon is a feature of ARM that is applied in the Cortex A8 and A9 processors. It is an enhancement SIMD (Single Instruction Multiple Data) combined with VFP.

NEON is implemented in co-processors cp10 and cp11 and is enabled by enabling both SIMD and VFP (using the designated control registers of VFP, combined with the SIMD control registers on cp15). It enables you to make similar parallel computations on the co-processors outside the CPU.

NEON improves the CPU throughput when doing image or video/audio processing, as well as when working in other domains. Using designated registers, NEON enables you to make various computations on multiple pixels at once, instead of a single pixel. It can process up to 8 units of 128 bit (16 byte) data, 16 units of 64 bit data, or 32units of 32 bits data working with banked registers. You can perform the following types of operations using both Integers (signed and unsigned) and floating points (half precision, single precision, double precision):

  • multiplying certain data by scalar
  • adding and subtracting registers from one another
  • shifts left or right
  • combinations of all listed operations

To make use of NEON implementation, the following special compiler options can be used:

  • armcc

armcc -cpu={Cortex-A8 | Cotex-A9} -Ono_special_regs_portregalloc -O3 - -vectorize

The -Ono_special_regs_portregalloc is used for updating all the fields of the current special purpose register (CPSR), including carry, signed, and overflow.

To be sure the compilation translates C/C++ code into NEON assembly instructions add the option -S.


  Note

This command line differs from the original command line which can be found on official sites. The original command line is:
armcc -cpu={Cortex-A8 | Cortex-A9} -Otime -O3 -vectorize
The -Otime prevented some NEON -VFP instructions from being generated and therefore was omitted.

  • GNU/gcc

gcc -O3 -funsafe-math-optimizations -mcpu={cortex-a8 | cortex-a8 }
-mfloat-abi=softfp -mfpu={neon-fp | neon-fp16}

To be sure the compilation translates C/C++ code into NEON assembly instructions, add the option -S.
