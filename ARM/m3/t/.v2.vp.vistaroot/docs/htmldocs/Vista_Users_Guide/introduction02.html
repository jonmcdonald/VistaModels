Vista Platform Elements

The main elements and features of Vista include:

  • Vista TLM2.0 Generic Model Library

Vista TLM2.0 Generic Model Library offers Fast ISS Models for various ARM, PPC, MPC and MIPS processors and additional peripheral models (Cache, Snoop, INTC, Interconnect models and many others). The models are fully functional with Timing/Power policies. For processors, users can also define timing and power per each instruction being executed. Vista support ARM Fast Models Processors library certified and sold by ARM. When integrated into Vista, users can add Timing and Power attributes to the processor models.

  • SystemC/TLM2.0 Native Block-Diagram

Vista includes a native SystemC/TLM2.0 graphical block-diagram for creation of hierarchical designs. Each model compiled into a Vista project can be easily instantiated in the block diagram and connected to other modes. Upon saving, a SystemC code is automatically generated. A unique capability allows users to maintain both graphical and textual views synchronized and updated constantly.

  • Model Simulation and TLM2.0 Transactions in Wave Window

Vista TLM2.0 models are fully synchronized so that their Functional and Timing layers are executed simultaneously on any incoming transaction. This allows debug of either pure functional code or timing at the same simulation run. Vista allows viewing TLM2.0 transactions of any socket in the wave window. This is key for debugging functional execution and transaction activities during simulation. With Vista TSV (Transaction Viewer) it is possible to view all TLM2.0 transactions details, including TLM2.0 GP, phase, status and timing.

  • Vista Analysis Toolset

Vista includes an enhanced analysis toolset for analyzing timing and power profiles of TLM2.0 systems. For timing you can trace Transactions on any instance/port and view statistical metrics of transaction or data Throughput, as well as Transaction Latencies. You can also analyze variables statistics allowing user customized analysis. The analysis toolset can now handle numerous transactions (100's Millions) and large data capacities instantly.

  • Virtual Prototype

Vista generates Virtual-Prototype (an executable object) of the TLM2.0 HW platform to be delivered to the SW group for early SW development and integration. This key capability enables a more efficient HW/SW integration and flow at early design stages. The TLM abstraction of the HW platform results with fast software execution. This will also allow software engineers with an insight into timing and power profiles while running their software.
