Work Procedures

For every Questa “sccom” use a “vista_sccom” wrapper script, and for every “vsim” command, use a “vista_vsim” wrapper script.

For example, assuming Questa “do” commands file: 

     “run.do” :
           vdel -all
           vlib work
           vlog -novopt *.v
           sccom -g my-design.cpp
           sccom -link
           vsim -novopt my-design

There are essentially two options for using Vista on top of Questa (for the design example, with GUI):

  • execute each command, one after the other;

or

  • prepend “vista_run” to Questa scripts, followed by a new shell recognizing the Vista environment.

After this point, you can work with Questa as usual, and all the necessary Questa programs will be wrapped automatically.

Example
  1. setenv PATH $QUESTA_INSTALL_DIR/modeltech/bin:${PATH}
  1. cd $QUESTA_INSTALL_DIR/modeltech/examples/systemc/sc_vlog
  1. $VISTA_INSTALL_DIR/bin/vista_run vsim -do run.do tcsh

Running Simulation and Debugging

Once a vista_vsim script is performed, Vista is invoked, and the Simulate dialog box is displayed with the name of the 'vsimk' executable appearing in the Target field of the General Tab.

To run the simulation:

  • Click the OK button.

Vista's simulation environment opens and, within the message pane, a “vsim” prompt is displayed.

Any of the `vsim' commands may be entered here.

gdb commands can also be issued. To do so, do one of the following:

  • At the vsim> prompt enter “gdb” followed by the desired command;
    for example: vsim> gdb info breakpoints
  • At the vsim> prompt, type “gdb” and <Enter>.

The gdb> prompt appears. At this point, any gdb command can be entered.

To return to the vsim:

  • At the gdb> prompt type “vsim” and <Enter>.
