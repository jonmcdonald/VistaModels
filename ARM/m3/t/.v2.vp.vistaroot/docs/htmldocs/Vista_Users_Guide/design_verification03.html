Customizing gdb Settings

Vista provides an initialization file, .gdbinit, which allows you to configure gdb settings at any of three different levels. When gdb is invoked by Vista's simulation, the .gdbinit file is sourced in the following locations:

  1. First, in the root of Vista's installation, where you can make settings that will be applied to all users.
  1. Next, at /home/$LOGNAME/.gdbinit, where $LOGNAME is the user name for this machine. In this file, gdb settings can be customized for all simulations run by the individual user. These settings will override those defined in location 1, above.
  1. Finally, in the directory in which simulation is run. Because this file is sourced last, its settings have the highest priority, overriding those made in locations 1 and 2. Here you can customize settings for simulating a specific project.
