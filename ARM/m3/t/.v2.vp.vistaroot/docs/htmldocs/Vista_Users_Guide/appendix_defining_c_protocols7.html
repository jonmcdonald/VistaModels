Compiling the Protocol Code

In order to compile the protocol, use the following tcl command: 

     parse_protocol_source {files {library_name "protocols"}}

Note that you can define more than one type of transaction in the same protocol. The name of the transaction function is defined as an attribute inside a transition (such as attributes
(* WRITE *) and (* READ *) above); if the transition belongs to more than one transaction, an attribute must be added for each type of transaction (see “usb” in the section on Predefined Protocols).

If there is only one transaction type in the protocol, the attribute name of the transaction should be in the first transition that issues a message (see “signal” in the section on Predefined Protocols).
