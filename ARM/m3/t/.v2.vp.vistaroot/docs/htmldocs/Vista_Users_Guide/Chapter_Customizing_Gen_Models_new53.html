VLAN - Virtual LAN

The VLAN (Virtual Local Area Network) package is used to enable TLM models to communicate with host physical network. From a structural point of view, VLAN, is composed as a LAN (local network) having a number of clients able to send/receive physical ETH packets to/from one another. Each client can see all the network traffic on the VLAN (just like in a real LAN) and choose to answer some of them (send answer packets into the VLAN). The following clients are available:

  • Modeled network card(s).
  • ARP server — answers to ARP requests according to the local configuration. Never sends physical ARP.
  • IP proxy client side — catches and parses IP packets and sends IP packet payloads over the host client sockets.
  • IP proxy server side + NAT — opens N server sockets on predefined ports, translates the IP addresses and port numbers according to a table, and injects the IP packets into the VLAN.
  • ICPM server for a limited number of IPs — answers to local ping requests.
  • DNS proxy — catches DNS requests and answers using real nslookup.
  • DHCP server

The VLAN API can be found in the mb/physical_io/vlan_client.h file. See Doxygen documentation for more information.

The following parameters (parameters.txt) are available. All VLAN parameters are global parameters.

  • IP address of the host computer as visible from the VP
           vlan:host (default "10.0.2.2")
  • Name of host computer as visible from the VP
           vlan:hostname (default "host")
  • First address assigned by the DHCP server
           vlan:dhcpstart (default "10.0.2.15")

This address is assigned to the first NIC. The second NIC address is assigned 10.0.2.16, and so forth.

  • DNS server visible from the VP
           vlan:dns (default "10.0.2.3")
  • Network IP of the VLAN; NetMask is computed from this value.
           vlan:net (default "10.0.2.0")
  • First MAC address assigned to the first NIC; additional NIC-s are assigned subsequent MAC addresses.
           vlan:macstart (default "52:54:00:12:34:56")
  • Table of TCP Network Address Port Translation
           vlan:tcp_napt (default "")

The value consists of tokens separated by a semi-colon (;) or comma (,). Each token has the following form:

[IP]:Port => [IP]:Port

The left side defines the source (host) IP and Port. The right side defines the VP IP and Port. If the IPs are omitted, then any IP on the host or VP is used.

For example, to forward echo server requests from host port 2007, the following line should be used:

           vlan:tcp_napt = :2007 => :7
  • Table of UDP Network Address Port Translation
           vlan:udp_napt (default "")

The value consists of tokens separated by a semi-colon (;) or comma (,). Each token has the following form:

[IP]:Port => [IP]:Port

The left side defines the source (host) IP and Port. The right side defines the VP IP and Port. If the IPs are omitted, then any IP on the host or VP is used.
