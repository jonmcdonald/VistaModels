Compiling Externally Built Executables

Vista supports simulating a design whose executable has been built externally.

To do so, for compiling and linking, use the following wrapper scripts supplied with Vista:

  • vista_cc, where gcc is to be used
  • vista_c++, where g++ is to be used

Capable of accepting all regular gcc and g++ arguments, these wrappers automatically add the appropriate SystemC headers to the include path; during linking, the suitable SystemC runtime kernel is used. The resulting executable includes the Vista SystemC runtime executable, and it can thus be debugged using Vista.

Both the static (libsystemc.a) and dynamic (libsystemc.so) versions of Vista's SystemC runtime kernel are supplied; vista_c++ decides which version to link based on the flags passed to it. The dynamic version of the runtime will be linked if any of the following flags are passed:-fPIC,
-fpic, -shared, or -rdynamic. However, you can force use of either of these versions by using the appropriate command-line argument.


  Note

If you want to link your executable using the standard OSCI runtime kernel provided with Vista, you will not be able to debug the resulting executable within Vista.
