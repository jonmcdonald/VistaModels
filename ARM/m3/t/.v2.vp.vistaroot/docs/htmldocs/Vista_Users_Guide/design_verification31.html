Event Debugging

Event debugging provides an intuitive way to follow a sequence of events in simulation, allowing an easy way of understanding the actual simulation behavior of the design.

Objects of two kinds are involved: processes (for process execution and process waiting condition) and events (for event notification and triggering).

There are four types of event debugging provided in Vista:

  • Setting breakpoints on events (including signals: break on signal value change)
  • Watching event status in the Vista browser, watch folder and Watch window
  • Acquiring events to wave to view their notification and triggering history
  • Cause from a specific simulation time to the responsible event notification statement in the code

All the above apply to thread processes as well: i.e.: set breakpoint, watch process status and waiting condition, acquire process to wave and “cause” to the specific wait statement.
