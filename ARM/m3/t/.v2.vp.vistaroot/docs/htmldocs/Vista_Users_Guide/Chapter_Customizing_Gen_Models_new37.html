ARM Cortex A9 MP (multi-processor)

Feature List
  • V7A ARM Instruction-Set
  • Thumb-2 Instruction-Set
  • Integrated level 1 D/I Caches
  • V7A Memory Management Unit (MMU)
  • VFPv3 Floating Point Unit
  • PMU (Performance monitor unit) (cycles and bus cycles counters are implemented)
  • Distributor Interrupt Controller
  • Snoop Control Unit
  • Accelerator Coherence Port
  • Local Timers
  • Global Timer
  • Watchdog Timer
  • NEON (A feature combining SIMD and VFP committed in co-processors)
Model Structure

Under cpu.PV:

  • “cpu<k>” : mb::models::arm::cortex_a9up
  • “snoop” : mb::models::snoop
  • “scu” : mb::models::arm::cortex_scu
  • “ic” : mb::models::arm::cortex_ic (PL390 distributor IC).
Interface Ports
  • master0 — Primary master port
  • master1 — Secondary master port
  • acp — Snoop ACP input port
  • irq_<k> — Shared peripheral interrupt inputs; active high; one per SPI
  • n_irq_<k> — Legacy IRQ inputs; one per CPU; Interrupt controller can be configured to pass it to the CPU core; active low
  • n_fiq_<k> — Legacy FIQ inputs; one per CPU; Interrupt controller can be configured to pass it to the CPU core; active low
  • n_reset_<k> — CPU reset inputs; one per CPU; active low
  • n_scu_reset — Snoop controller reset input; active low
  • n_periph_reset — Global Timer reset input; active low
  • n_wd_reset_<k> — Private timer/Watchdog reset inputs; one per CPU; active low
  • wd_reset_req_<k> — Watchdog reset request outputs; one per CPU; active high
  • pmu_irq<k> — IRQ output for PMU; one per CPU; active high
Generation Parameters (Generics)
  • NUM_CPU - Number of A9UP CPU involved
  • NUM_SPI - Number of shared peripheral interrupts (0..224)
  • BITNESS - AXI bus width (32, 64, 128)
  • HAS_MASTER1 - A Boolean specifying whether the MASTER1 is used. If the MASTER1 port is present, transactions will be issued evenly on MASTER0 and MASTER1, so both the initiator ports must be connected.
  • HAS_ACP - Whether the ACP port is present (0, 1)
CPU Specific Parameters

Parameters paths shall be provided as Inst.PV.PATH.

  • dcache_size — Propagated as L1 data cache size for all CPUs. Default is 32K.
  • icache_size — Propagated as L1 instruction cache size of all CPUs. Default is 32K.
  • endiannes — Propagated for all components. Default is little.
  • high_vec — Specifies location of interrupt handle routines: 0 - low addresses; 1 - high addresses. Default is 0.
  • periph_base — Specifies base for private peripheral region address. Default is 0x1f000000.
  • periph_clock — Specifies clock used by timers/watchdogs. Default is 2 * clock.
  • cluster_id — Specifies Cluster ID value available in the SMP Affinity register.
  • scu.control — Specifies reset value for SCU control register. Default is 0.
  • scu.config — Specifies initial value of SCU config register. The number of CPU bit masks (bits 0..3) is replaced by the actual value. If scu.config is not specified, the following parameters are taken into account:
  1. scu.cpu_smp — Indicates CPU symmetric multiprocessor mask (numbers from 0 to15, bit per CPU).
  1. scu.cpu_tag_ram — Indicates CPU snoop tag RAM mask.
  • scu.power — Specifies reset value for SCU power register. Default is 0.
  • scu.filtering_enable — Specifies reset value for filtering enable bit in SCU control register.
  • scu.filter_start — Specifies reset value for SCU filter_start register. Default is 0.
  • scu.filter_end — Specifies reset value for SCU filter_end register. Default is 0.
  • All common core parameters are available under cpu<k>.core.*
  • All common cache parameters are available under cpu<k>.dcache.* and cpu<k>.icache.*.
  • All common snoop parameters are available under snoop.*.
  • pmu_implemented_counters — Indicates which counters are implemented in the PMU. Each bit stands for a different counter. The pmu_implemented_counters value is assigned an AND operator with an internal PMU register. The results show which counters the PMU knows are implemented. Value range varies from 0 to 0xFFFFFFFF.
  • priority_bits — Tells the core which priorities exist, by defining the number of most significant bits to represent a priority. Each priority assigned to an interrupt is being masked with the implemented bits and thus is assigned as the actual priority. Values vary from 4 to 8; 8 means all priority bits are used and there are 256 interrupt priorities known to the CPU (0 to 255), while 4 means that only the four most significant bits of the priority are used and thus there are only 16 interrupt priorities known to the CPU.
  • arm_v7_poc — Contains the cache level defined as Point of Coherency (PoC) by the ARM V7 specification. Used, in most cases, by the MVA untility as a final level in which to make cache commands (invalidate and clean). Default value is 1.
  • arm_v7_pou — Contains the cache level defined as Point of Unification (PoU) by the ARM V7 specification. Used as the cache level starting point for operation commands to consider the given cache as unified. Default value is 1.
  • teinit — Specifies whether the core should start working in thumb mode from reset or not. Range is Boolean (0 - false, 1- true). Default value is 0 (false).

NEON

Neon is a feature of ARM that is applied in the Cortex A8 and A9 processors. It is an enhancement SIMD (Single Instruction Multiple Data) combined with VFP.

NEON is implemented in co-processors cp10 and cp11 and is enabled by enabling both SIMD and VFP (using the designated control registers of VFP, combined with the SIMD control registers on cp15). It enables you to make similar parallel computations on the co-processors outside the CPU.

NEON improves the CPU throughput when doing image or video/audio processing, as well as when working in other domains. Using designated registers, NEON enables you to make various computations on multiple pixels at once, instead of a single pixel. It can process up to 8 units of 128 bit (16 byte) data, 16 units of 64 bit data, or 32units of 32 bits data working with banked registers. You can perform the following types of operations using both Integers (signed and unsigned) and floating points (half precision, single precision, double precision):

  • multiplying certain data by scalar
  • adding and subtracting registers from one another
  • shifts left or right
  • combinations of all listed operations

To make use of NEON implementation, the following special compiler options can be used:

  • armcc

armcc -cpu={Cortex-A8 | Cotex-A9} -Ono_special_regs_portregalloc -O3 - -vectorize

The -Ono_special_regs_portregalloc is used for updating all the fields of the current special purpose register (CPSR), including carry, signed, and overflow.

To be sure the compilation translates C/C++ code into NEON assembly instructions add the option -S.


  Note

This command line differs from the original command line which can be found on official sites. The original command line is:
armcc -cpu={Cortex-A8 | Cortex-A9} -Otime -O3 -vectorize
The -Otime prevented some NEON -VFP instructions from being generated and therefore was omitted.

  • GNU/gcc

gcc -O3 -funsafe-math-optimizations -mcpu={cortex-a8 | cortex-a8 }
-mfloat-abi=softfp -mfpu={neon-fp | neon-fp16}

To be sure the compilation translates C/C++ code into NEON assembly instructions, add the option -S.
