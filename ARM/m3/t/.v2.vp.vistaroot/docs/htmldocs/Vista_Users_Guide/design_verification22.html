Watching Events

Events (charge/discharge) can be watched. Watched events appear in the Browser and Watch folder, as well as in the simulation control Watch tab. The status “charged” or “discharged” is indicated for each event. For charged events, the time that the event will be “discharged” is also indicated. The following notification types are displayed to reflect execution of various notification statements:

  • Immediate notification (for example, e2=discharged (notified))
  • Notification delayed for a specified time period (for example, “e2=charged (discharged at 130 ns)”)
  • Notification delayed until the next delta cycle (for example, “e2=charged (discharged in the next delta cycle)”)
  • Cancelled notification (for example, “e2=discharged (cancelled)”)

When this activity is displayed in the waveform, the two display modes present the data differently:

  • In timed simulation, both notification delayed for one delta cycle and immediate notification appear as vertical bars (spike)
  • In delta cycle simulation, a notification delayed for one delta cycle appears as a wave (span for one delta cycle) and an immediate notification appears as a vertical bar.
Event Triggering/Discharging

Event discharging (event triggering) is always called from the simulator kernel.

If there are no processes sensitive to an event, then its triggering does not advance simulation time or execute simulation cycles. As a result, the simulation history (wave form) will not show this event triggering (event discharge) at the correct time (or simulation cycle); rather, it will appear only at the first time (or cycle) that a process was sensitive to the event.

For example, if an event's notification is delayed for SC_ZERO_TIME (one delta cycle), but there is no process sensitive to it, it could appear in the simulation history (wave form) as though it is delayed for much longer period (i.e., until the first process execution upon which the event triggering is acknowledged).
