2 - Declaration of Protocol Messages and Transactions

Each Protocol generates and accepts messages as either:

  • "master message” - a message that originates at the master side of the protocol (with optional master signals as arguments), or
  • "slave message” - a message that originates at the slave side of the protocol (with optional slave signals as arguments).

The message functions are regular Verilog functions. One can insert code (such as $display) in order to debug the protocol outside of Vista. Vista ignores the body of the function.

The transaction functions are regular Verilog functions; however, they only serve to define the interface of the transaction, and they are never called. You can choose the names of the functions; no predefined names are proscribed.

The following guidelines are illustrated in the code sample below:

  • The arguments of the message functions should be either names of signals defined above, or variables defined in the code.
  • The arguments of the transaction functions should contain (in the following sequence):
  • the signal that contains the address
  • the signal that contains the data
  • all the other signals or variables that were used in the messages that constitute the transaction.
     int BurstCount = 0; 
     // Messages
     (* master *)
     function void G_WRITE_DATA (int wDATA);
     endfunction
     
     (* slave *)
     function void g_write_resp (int STATUS, int BurstCount);
     endfunction
     
     // Transactions
     (* transaction *) (* default_read *)
     function void READ  ( int ADDR,  input int rDATA[], output int PRIORITY, output int BURST, output int SIZE, input int STATUS);
     endfunction
     (* transaction *) (* default_write *)
     function void WRITE ( int ADDR, output int wDATA[], output int PRIORITY, output int BURST, output int SIZE, input int STATUS);
     endfunction
