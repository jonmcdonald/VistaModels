Waveform Window

The Waveform window plots, in time, the behavior of any of the SystemC ports, channels, or objects that are in your design. However, in order for a port or channel to appear in the Waveform window, it must have previously been traced or “acquired.” (See "Acquiring Ports and Channels")

You can acquire any C++ object to a waveform, including full C++ classes and SystemC sc_event objects. For example, transactions of a complete DMA cycle can be viewed in the waveform, by selecting the DMA TLM2.0 sockets (slave, master and irq); As a result, the waveform will show:

  • Configure DMA via slave socket
  • Read from memory source
  • Write to memory destination
  • Issue interrupt
  • Acknowledge interrupt
  • De-assert interrupt

Trace Sockets Option

Sockets tracing is influenced by a new option; the trace socket mode. There are two possibilities:

  • Trace Socket with Data and Phase — In this case, tracing of a socket will also trace all the fields in the Data Structure and Phase and Status folders beneath the socket.
  • Trace Socket Only — This will only trace the socket. This is the default scenario.

Whatever the current option is, it will be possible to individually trace or untrace fields from the Data Structure and Phase and Status folders.

Acquire Sockets

Acquiring a socket results in the following:

  • If only a socket is traced then the single socket's transactions track will be displayed in the wave.
  • If a socket is traced and additional fields are traced in the Data Structure or the Phase and Status folders, acquire socket creates a compound in the wave consisting of the socket transactions track and all the additional fields in the Data Structure or Phase and Status folders that are currently traced. Acquiring fields from Data Structure/Phase and Status folders

Fields from Data Structure or Phase and Status folders cannot be acquired to the wave without the socket they belong to. Acquiring such fields results in a compound being created in the wave that includes the parent socket and all the fields currently traced in this socket's Data structure and Phase and Status folders.

Last_field-name Representation

Transaction time can be long, short, or even a spike (no-time), yet transaction fields can have long enumeration values. To provide an easy way to view the complete value (no matter how long the actual transaction is), Vista waveform fields values span until their value changes. To reflect, that a hint is provided: the value you see is the Last value of that field. To see the actual duration, refer to the transaction time-line field that is above the transaction fields (either red WRITE or green READ).

Pipeline representation in Wave

If a pipeline is discovered on a socket during the simulation, the socket's compound in wave is modified accordingly to represent the pipeline. The socket's compound becomes an array of compounds, each representing a separate channel of a pipeline. For each compound of the array, the same rules defined above will apply and in the same way for every element of the array. See Figure 12‑6.

Figure 12‑6. Pipeline Representation in Wave

Representation of Sockets With Signal Protocol in Waveforms

Sockets that have a protocol signal have a special representation in a waveform. Unlike other socket protocols that are represented in waveforms as read/write transactions with the possible addition of different Data Structure and Phase attributes, because of due their specific semantics, sockets that have a signal protocol are represented in a waveform as signals, not as transactions.

The value of the signal is read from the general payload data at the end of the write transaction and represented according to the corresponding port bit width: bit width 1 is represented as a boolean, bit width 32 as an integer, and so on (see Figure 12‑7).

Waveform Window Regions

From left to right, the Waveform window is divided into the following regions:

  • The leftmost is a column showing the name of each object acquired for simulation.
  • The second column displays the type of the object.
  • The third column displays object value at the time indicated by the currently active locator.
  • To the right of the third column, the waveform pane displays object waves. At the top of this pane is a time ruler.

The topmost part of the Waveform window contains menus and icon bars. The controls at the bottom of the window allow you to adjust various parameters of a Waveform view.

Configuring the Display

The following paragraphs describe the layout of the Waveform window as well as the controls that may be used to view the displayed simulation results.

Displaying the Grid

The grid is a useful means for aligning signal edges. For example, you can display the grid and then add signal edges that are aligned with it. The grid appears as dashed vertical green lines on the waveform pane.

Figure 12‑9. Signal Edges Aligned to Grid

To set the spacing between grid lines:

  1. Choose the View > Set Grid command.
  1. In the dialog box now displayed, specify the offset and cycle (spacing) of the grid.

To toggle grid lines on or off:

  • Use the View > Show Grid flag.

To move a wave edge to the next/previous closest grid line:

  • Select the edge and press the right/left arrow keys while holding down the Control key.

You can also add up to two sets of grid points to existing grid lines. Grid points are indicated by small, red `x' characters and have all the attributes of grid lines. To do so:

  1. Choose the View > Set Grid Points command.
  1. In the dialog box now displayed, enter up to two integers separated by a comma. These integers specify the offset of each set of grid points from the set of existing grid lines in absolute simulation time units, or as a percentage of the spacing between grid lines, depending on whether you select the Time or Percent option button.

Adjusting Waveform Scale

To adjust the Waveform scale:

  • Type the desired value in the Scale text box at the bottom of the Waveform window and click the Scale button.

This value specifies the number of time units between two adjacent notches on the time ruler.

To instantly scroll a particular point of time to the left edge of the waveform:

  • Type a value in the Go To text box and click the Go To button.
