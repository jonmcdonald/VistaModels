Creating a New Project

To create a new project:

  1. Choose the Project > New Project command.
  1. In the New Project dialog box displayed, browse to the location in which to store the project to create.
  1. Enter the name of the project in the File menu name text box and click Save.

The General tab of the Create New Project dialog box is displayed. Set the following additional fields/controls as needed:

  • Project Name — an editable field used to specify the project name.
  • Platform Provider — an editable field used to specify the project provider name.
  • Platform name — an editable field used to specify the platform design name.
  • Platform version — an editable field used to specify the current version of the platform.

  Note

The Project Name, Platform Provider, Platform name, and Platform version are saved in the design executable and extracted when the design is registered by tools that interact with it (for example: software debuggers) and could present it.

  • Project build directory — directory in which to store the files created during a build operation.
  • Project graphical path library — used when you generate graphical components from project modules and the “Don't ask for graphical library path” option is selected. If you deselect the option, you are prompted to enter the name of the directory in which to write the graphical components.
  • Transaction Sequence Recording Support — used to indicate Vista should record transaction activity on ports in your design during simulation. Click the option before compiling or building your project. For more details, see "Transaction Sequence Viewer".
  • Always compile in C++ mode — overrides the Vista default of compiling source files as C files having the suffix .c instead of as C++ files. Selecting this option instructs Vista to compile such files as C++ files (as if their suffix was .cpp.
Defining Path Variables to Specify Directories

The Path Variables tab can be used to define a variable name for any full path in your file system. Doing so lets you make your designs portable.

Vista recognizes the association between any file directory in your system and the path variable that specifies this directory in either of two possible directions. The following example will illustrate this two-way recognition:

  • When you browse to a directory (for example, in the Project > Settings > Files dialog box), and select a file within this directory, the Use Path Variable drop-down list displays the path variable that has been defined for this directory, or the one closest to this directory.
  • In addition, if you activate the Use Path Variable check box, the drop-down list will display all of the path variables that have been defined for this project. When you select a path variable in this list, the appropriate directory and file contents will automatically be displayed in the Select Files directory panes.

Note that this same two-way recognition is implemented from a number of additional contexts enumerated below:

  • When you click the ellipses (...) button at the right end of the list in a dialog box
  • When the Select Directory or Select Object dialog box is displayed

Among the additional contexts in which path variables can be used are in the following tabbed dialog boxes accessed from Project > Settings dialog box:

  • The project build directory specified using the General tab
  • Any of your system directories referenced from the Files tab
  • Any of the header directories for which you specify an include path using the Include Path tab
  • Pre-compiled objects or shared libraries that participate in the design using the Link tab

To define path variables for a specific project:

  1. Click the Path Variables tab. The Path Variables dialog box is now displayed.
  1. In the left-hand column of the dialog box, specify a legal name for the path variable to be defined.
  1. In the adjacent right hand column, click the browse button (...).
  1. Using the Select Directory dialog box now displayed, navigate to a desired directory and click OK. The specified directory will be added to the Path column.
  1. Carry on defining additional path variables as required.
  1. Click OK to save your definitions.

Note that, within the Path Variables tab, you can define a new path variable based on a previously existing one.

You can also globally define path files to be used by all commonly developed Vista projects, or by all the projects developed by individual users. Such definitions are made using the standard makefile format in the file .vista.preMakefile. Whenever a makefile is created in Vista, it sources .vista.preMakefile in the following locations:

  1. First, in the root of Vista's installation, for all Vista users.
  1. Then, on Linux/Unix, at /home/$LOGNAME/.vista.preMakefile, where $LOGNAME is the user name for this machine.

These latter settings will override those defined in location 1, meaning that here every user can individually define path variables settings that will be applied globally to all of his or her projects.
