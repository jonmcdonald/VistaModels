add_pipeline_policy

Adds a 'pipeline' policy. (See "Pipeline Policy".)

Syntax

add_pipeline_policy [-h | -help] [-usage] [{-b | -buffer-size} buffer_size] [-d | -disabled]
[{-l | -latency} latency] [{-s | -sync} sync] [{-p | -power} power] port_or_transaction cause

Arguments
Common (Optional)
  • [-h | -help]
Provides help.
  • [-usage]
Displays the syntax for this command.
Optional
  • {-b | -buffer-size} buffer_size
Buffer size in bytes
  • -d | -disabled
A flag for not using this policy
  • {-l | -latency} latency
Latency in clocks.
  • {-s | -sync} sync
Specifies that waits are to be used to synchronize PV and T.
  • {-p | -power} power
Specifies the power value in mW.
Required
  • port_or_transaction
Effect port or port.transaction
  • cause
Cause port or port.transaction
