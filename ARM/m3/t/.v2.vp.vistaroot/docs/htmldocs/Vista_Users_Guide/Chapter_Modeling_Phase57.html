Timing and Power Policies

To model the timing of power state traversals, a string or enum type mb_variable can be defined. The variable holds the current power state. Timing and power policies can be defined with regards to the variable. Please refer to the Model Builder policies document for more details.

Note: It is important that the user make these variables match key signals in an HDL implementation of the component, in order to help the learning process (see "PV Variables Propagated to T", and "Extracting Power from HDL Models").

  • The data may affect the amount of output data that is generated for a given amount of input data. This does not affect the sequential policy. For a pipeline policy with a split at the output, there are two options:
  • The default implementation is that the ratio of output data to input data is constant for an operation. For example, if, for an operation, N input data words result in M output data words that are split into bursts of K words, then roughly each (M/N)*K input data words will cause an output burst. This behavior is sufficient for bus load, utilization and throughput, if averaged over the appropriate time interval.
  • If the previous behavior is insufficient and there is a need to test different timing scenarios (for example, getting most of the outputs first or most of them last), the user can implement the split policy callback and specify a variable latency (even randomly-generated) in the callback.
