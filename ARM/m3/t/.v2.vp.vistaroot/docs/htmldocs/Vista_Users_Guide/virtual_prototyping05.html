Vista Virtual Prototype for Windows

Vista is now able to create a Virtual Prototype that is created on Linux. But making use of MSVS tools, it can run on Windows OS. This capability is preconditioned as follows:

  1. You must install Microsoft Visual Studio on Windows, on a drive visible by the Linux machine on which Vista is running. The shell variable MSVS_HOME must point to this location. The Microsoft compiler should be found at $MSVS_HOME/VC/bin/cl.exe. The MSVS_HOME variable is used by the vista_init_wine program (see point number 3 below).
  1. Install Microsoft Windows SDK on a drive visible by the Linux machine on which Vista is running. The shell variable MSSDK_HOME must point to this location. The Windows include file is found at: $MSSDK_HOME/Include/Windows.h. The MSSDK_HOME variable is used by the vista_init_wine program (see point number 3 below).

  Note

If mounting a Windows directory on a drive visible to Linux is not an option, copy the Visual Studio installation to a drive visible from Linux. The 32-bit Microsoft Visual Studio installation normally resides in a folder under C:\Program Files on a 32-bit machine, or under C:\Program Files (x86) on a 64-bit machine. The name of the folder is Microsoft Visual Studio 10.0, Microsoft Visual Studio 9.0 or Microsoft Visual Studio 8.0 by default.

  1. The command vista_init_wine (found in the “bin” directory under Vista installation) must be executed to set up the environment of Microsoft Windows emulation (WINE). MS Visual C++ run-time libraries are downloaded and installed at this stage.
  1. You can optionally set the environment variable $VISTA_WINE_HOME to point to the location of the WINE environment. By default Vista will use the default under $HOME/.vista. This VISTA_WINE_HOME variable is used by the vista_init_wine program, and when creating a Virtual Prototype.

  Note

For proper operation on Windows OS, you must install Service Pack 1 for Microsoft Visual Studio 2005, 2008 and 2010.
