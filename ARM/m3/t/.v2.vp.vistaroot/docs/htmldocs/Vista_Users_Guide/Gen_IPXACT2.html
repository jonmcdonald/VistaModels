The IP-XACT Flow in Vista

The IP-XACT flow in Vista is described in details in the following sections.

Predefined IP-XACT Library

For every predefined protocol in Vista, the IP-XACT library contains:

  • A Bus Definition.
  • A TLM abstraction definition named protocol_name_tlm20 that describes the TLM port (socket).

For each model in the predefined Models Catalog, the IP-XACT library contains:

  • A component description that describes the predefined model. This component description's bus interfaces will reference the TLM abstraction definition of the needed bus type definition.

The predefined ahb_master and ahb_slave protocols are represented by a single AHB bus definition/abstraction definition pair.

Adding IP-XACT Bus Definitions and Abstraction Definitions

The Vista Protocol Compiler is enhanced to generate the appropriate IP-XACT Bus Definition and Abstraction Definition when compiling a protocol.

These definitions are saved into the IP-XACT library.

The TLM Abstraction Definition created for a MB protocol contains one single transactional port description (presence, service initiative and service typeName) for each usage case (onMaster, onSlave, ...etc).

Adding IP-XACT Component Descriptions

The Model Generator is enhanced to create an IP-XACT Component description from an existing or a new model builder model.

In the Model form's Generate tab, a new checkbox is added; “Generate IP-XACT component”. When this checkbox is checked, the Model Generator creates and saves the IP-XACT Component description for this model in the specified library.

The generated component includes one busInterface for each socket (port) of the model, referencing the bus definition/abstraction definition for the protocol, the interfaceMode (master for initiator socket, slave for target socket) with the needed address space or memory map description in case the model has memory and/or register information.

Each Component's description contains a model view, specifying the files (.h, .cpp) associated with this component.
