Cache Hit/Miss Ratio Analysis

Vista automatically computes the cache hit and miss ratio attribute of CPU cores that include Icache and Dcache components. Cache tracing should be explicitly specified from the Tracing tab sheet of the Simulation Dialog. Cache Hit/Miss ratio attributes reside under the Core PV and are further split between cache for READ and cache for WRITE. Both Hit and Miss ratio are computed within a specific timing interval as:

(Number of Hit(Miss) Access Events / Total Number of Access Events) x 100

Cache Analysis in LT mode

To enable Cache Analysis under LT mode, specify the following set of parameters in the parameter.txt file:

path_to_CPU_model_component.dmi_enabled = 0

(For example: If path_to_CPU_model_component is Top.CPU1.PV.d/icache; set following in parameter file: Top.CPU1.dmi_enabled = 0)

path_to_memory_component.disable_fast_memory = 1

path_to_memory_component.disable_dmi_memory = 1

Once the DMI (and fast memory) are disabled for the memory components, then accesses to the cache components (external to CPUs) are also enabled.
