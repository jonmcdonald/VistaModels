Binders, Signals and Channels

SystemC block diagrams provide a number of types of connections between blocks: binders, signals, communication channels (channels, for short), and hierarchical channels. Each type can be either placed manually or routed.

You can specify individual data types or use the default value. The default data type is specified in Tools > Options > Default Types.

Binders

The most common way to connect TLM models is to bind their sockets (initiator socket to target socket). This can easily be achieved using the Vista block diagram binder object (appears as a white channel-like object).

Signals

Signals implement SystemC's sc_signal channels. You can specify the data type they carry, and an initial value.

Signals appear as thin black lines.

Signal Attributes

Used to assign attributes to a SystemC signal created in a block diagram.

  • Channel Class: sc_signal
  • Name: The name of the signal. Use the adjacent Visible check box to specify whether or not this name should be visible in the block diagram.
  • Data Type: The data type class of the signal being defined. You can select from the list of C++ and SystemC types, or manually enter a name of a type defined in a SystemC package. If you are defining a signal that is a vector type, specify its width using brackets <>. For example, sc_int<3>.

Note that “Line” colors are assigned separately. Selecting the (…) will open a color table from which a color can be selected.

Communication Channels

The following types of channel representations are found in the Block Diagram sidebar (see Table 9‑1).

  • Binder (White) - this does not represent an object; rather, it simply indicates a binding function between two sockets
  • Channel of type sc_core::sc_signal only (Black)
  • Channel (Red) where you can choose among several channel classes:
  • sc_core::sc_buffer
  • sc_core::sc_fifo
  • sc_core::sc_signal
  • sc_core::sc_signal_rv (resolved vector)

Depicted as bold red lines, communication channels should be used for implementing sc_fifo or sc_buffer primitive channels, or user-defined channels.

Channel Attributes

Used to assign attributes to a channel.

  • Name: The name of the channel.
  • Channel Class: Allows you to specify the class of the channel.
  • Data Type: The type of data to be transmitted through the channel. You can enter any of the standard C++ or SystemC types, or define your own types. User-defined types should be declared in one of the packages used by the unit.
  • Visibility:
  • Name: Show channel object name
  • All: Show channel class name and data type (template) class name
Hierarchical Channels

Hierarchical channels are in fact themselves blocks that can be used to encapsulate other channels, modules, processes, and protocol implementations. In a block diagram, hierarchical channels appear as a special type of component colored green. Port-channel binding of hierarchical channels is performed graphically using the binder tool icons.
