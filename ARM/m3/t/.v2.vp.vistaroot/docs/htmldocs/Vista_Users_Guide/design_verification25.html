Using the Simulation Control Window

The Simulation Control Window integrates, in a single tabbed display, all information related to your current simulation: Processes, Breakpoints, Watches, and Memory. Thus it offers a convenient means of monitoring simulation and debugging your design.

To open the Simulation Control window:

  • Click the Simulation Control icon in the Vista Simulation tool bar:

Using the tool bar or menu commands displayed in the Simulation Control window, you can invoke the same operations that are available from the Main Simulation window.

Process Tab

When you select the Processes tab, a 3-pane window presents information related to the processes, stack, and local variables involved in the design currently being simulated.

The Process window is fully synchronized with the Main Simulation window and commands; When you perform some operation in either of these windows, the display of the other is appropriately updated immediately.

The Process window is divided into three regions: Process Browser, Stack List and Local Variable List

Figure 12‑12. Process Window.

Process Browser

Located in the left side of the window, this pane displays the processes defined in each of the modules involved in the simulation run. The browser distinguishes between three different kinds of processes: Methods (M), Threads (T), and Cthreads (C). The status of each of these is color-coded as follows:

At any given time the status of any process may be represented by up to two colors:

  • The inner color of the circular status icon depicts the current state of the process.
  • The color of the outer ring depicts the previous status of this process in the current or an earlier delta cycle.
  • If an icon is a solid, single color, this indicates that the process has maintained the same status throughout the current delta cycle.

When you double-click a Thread/Cthread or an active Method in the Process Browser, the information in the adjacent two panes is updated, as described below.

Stack List

Located in the upper right side of the window, this pane displays the list of function calls of the process that was double-clicked in the Process Browser. If you now double-click a function call in this list, the display in the Local Variables List is updated accordingly.

Local Variables List

Located in the lower right side of the window, this pane displays the local variables used by any function call that is double-clicked in the Stack List.

Breakpoints Tab

Selecting the Breakpoints tab displays a list of all the breakpoints defined for simulation.

Watches Tab

Selecting the Watches tab displays a list of all of the defined watches.

Memory Tab

You can use this tab to examine consecutive segments of memory during the simulation process. Essentially this tab provides a graphical interface to the gdb “x” command.

Enter appropriate values in the following fields in order to configure how the memory elements will be displayed

.
