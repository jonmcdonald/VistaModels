Vista Tracing Commands

Name

Vista tracing commands: trace, untrace, trace_transactions, untrace_transactions, find, wave, acquire, @.

Synopsis

trace [path | -tree | -kind kind(s) | -name name-pattern | -wave wave-name |
-limit number | -unlimited]*

untrace [path | -tree | -kind kind(s) | -name name-pattern | -wave wave-name |
-limit number | -unlimited]*

trace_transactions [path | -tree | -kind kind(s) | -name name-pattern | -wave wave-name |
-limit number | -unlimited]*

untrace_transactions [path | -tree | -kind kind(s) | -name name-pattern |
-wave wave-name | -limit number | -unlimited]*

find [path | -tree | -kind kind(s) | -name name-pattern | -wave wave-name |
-limit number | -unlimited]*

wave use [wave-name]

wave create [wave-name]

wave open [wave-name]

wave clear [wave-name]

acquire [path | -tree | -kind kind(s) | -name name-pattern | -trace | -limit number |
-unlimited]*

@ time-specification command command-arguments....

Description

Vista tracing commands are used to manage tracing of specified elements in a design during simulation.

The command `trace' requests Vista runtime to start dumping the values of specified data elements into the simulation history file. The dumping occurs on every time or delta cycle, depending on the options under which the simulation runs. These values can then be viewed using the Vista waveform viewer.


  Note

The original Tcl `trace' command is renamed `tcl_trace'.
 

The command `untrace' requests Vista runtime to stop dumping the values of specified data elements into the simulation history file.

The command `trace_transactions' requests Vista runtime to start recording transactions on the specified ports. The transactions can then be viewed in Vista transaction sequence viewer.

The command `untrace_transactions' requests Vista runtime to stop recording transactions on the specified ports.

The command `find' prints full paths to the data elements specified on the command line.

Vista wave commands allow manipulation of Vista wave files.

  • The `wave use' command makes the wave file specified by <wave-name> (which is actually a file in the simulation directory) the current wave file. If the command is invoked without the <wave-name> parameter, the current wave name is returned.
  • The `wave create' command creates the wave file specified by <wave-name> and makes it the current wave file.
  • The `wave clear' command clears the wave file specified by <wave-name> and makes it the current wave file. If the command is invoked without the <wave-name> parameter, the current wave file is cleared.
  • The `wave open' command opens the waveform viewer for the wave file specified by <wave-name> and makes it the current wave file. If the command is invoked without the <wave-name> parameter, the current wave file is opened.

The `acquire' command acquires the specified design elements into the current wave file. If -trace option is specified, the `acquire' command also causes the specified design elements to be traced. Other parameters and options of the `acquire' command are identical to those of the `trace' command.

The `@' meta-command causes the command to be executed at the specified simulation time.

Spaces between `@' and the time specification are optional.

Parameters and Options

path

Each path parameter is a design path with wildcards. See Vista Design Element Path Format for the description of the format.

If the -tree option is specified, Vista tracing commands descend the design tree to find data elements specified by -name and -kind options, starting with roots specified in the path parameters.

If the -tree option is specified, but no <path> parameter appears, the design tree is descended from the top of the design.

If -tree option is not specified, the trace command traces the data elements specified by path parameters, restricted by -kind and -name options.

-tree

See explanation on the path parameter.

The descent through the design tree follows the SystemC design hierarchy, as well as fields in classes and base classes.

-kind kind(s)

Only the elements of the design having the specified kind(s) are considered.

To specify more than one kind use the `|' (pipe) delimeter.

If more than one -kind options appear, all the specified kind(s) are considered.

The following kinds are supported:

  • any — any elements of the design;
  • module — SystemC modules;
  • channel — SystemC channels;
  • primchannel — SystemC primitive channels;
  • hierchannel — SystemC hierarchical channels;
  • port — SystemC ports;
  • export — SystemC exports;
  • process — SystemC processes;
  • binding — port/export bindings;
  • field — fields in classes;
  • base — base classes;
  • deref — dereferenced pointers and references;
  • index — indexes of arrays and pointers;
  • variable — variables in SystemC threads;

-name name-pattern

Only the elements of the design having names satisfying the given pattern are considered. If more than one -name options appear, all the specified patterns are considered.

-wave wave-name

Consider design elements from the wave specified.

-limit number

-unlimited

The number of design elements considered is by default limited by 100.

To change the limit, use the -limit option.

To disregard the limit, use the -unlimited option.

time-specification

The simulation time at which the specified command must be invoked.

The time specification is a number followed by an optional time unit name.

If the time unit name is omitted, the SystemC simulation default time unit is used.

Examples
  • trace top.port*
  • trace *.* -kind port|export -name *_nb
  • trace top.(port|channel)*bus*
  • @10ns trace -wave mydata
  • trace_transactions *.*slave*
  • @ 15 ns untrace -tree
  • wave open mysignals
  • acquire -trace top.m*.(field)counter
