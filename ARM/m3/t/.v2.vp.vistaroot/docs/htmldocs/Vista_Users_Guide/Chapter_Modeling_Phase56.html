Power-up and Power-down Modeling

Power-up and power-down operation is modeled using enable, disable, and reset operations on corresponding domains (or on individual objects, if it is convenient). Use the string protocol to pass global power state information in the form of strings between different models; for example from a PMU controller to peripherals. The following is an example of a behavioral callback for a PMU slave port: 

     void pmu_port_write(std::string str) {    if (str == "ON") {      wic_domain.enable();      core_domain.enable();    } else if (str == "SLEEP") {       wic_domain.enable();       core_domain.disable();         } else if (str == "OFF") {       wic_domain.disable_and_reset();       core_domain.disable_and_reset();    } else {       mb::sysc::warning(str + ": invalid PMU state");    }  }
