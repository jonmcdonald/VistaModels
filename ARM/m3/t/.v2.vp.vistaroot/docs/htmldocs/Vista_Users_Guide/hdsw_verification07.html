Controlling Hardware Simulation

To aid in controlling simulation, a set of gdb monitor commands are available from the Sourcery CodeBench software debugger when simulating designs with Mentor/QEMU-based ISSs in Vista (or Virtual Prototype). The gdb monitor commands are subdivided into sections. To invoke a gbd monitor command, use the following format:

(gdb) mon [SECTION-NAME] COMMAND-NAME arguments

The SECTION_NAME value can be one of the following:

  • process — Used for commands related to the simulation process.
  • systemc — Used for commands related to the SystemC simulation.
  • tlm — Used for commands related to the Transaction Level Modelling (TLM).
  • core — Used for CPU/core specific commands.

The following types of gdb monitor commands are supported:

  • Help Command
  • Reset Commands
  • Breakpoint Commands
  • Software Image Control Commands
  • TLM Control Commands
  • SystemC Control Commands
  • Other Commands
Help Command

mon help [SECTION-NAME]

Prints a help message about available commands.

This command can be invoked globally (without arguments) or more specifically for a given section or command. When invoked globally, the help command generates a list of all available commands.
