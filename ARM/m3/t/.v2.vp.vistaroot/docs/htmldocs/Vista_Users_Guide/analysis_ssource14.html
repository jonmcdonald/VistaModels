Arbitration Time Analysis

Arbitration-time analysis is a Bus-socket attribute provided for the Vista model builder Generic Bus. Arbitration-time analysis is automatically performed once the bus socket is traced and provides the average time taken for transactions to access the bus from a specific bus socket. There are two types of Arbitration time analysis performed: Address and Data.

  • Address Arbitration time (Forward arbitration)

Address Arbitration time is an attribute of a Slave socket of the Bus. Address Arbitration time analysis provides the average time taken between when a Begin_Request is sent from the Slave socket of the bus and when the Begin_Request reaches the Master socket of the Bus. This is the time it takes for a transaction request to actually access the bus.

  • Data Arbitration time (Backward arbitration)

Data Arbitration time is an attribute of a Master socket of the Bus. Data Arbitration time analysis provides the average time taken between when a Begin_Respond is sent from the Master socket of the bus and when the Begin_Respond reaches the Slave socket of the Bus. This is the time it takes for the respond phase from the time it begins until the time the respond gets to the bus slave socket at the other side of the bus.
