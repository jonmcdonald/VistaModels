Analysis from a Virtual Prototype

When a Virtual Prototype is executing you can trace and create analysis data. Analysis data is protected by license. A Virtual Prototype is not only capable of tracing and generating analysis data but can also invoke the Analysis Viewer.

Use the following flow and options to trace and view analysis results created by a Virtual Prototype:

  1. Install VP (installing the Virtual Prototype executable is mandatory for using the Virtual Prototype Analysis flow).
           VP-name.exe -install dirname

dirname must be a new directory

After successful installation, a run.exe is created under the install directory. You will run the created VP instead of the original VP executable. For more information, refer to “Installing Virtual Prototype Executables”.

  1. Set your simulation directory option:

-vp_simdir (or -vp-simdir, -vpsimdir, or -simdir)

If a simulation directory option is not specified, a default simulation directory, called sim, is created in the Virtual Prototype working directory.

The following example creates the simulation results in the directory /mypath/simdir. 

           vp-install-dir/run.exe -vp_simdir /mypath/simdir
  1. Add trace results for the different Analysis types:

The following trace options are available:

  • Trace power for instance and complete design tree:

-vp-trace-power (or -vp_trace_power)

  • Trace transactions timing:

-vp-trace-sockets (or -vp_trace_sockets)

  • Trace user-defined attributes:

-vp-trace-attributes (or -vp_trace_attributes)

  • Trace cache hit/miss ratio:

-vp-trace-cache (or -vp_trace_cache)

If no trace option is specified, VP will not create the simulation directory.

The following example adds trace results for both power and transactions timing under the simulation directory: 

           vp-install-dir/run.exe -simdir /mypath/simdir -vp-trace-power 
              -vp-trace-sockets
  1. Create analysis data:

To create analysis data from the simulation and traced data, issue the command:

prepare_analysis_database.exe

The following example creates analysis data in the specified simulation directory. The analysis data is based on trace options used during simulation.

           vp-install-dir/prepare_analysis_database.exe -simdir /mypath/simdir
  1. Show the analysis results:

To show the analysis results in the Analysis Viewer, issue the following command:

vp_analysis.exe

The following example opens the Analysis Viewer with the analysis results stored in the specified simulation directory.

           vp-install-dir/vp_analysis.exe -simdir /mypath/simdir
