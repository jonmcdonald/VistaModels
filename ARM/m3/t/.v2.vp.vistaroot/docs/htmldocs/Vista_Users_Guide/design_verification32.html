Event Objects (sc_event) in the Vista Browser

Event objects (sc_events defined in the class or in inherited classes) are listed under the “Events” folder. At simulation, the current event status (“charged” - i.e., notified but not triggered yet - or “discharged” - i.e., triggered and before it is notified again) is shown as the event value, including a time expression indicating when it is expected to be discharged and, in some cases, the reason for the discharge (e.g., “canceled”).

One can set a breakpoint on any event under the Events folder, which will break upon notification on that event, from any thread.

For details on breaking on signals and events, see "Breaking on Signals or Events".
