Statistics Modeling Package

The Statistics Modeling Package defines various distributions available for use with model builder Models. These are modeling classes and functions that can be embedded in models to initiate various activities (transactions), random or distributed, into master ports for exploration of performance in a system under various activity scenarios.

The distributions are defined using a common base class (mb_Distribution), which is an abstract class. Each specific type of distribution (gaussian, exponential, etc.) has its own separate class, derived from the base class.

To use a distribution for generating random numbers, you need to define a variable of type mb_Distribution (or one of the derived classes), and construct an object with the appropriate properties.

For this purpose a general function is provided having the following prototype: 

           mb_Distribution* mbCreateDistribution(const char* dist_string);

The function returns a properly-initialized mb_Distribution object created according to the specification in the dist_string parameter, which has the format:

“keyword parameters”

where keyword options are File, Uniform, Gaussian, Exponential, Poisson and Constant.

The keyword is not case sensitive. Each keyword results in the creation of an object of the appropriate derived class of the mb_Distribution class:

  • File: mb_FileDistribution(const char *filepath)
  • Uniform: mb_UniformDistribution(double min, double max, uint64 seed)
  • Gaussian: mb_GaussianDistribution(double mean, double stddev, uint64 seed)
  • Exponential: mb_ExponentialDistribution(double mean, uint64 seed)
  • Poisson: mb_PoissonDistribution(double mean, uint64 seed)
  • Constant: mb_ConstantDistribution(double value, uint64 seed)

The parameters for each type of distribution are the same type and in the same order as needed by the constructors of each class, as specified above. Whenever the distribution constructor takes a seed, it is optional.

In the input string, a space separates the keyword from the parameter list, and parameters are separated by spaces as well.

The following is an example of using mb_Distribution object in a testbench for generating various data sizes and delays: 

           void data_generator_pv::testbench() {
                uint64 waitCycles, genSize;
              uint64 cycles = 1000;
              while(cycles--) {
           waitCycles = waitDistribution->getNextInt64();
                   genSize = sizeDistribution->getNextInt64();
                 …
                 master_read(address, rData, genSize);
                 wait(waitCycles, SC_PS);
                 …
           }     
           }
           
           data_generator_pv::data_generator_pv(sc_module_name _name):data_generator_base_pv(_name)
           {
                m_sizeDistribution = mb_CreateDistribution(sizeDistributionStr);
                m_waitDistribution = mb_CreateDistribution(waitDistributionStr);
               SC_THREAD(testbench);
           }

Note that sizeDistributionStr and waitDistributionStr are parameters and can be defined in the parameter file. One possible definition could be: 

           <pv_instance_path>.waitDistributionStr = gaussian 5 2
           <pv_instance_path>.sizeDistributionStr = uniform 1 5

For more details, see the header file mb_Distribution.h in the directory: mb_install_directory/include/esl/sc_sim.
