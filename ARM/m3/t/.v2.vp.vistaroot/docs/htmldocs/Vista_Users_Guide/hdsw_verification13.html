Options to Trace ISS Instructions

Vista provides different ways through which a QEMU based core can be traced and prints its executed instructions, both verbosity parameters: Trace API or software semi-hosting are possible.

Core Debug Verbosity

The core model provides several informative debug messages; you can control the verbosity level and the type of the required debug information.

The verbosity level and category can be controlled by the hardware model through the relevant configuration file (for example, parameters.txt) or through the software running on the ISS.

The following values (or any OR-d combination), among many others, can be used as a value of the core verbose parameter:

  • INSTRUCTIONS can be used to is print a message on each instruction which is executed.
Example:

Parameters.txt: 

     top.cpu0.PV.core.verbose = INSTRUCTIONS
Core Trace API

The core models offered by Vista provide an enhanced tracing mechanism in which you can use a pre-defined set of callbacks on significant core events.

For instance you can receive a callback when the core has initiated a read transaction or each time the core has entered sleep mode.

Among many others, the following callback can be monitored (called when an instruction is executed by the code): 

     virtual void instruction (uint64_t pc, uint64 opcode, unsigned size, 
        const sc_core::sc_time &annotate)
Semi-Hosting

Vista QEMU based cores provide semi-hosting and various simulation control API.

Among many others, the mb_core_set_verbose_mode and mb_core_get_verbose_mode semi-hosting functions can be used with the value MB_CORE_VERBOSE_INSTRUCTIONS to print a message on each instruction which is executed.

Target Debugger Command

mon core set-verbose STRING

STRING is OR-d list of literals, as defined in verbose parameter documentation; for example, INSTRUCTIONS or READ.
