Invoking Simulation

To invoke simulation on a design:

  1. In the browser, select the top of the design, sc_main source marked by its special icon in the browser
  1. Chose the Simulation > Simulate command or click the Simulate icon . (This will fully elaborate the design and enable the simulation and debugging tool bar.)
  1. In the Simulation dialog box now displayed, set the simulation options and parameters. The parameters that you set are specific per simulation session. They include:

General Tab

  • Target — Select Top to simulate from the top of the current design (the default), or Executable to simulate a design executable external to Vista built using your own makefiles. If you select Executable, you must browse to the location of the executable in your system file directory.
  • Command-line arguments — Specify command-line arguments (argc, argv) for the sc_main function.
  • Parameters File — Use to overwrite the parameters specified in the modeling dialog (see "Parameters"). You must specify the name of the file that contains the new parameter values.
  • Working directory — Specify a non-default working directory for the files to be generated during simulation. (The default simulation working directory is your current working directory.)
  • Simulation Results
  • Directory — Specify a directory anywhere in the file system in which to store the output files from the simulation run. If the specified directory does not yet exist, it is automatically created.
  • Overwrite — If you specify the name of an existing file, use to indicate whether the contents of the existing file should be overwritten.
  • Debugging
  • Stop before elaboration — Select to debug the elaboration process itself (using step, set breakpoints, and any other available commands) before elaboration is completed.
  • Stop after elaboration — Select to halt simulation once the design is fully elaborated. Stopping simulation after elaboration enables you to set breakpoints, open waveforms, or perform other operations before simulation continues. (By default, this option is selected.)
  • Stop At Time — Specify the simulation end-time. Use the adjacent drop-down to specify the time units (fs, ps, us, ms, s).
  • Gdb logfile — Specify the name of a gdb file where information about the simulation run should be stored.
  • SystemC logfile — Specify the name of a SystemC file where information about the simulation run should be stored.

Tracing Tab

Scripting options:

  • Use Scripts - use an initial script, a transcript, or both, for trace/ untrace/ trace_methods/ untrace_methods commands. trace_methods is equivalent to trace_transactions. (Additional commands for scripts and transcripts will be supported in future releases.)

A Transcript records the last session's trace/untrace/trace_transactions/untrace_ transactions commands, allowing you to replay the same session or edit the session and resimulate with slight changes as needed. You can also copy-and-paste transaction commands to a file (tcl script with .tcl extension), which can be used as an Initial script. Standard tcl commands can be included as well.

  • Restore Last Tracing - freeze object tracing at the time of re-simulate (or simulate) and use them as is.

(If you mark the Use checkbox for Initial Script or Use checkbox for Trace Transcript, the Use Scripts option will automatically be selected. If you select the Restore Last Tracing option, the two Use checkboxes will automatically be unmarked.)

TLM Options:

  • Trace all sockets — Select to trace all sockets on TLM models in your design for both verification and analysis.
  • Enable Transaction Sequence Tracing — Enables the tracing of transaction activity on ports in your design during simulation. For more details, see "Transaction Sequence Viewer".

Advanced Options:

The advanced checkboxes enable you to:

  • Trace Delta Cycles — Sets the trace precision to be up to one delta cycle. By default, trace precision is set to simulation time.

This option is valuable if no time elapses during the simulation or if multiple events occur at the same simulation time.

Activating this option slows simulation performance.

  • Enable Memory Allocation Profiling — Enables profiling of dynamic memory allocations for each SystemC process and displays those allocations that were not freed until the current simulation time. For more information see "Virtual Prototyping".
  • Trace all signals — Generates data for ports and channels of type (direct or inherited) sc_signal, sc_clock, or sc_fifo. Following simulation of the design, you are able to acquire the complete simulation data for any port or channel that was not listed in the waveform during simulation.
  • Enable Event Debugging — Saves event information so that it can be viewed after simulation.
  • Enable Cause — When event debugging is enabled, enables you to go to the line of code that triggered an event displayed in the waveform.

Analysis Option:

  • Trace Cache HIT/Miss Ratio — Enables and disables the tracing of cache attributes in the internal caches of the core.

Software Tab

Enables features associated with the software debugger that runs on top of the design (hardware platform).

  • Enable Go to Next Software Instruction — option enables the command (icon) in the debug toolbar. When clicked, the command causes the software debugger (Sourcery CodeBench or a regular GDB) to stop at the next software instruction, after the one that caused the hardware breakpoint to hit.
  • Enable VP PLUS License — when set, option enables all feature sets provided with the VP-Plus license; mainly, hardware visibility from within the software debugger (peripheral registers, local variables, and so forth) and hardware simulation control from the software debugger (the mon commands, including loading a new software image, resetting the system, and switching between LT and AT)

Environment Tab

Enables you to modify environment-variable values before each simulation. Using this you can reconfigure the design for each simulation without the need for modifying the design. You must specify the following:

  • Name — Variable name
  • Value — Variable value

The variable settings (name-value pairs) can be saved via the Save button, and loaded via the Load button.

Representation Tab

Using this tab, you can specify whether the TLM transaction data should be represented in little endian or big endian and whether leading zeros should be removed. By default, TLM transaction data is represented as little endian values.

IP_XACT Tab

The IP-XACT tab enables you to generate an IP‑XACT description for your design. If you check the Generate IP-XACT checkbox, then you need to specify the name of the output directory in which to create the description files.

  1. After you have entered to desired settings, click OK to start simulation.

As simulation proceeds, the simulation clock advances. By default, simulation will execute for the time specified in the project's sc_main. However, it will stop before this time when a breakpoint is reached or when you pause the run.

When simulation has stopped — having finished, having been halted by you, or because a breakpoint was reached — all displayed simulation data is updated: Waveform displays are refreshed and watches that you have defined have their latest values shown. (For details on setting up the Waveform and Watches, see the appropriate sections, below.)


  Note

Vista supports directly invoking an executable simulation session using the -exec command-line switch. When you do so, Vista is invoked with the Start Simulation dialog box displayed, and the name of the executable and arguments that you specified already entered in the appropriate fields. For details, see the table of command-line options in "Invoking Vista".
