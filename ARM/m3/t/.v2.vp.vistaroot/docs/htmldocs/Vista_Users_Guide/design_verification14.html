Working with Breakpoints

Breakpoints provide one of the main tools used in debugging a design.

Defining Breakpoints

Vista offers a number of ways to define breakpoints:

  • attaching breakpoints to one or more lines in a file
  • specifying a breakpoint at a given simulation time
  • loading a pre-defined breakpoint file
Attaching a Breakpoint to a Process or Function

You can attach a breakpoint to either a process or a function. A break-on-process is instance-specific. Thus, in a design unit containing multiple instances of a process, you can specify the specific instance where the break-point should occur. A break-on-function is not instance-specific.

To set a breakpoint at the specified line of the source code, or on selected processes in the browser:

  1. Select the representation of the process or function in the browser.
  1. Choose the Simulation > Breakpoints > Set Breakpoint menu command or the click the Set Breakpoint icon:

  Note

You can set a break-on-function from the expanded header file (.h) or source file (.cpp). This is particularly important for firmware debugging.

Breaking on Signals or Events

Breaking on events is an essential part of event debugging, allowing for monitoring of the occurrences of events during simulation.

Break on event object (sc_event) is supported, as well as break on primitive channel (upon the event that caused the channel to change its value -- i.e., break on signal change).

In cases where a port binds a primitive channel (but not a hierarchical channel), it is possible to specify the break on the port itself.

Once such an event occurs, causing simulation to stop, Vista updates the text editor and highlights the line in the code that was the cause of the event (sc_event notification, or signal assignment).

Break on Port

In cases where a port binds a primitive channel, it is possible to break on the port (which will actually break on the bound primitive channel). When the break occurs, the primitive channel assignment is shown.

Break on Signal Change

In the design browser, you can set breakpoints on hardware signals (any object derived from sc_prim_channel) and on events. A signal's “waiting list” folder contains processes from which the event was activated (signal assignment).

Specifying a Breakpoint at a Given Simulation Time

To specify a breakpoint at a given simulation time:

  1. Choose the Simulation > Breakpoints > Break at Time menu command.
  1. In the Break at Time text box now displayed, enter the time at which the break in simulation is to occur.
  1. Use the Time option buttons to indicate whether the specified time is Absolute, or Relative to the current simulation time.
Loading a Pre-defined Breakpoint File

To load a file of pre-defined breakpoints:

  1. Choose the Simulation > Breakpoints > Load Breakpoints command.
  1. In the Load Breakpoints from the File menu dialog box now displayed, select the name of the breakpoints file and click Open.

For a description of how to create a file of pre-defined breakpoints see "Saving a Set of Breakpoints".
