Creating a Virtual Prototype Using Vista GUI

To create a Virtual Prototype for a design in a Vista project using the Vista GUI:

  1. Select an sc_main file in the Virtual Prototypes folder of a Vista project tree,
  1. Click the Create Virtual Prototype icon, or choose Tools > Create Virtual Prototype.

The Virtual Prototype Dialog opens.

Figure 15‑1. Virtual Prototype Dialog

  1. In the Virtual Prototype Target Platform, choose the Virtual Prototype for Windows option if your target platform is Windows. For more information, refer to Vista Virtual Prototype for Windows.
  1. In the General tab sheet, specify the name of the Virtual Prototype File to generate.
  1. Specify the pathname of the Virtual Prototype Project File.

The settings in the Virtual Prototype Dialog are stored in the Virtual Prototype Project File. This file is a script that can be passed to the vista_create_vp command.

  1. Specify the Virtual Prototype Simulator Program to use to run the actual simulation.

The program and all shared libraries upon which it depends are copied into the Virtual Prototype runtime during generation.

  1. To create a Virtual Prototype Simulator for a design in a Vista project, click Top then specify the name of the program; for example, Project(simple_bus_main.cpp).
  1. To create a Virtual Prototype Simulator for a design in an external executable, click Executable then specify the pathname of the program; for example, /home/me/MyPlatform.exe. Only an external executable that was created with a vista_c++_vp or vista_cc_vp command can be selected.
  1. Specify any command line arguments to pass to the Virtual Prototype Simulator program when a Virtual Prototype is invoked; for example:
           -image $VISTA_VIRTUAL_PROTOTYPES_ROOT/sw.image.

The arguments can refer to Virtual Prototype runtime environment variables. Additional arguments can be passed to the Virtual Prototype invocation by adding them at the end of the command line.

  1. Specify the path to the working directory in the Virtual Prototype runtime; for example, sim_work.

The path can refer to Virtual Prototype runtime environment variables. Relative paths are prepended with $VISTA_VIRTUAL_PROTOTYPES_ROOT/. If a working directory is not specified, the directory where Virtual Prototype is invoked is used.

  1. Specify the Vista Parameters file to use during Virtual Prototype simulation; for example, home/me/MyPlatform.params.

The file is copied into the Virtual Prototype runtime environment during generation.

You can specify an additional Vista Parameters file using the environment variable VISTA_PARAMETERS_FILE. Settings in the parameters file specified via this environment variable take precedence over those in Virtual Prototype runtime parameters file.

  1. In case an installer with “walk-through” agreement file is required, you may specify the file and its path under “License File” field.

It is up to you (Virtual Prototype creator) to decide about the EULA (End-User-License-Agreement) file content (The agreement file may be only the provider file, or the provider append to Mentor EULA file or anything else).


  Note

The user will be required to accept the license terms (EULA) only once - the first time he/she executes the VP executable.

  1. Click the Environment tab and specify any required environment variable operations as shown in Figure 15‑2. If no operation is specified, a set operation is assumed.

Figure 15‑2. Virtual Prototype Dialog Environment Tab Sheet

  1. To copy additional directories into the Virtual Prototype runtime environment, click the Directories tab and specify the Source Path and Target Directory or Environment Variable (not both) as shown in Figure 15‑3.

Figure 15‑3. Virtual Prototype Dialog Directories Tab Sheet

  1. To copy additional files into the Virtual Prototype runtime environment, click the Files tab and specify the Source Path and Target Directory or Environment Variable (not both) as shown in Figure 15‑4.

Figure 15‑4. Virtual Prototype Dialog Files Tab Sheet

  1. Click Save or Save & Generate.

The Virtual Prototype Project File is saved and added to the Vista Projects tree of the Vista GUI.

If you clicked Save & Generate, the Virtual Prototype is generated in the location specified in the General tab sheet.
