Protocols at the TLM level

The model builder in Vista models the communication between the models through a general scheme of high accuracy timing simulation of any protocol using the OSCI TLM2.0 abstraction level. The timing simulation uses the TLM2 approximate timing (AT) non-blocking calls and phases. The target of the Vista model builder is letting the user model the protocol with any accuracy at burst-level. For example, not modeling the transmission of each word.

In order to achieve this modeling, the notion of hardware channels is followed. Modern hardware bus protocols are divided into channels that enable pipelining and parallelization of the transactions. Because most protocols allow multi-word bursts, the data channel(s) are the ones whose utilization limits and defines the throughput of the bus. This is true as other channels (for example, address), pass smaller data volumes and often execute in parallel with the data channel. Although it is sufficient, in principle, to model only the data channel(s) and arbitration in order to get a good approximation of the performance, yet, both data and address channels are modeled.
