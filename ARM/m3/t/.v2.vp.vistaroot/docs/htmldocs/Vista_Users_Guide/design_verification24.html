Displaying Process-Related Information

SystemC Process Types

Vista recognizes three different process types:

  • Methods (M)
  • Threads (T)
  • Cthreads (C)

Prior to a simulation run, the type of a process is displayed in the Design Browser.

Once you run simulation, the process type is also displayed in the Simulation Control window (see below). At this stage, selecting a process from the Processes list located in the Simulation tool bar will display in the simulation console the wait statement at which the process is currently waiting.

In addition, when you click the P icon to the right of the Processes list (or alternatively choose the Simulation > Print Processes command), the Message pane displays the current thread processes and the active process (the process where simulation has stopped).

At any given moment, the status of any process is displayed; this status is color-coded (see "Process Browser").

Spawn Processes

In Vista 2.1 (and later), dynamic (spawn) processes can be viewed in the same way as static processes. (This feature is currently supported for SystemC 2.2.0 only.)

The Vista browser allows you to expand a static process and view the corresponding underlying dynamic processes. Dynamic processes receive the same color-coded status as regular processes.


  Note

Dynamic processes are not shown prior to being spawned; once they return, however, their icon remains in the “terminated” state (gray color).

Break Upon Process Spawn

It is possible to break on spawn of a dynamic process, using the GDB pbreak command. (A GUI option will be available in a later Vista release.)

To use this operation, issue the pbreak command followed by the full path name of the process: for example (see figure below): 

           pbreak Top1.main.1

Upon return, the spawn processes remain with the status of “terminated” (grey).
