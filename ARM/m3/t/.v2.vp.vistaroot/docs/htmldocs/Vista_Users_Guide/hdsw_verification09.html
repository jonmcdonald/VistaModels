Breakpoint Commands

Breakpoints enable you to stop simulation upon accessing a specified hardware object. A hardware object can be a Hardware Register or a Hardware variable (mb_Variable). You can set, query for information on, and delete breakpoints.

Setting a Breakpoint

mon break [r | w] hw-path

Set a breakpoint upon accessing the hardware object specified by hw-path.

Optionally, you can set a break upon either a Read access or Write access. If neither a read or write (r or w) option is specified, the break is applied to both Read and Write accesses. The stop is implemented at the next software instruction (the software instruction after the instruction causing the break).

Example:

mon break w top.dma_inst.reg1

Simulation stops on writing to register top.dma_inst.reg1.

Querying for Information On Breakpoints

mon info break

Displays all hardware paths where a breakpoint is set (including breakpoints in disable mode).

Deleting Breakpoints

mon delete hw-path

Deletes the breakpoint whose path is specified.
