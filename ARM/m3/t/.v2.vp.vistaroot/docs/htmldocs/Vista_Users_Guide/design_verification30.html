Monitoring STL Objects

Vectors, Lists, Queues and other STL containers are represented in Vista's Design Browser in the form of arrays and vectors. STL objects can be watched, traced and acquired to a Vista waveform as any other Vista objects.

Both individual index values and ranges of values can be displayed.

A right-mouse-button menu includes all relevant operations for the array or vector object: Radix, Index Radix, Range and other settings via the Radix & Range Options dialog (see "Radix & Range View Options").

Monitoring STL Arrays

The following STL container types are supported in the Vista workspace:

  • Vector
  • Queue
  • Dequeue
  • List
  • Stack

For both the values and the indexes of the arrays, you can specify the radix: binary, decimal, hexadecimal, or octal).

Using STL Objects
Large Array Ranges

Navigating within very large arrays is facilitated by a hierarchy of Array folders. This applies to C arrays, and to STL types except lists.

If there are over 100 elements in a vector-type array, one or more array folders are used. An array folder groups 100 entries, and puts the remaining entries in corresponding folders.

An array of 1000 is split into ten folders of 100 entries each; an array of 10,000 into ten folders of 1000 each, and so on.

STL List Ranges

An STL list with more than 100 elements contains Begin and End folders. The Begin folder displays the first 100 elements and a folder called Next.

The Next folder in turn shows the next 100 elements and, if there are still elements to display, another folder called Next.

The Begin folder and its subsequent folders thus allow traversing a list from the beginning to the end. Similarly, the End folder and its subsequent folders allow traversing a list in the opposite direction.

Large list (“L_big_list”) after elaboration has a size of 0. For “L_big_list” after 10 ns, one can view list items from the beginning or from the end.

Array Range for Pointers

Specifying an array range for a pointer displays an array of de-referenced elements under the pointer.

Radix & Range View Options

Tree-related properties such as value radix, array index radix, and array range can be configured in the View Options dialog.

The following options/properties are available:

  • Value radix
  • Array index radix
  • Array range

Path Pattern: defines a subset of objects for all settings in the dialog, using a pattern, such as top.s*; can be further constrained in the DataTypePattern fields,

Radix and Index Radix
  • Radix/Index-Radix options:
  • As is (inherited from the parent level)
  • Binary
  • Octal
  • Decimal
  • Apply to Subtree: indicates whether the setting is to be applied to children of the highlighted object as well as the object itself
  • Data Type Pattern: defines a subset of types using a regular expression, such as *int or my_type_*
Range
  • Range options:
  • As is (inherited from the parent level)
  • All: show from the start to the end of the range
  • Range: specifies a min and max value separated by a dash (“-”); can also be a combination of integers and/or ranges (for example: 0, 7, 11, 24-56, 256-512, 1024)
  • Tail: specifies the number of units to display, starting from the end of the array
  • Apply to Subtree: indicates whether the setting is to be applied to children of the highlighted object as well as the object itself
  • Data Type Pattern: defines a subset of types using a regular expression, such as *int or my_type_*
View Options File Syntax

A set of View Option settings are loaded from (and saved in) View Options text files with a specific syntax.

They can be modified via the View Options dialog, or they can be edited directly in the View Options file.

The syntax of the View Options file is similar to the syntax of resource files such as .Xdefaults. Options can be specified for a group of items that have a common name pattern: 

     <tree_item_path_pattern> option_name option_value -flag1 value -flag2 value

where:

  • <tree_item_path_pattern> is a visible path name pattern (for example, the one in the Trace dialog)
  • tree_item_path may include regular expressions such as * (match all) or [a-z] (character range)
  • option_name is a name of a specific option ('radix', 'index_radix', or 'range')
  • option_value is a value for a specific option
  • flag1, and flag2 are additional flags constraining the scope of the setting (for example -type short to have the setting applied only to those entries of type short)

If a tree item has the same view options specified more than once in the file, the last specified value is used.

Examples from a View Options file:

# Specifying a hexadecimal value radix for all the elements in the design hierarchy which are named “address”: 

     *.address radix hex

# Specifying a decimal value radix for all the elements in the design hierarchy named “counter” # and of data type “int”: 

     *.counter radix dec  -type int

# Specifying a binary value radix for all the elements in the design hierarchy with the hierarchy # path starting with “top” and ending with “register” with data type “short”: 

     top*register radix bin  -type short

# Specifying a hexadecimal value radix for all the design hierarchy elements in the subtree top.multiplier: 

     top.multiplier.* radix hexadecimal

# Specifying a hexadecimal array index radix for all the arrays named <anything>”my_array”: 

     *my_array index_radix hex

# Specifying an array range from 0 to 100 for all arrays 

     * range 0-100

# Specifying an array range from 1000 to 2000 for all the arrays in the design hierarchy named “big_array” (this overrides the previous view option specified for all arrays in general): 

     *.big_array range 1000-2000

# Specifying array range 1,2,20-40,100 for all the arrays in the design hierarchy named “my_vector” (this overrides the previous view option specified for all arrays in general): 

     *.my_vector range 1,2,20-40,100
Hierarchy of View Options Files

View Options files have a hierarchy similar to the .vistarc file hierarchy.

  1. The file .vista_view_options resides in Vista's installation root directory and is read by Vista.
  1. After loading this file, the file /home/$LOGNAME/.vista_view_options is also loaded, so that each user can define individual view-options settings.
  1. Each Vista project also has its own view-options file residing in the same folder as the project v2p file and named .project name_view_options. Here, you can specify view options for a specific project.
  1. Each Vista simulation also has its own view-options file residing in the simulation directory and named .vista_view_options.

Values specified in different files have different priorities. The priority of these values (in descending order) is:

  • Values specified in simulation view options file (highest priority)
  • Values specified in the project view options file
  • Values specified in /home/$LOGNAME/.vista_view _options
  • Values specified in the installation root file

If a view option setting appears more than once in a file, the last-specified value overrides any value specified earlier.

View Options with RMB

Each view option can be set via a dialog accessible via the RMB, for a single object selected in the Browser or for a set of selected items.
