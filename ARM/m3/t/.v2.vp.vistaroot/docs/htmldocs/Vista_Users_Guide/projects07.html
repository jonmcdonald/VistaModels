Defining Project Folders

The Folders tab is used to define the folders that will be created for the project, and that will subsequently be displayed as branches for the project within the browser.

In the right-hand column you can specify the file extensions for the different files to be referenced by each of the folders. Defining these extensions means that, when you add files to a new or existing project using the Add by Extension option, the files will automatically be referenced to the appropriate folder. For details, see "Defining Project Files".

Virtual Folders

Note that Vista project folders are not real folders but are, rather, virtual folders. This means that all project files retain their original location in the file system, while Vista project folders simply reference them. For this reason:

  • An operation such as adding files to a project is in fact referencing one or more new files.
  • Cutting and pasting files between folders is in fact simply changing the reference to the file.
  • Removing a file is actually deleting the reference to the file, but not the file itself.
Default Folders

By default, the following folders are defined for a project:

  • Sources (mandatory)
  • Public Headers (mandatory)
  • External Headers (where relevant)
  • Documentation (where relevant)
  • Others (where relevant)
  • Schematics (created automatically) - h file per schematic, when a schematic is saved
  • Simulation (created automatically) - after a successful design build

Optional folders can be renamed, deleted, or used to reference files with extensions different from the defaults.
