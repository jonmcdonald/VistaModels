Invoking Vista

Before invoking Vista, make sure that your license is active. For details, see the Installation Guide provided with this release.

Invoking Vista on Linux/Unix

To invoke Vista on Linux/Unix:

  • Enter the following command line at your system prompt:
     > installation-dir/bin/vista [command_line_options] &

(For command line options, see the Table 2‑3.)

If you want to be able to reference the installation directory using your PATH variable, you can do so by first appending this directory to your system PATH. To do so, enter the following command: 

     > setenv PATH installation-dir/bin:$PATH

The directory from which you invoke Vista will be your working directory. It is here that the initialization file (vista.ini) will be created and stored.

The command line used to invoke Vista can include any of the following options: 

     vista [-license <license-file-or-port>]
        [-wd <working-directory>] 
        [-ini <ini-file-name>] 
        [-p <project-file-name>]
        [-gcc412|-gcc434]
        [-sc22]
        [-exec <path_to_executable> [<arguments>] ]
        [-source <shell_script>]
        [--help-url <URL>]
        [--help-command <Help_Command>]
        [-display <display_id>]
        [-xemacs-exe <path_to_xemacs>]
        [--xemacs-read-only | --xemacs-writable]
        [--external-editor <editor_command_line>]
        [-help]

where:

Parameters

All Vista scripts (vista, vista_run, vista_c++ ...) accept the following parameters:

Group 1
  • -default-tlm1.0
  • -default-tlm2.0draft1
  • -default-tlm2.0draft2 (DEFAULT)
  • -default-tlm2.0
  • -default-tlm-version <version>
Group 2
  • -with-tlm1.0
  • -with-tlm2.0draft1
  • -with-tlm2.0draft2
  • -with-tlm2.0
  • -with-tlm-version <version>
About Group 1 Options

The options in Group 1 (beginning with “-default-tlm”) define the default TLM version Vista will work with, but do not cause vista_c++ to compile with TLM.

To compile with TLM:

  • Specify the +tlm option for vista_c++

or

  • Select “tlm check-box” in the Project Settings dialog.

You can also specify the default TLM version using the VISTA_DEFAULT_TLM_VERSION environment variable (from shell or in .vistarc).

Note that in Vista, the default TLM version is the newest one. Currently it is tlm2.0draft2; in subsequent versions of Vista this default could change.

About Group 2 Options

Options in the second group (beginning with “-with-tlm”) do the same as -default-tlm...options, and in addition they cause vista_c++ to compile with the specified TLM.

Examples

The following commands compile my.cpp with tlm2.0, draft2: 

     vista_c++ -with-tlm2.0 -c my.cpp 
     vista_c++ -with-tlm2.0draft2 -c my.cpp
     vista_c++ +tlm -c my.cpp

The following commands compile my.cpp with tlm2.0, draft1:

  • vista_c++ -with-tlm2.0draft1 -c my.cpp
  • vista_c++ -with-tlm-version 2.0draft1 -c my.cpp
  • vista_c++ -default-tlm2.0draft1 +tlm -c my.cpp

Invoking Vista in Batch Mode

To invoke Vista in batch mode, add the following to the command line: 

     vista -batch VISTA_INVOCATION_FLAGS SIMULATION_FLAGS TARGET_SPECIFICATION -- EXECUTION_PARAMETERS

For further details, see the Batch and Command-Line Options chapter.
