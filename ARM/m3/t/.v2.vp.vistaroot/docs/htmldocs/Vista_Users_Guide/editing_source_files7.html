Specifying a Different Text Editor

Vista supports integrating your own preferred textual editor alongside XEmacs in the Vista work environment. Two environment variables and corresponding command line switches accommodate this functionality. These are described below.

VISTA_EXTERNAL_EDITOR / --external-editor

Use this environment variable or switch to specify the command line that invokes the textual editor that you want to use for creating and editing files within Vista. When you do so, two items will be added to Vista's GUI, either of which can be used to launch the external text editor that you have specified:

  • a new menu command, File > Invoke External Editor
  • a new tool bar icon

For example, to use graphical “vim” as your external editor, you should set the VISTA_EXTERNAL_EDITOR environment variable as follows:

VISTA_EXTERNAL_EDITOR=“vim -g”

(For details on the different means of setting environment variables, see"Controlling Vista Behavior".)

Alternatively, you could override the currently set external editor and specify that “vim” be used during the current work session by entering the Vista command line using the corresponding switch:

vista --external-editor ="vim -g"

Note that, by default, Vista on Linux/Unix uses “vi” in an “xterm” as its external editor. Thus, the default value for the VISTA_EXTERNAL_EDITOR environment variable is: 

     "xterm -e vi % -n"

The percent (%) character may be used as necessary to specify the current file either in setting the value for the VISTA_EXTERNAL_EDITOR or using the ‑‑external‑editor switch when invoking Vista. If the % character is not used, by default the name of the currently displayed file is appended to the end of the command line that invokes your external editor.

VISTA_XEMACS_READ_ONLY / --xemacs-read-only

Regardless of whether or not you have specified an external text editor as described above, XEmacs remains active throughout your Vista session. By setting the VISTA_XEMACS_READ_ONLY environment variable (or using the ‑‑xemacs‑read‑only switch when invoking Vista), XEmacs will display your designs in a read-only mode.

Note that this setting can be overridden using an additional command line switch that is accepted by all Vista scripts: --xemacs-writable.
