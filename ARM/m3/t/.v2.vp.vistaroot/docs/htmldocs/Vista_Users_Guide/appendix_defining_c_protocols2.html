Protocol Code Structure

The protocol is described in standard SystemVerilog 3.1. The protocol “listens” to the HDL signal interface, retrieves the transactions from the signals, and reports them to a transaction database. Attribute declarations are used to characterize the elements of the protocol. See "Protocol Attributes" for additional information, including attributes for protocols involving multiple state machines.

The protocol code can include any legal Verilog code.

There are four main sections in the Protocol code:

1 - Declaration of Protocol Parameters

2 - Declaration of Protocol Messages and Transactions

3 - Declaration of the Protocol States

4 - Definition of the State Machine
