Analysis from a Virtual Prototype

When a Virtual Prototype is executing you can trace and create analysis data. Analysis data is protected by license. A Virtual Prototype is not only capable of tracing and generating analysis data but can also invoke the Analysis Viewer.

The following are the options to trace and view analysis results created by a Virtual Prototype:

  • Simulation directory option:

-vpsim-dir (or -vp_sim_dir or -simdir)

The following example creates analysis results in the directory /my_path/my_sim_dir:

vp.exe -vp_sim_dir /my_path/my_sim_dir vp-trace-power

If simulation directory option is not specified, a default simulation directory called sim is created in Virtual Prototype working directory.

  • There are 3 possible ways to invoke the Virtual Prototype for analysis purposes.

In the following vp.exe is the Virtual Prototype executable and run.exe is an executable available after Virtual Prototype is installed (vp.exe -install DIRNAME).

  • Invoke Virtual Prototype with trace options but do not convert or view analysis database at this stage:

vp.exe (or run.exe if VP is installed) -vp-trace-power -vp-trace-sockets

This is the fastest way to run simulation but additional time is needed to convert (prepare) analysis database when analysis viewer is opened.

If Virtual Prototype is installed, the analysis viewer can be opened using the following command:

vp_analysis.exe simdir

The simdir is an optional parameter, by default the directory ./sim is opened if it exists. The analysis database is created (if needed) and displayed.

If VP has not been installed the analysis viewer can be invoked using the following command:

vp.exe -view_analysis -simdir simdir

While the analysis database is being created, the dialogue box Creating Analysis Database is displayed. To stop database creation, click Abort. If you abort database creation, to recreate the database click Refresh or reopen the simulation directory in the analysis viewer.

While the analysis database is created, the Analysis Viewer GUI is updated periodically with the new simulation time, etc.

  • Invoke Virtual Prototype with trace options and with analysis running in background; this prepares the analysis database in parallel with running simulation. Analysis results can be viewed later as described in bullet 1, but no additional time will be needed to prepare analysis database.

vp.exe (or run.exe if VP is installed ) -analysis -vp-trace-power -vp-trace-sockets

  • Invoke Virtual Prototype in simulation mode and at the same time open the analysis viewer:

vp.exe (or run.exe if VP is installed ) -analysis_gui -vp-trace-power -vp-trace-sockets

While the analysis database is created, the Analysis Viewer GUI is updated periodically with the new simulation time, etc.

At the end of analysis database creation the message “Analysis database created successfully.” is displayed.


  Note

The Virtual Prototype Analysis viewer is supported under Linux. It is possible to create analysis data with a Virtual Prototype created for Windows using the Virtual Prototype trace options (for example, -vp-trace-power) then view the analysis results in Linux.
