Capturing HTML

To generate an HTML representation of the currently active application window, choose File > HTML Capture. The HTML Capture dialog is displayed.

Figure 9‑15. HTML Capture Dialog

You must specify the name of the HTML file in which to save the captured window. Enter the full pathname or click Browse to navigate through your directory tree using the dialog box displayed after you press the Browse button. Click OK.

If the captured window is a graphic editor, Browser window, or Waveform window, the captured representation consists of an HTML file that references a graphic .gif file, which shows the window contents. For graphical units having global data, an additional HTML file is generated, referenced by an HTML hyperlink.

First, Last, Next, and Previous HTML hyperlinks are provided to enable you to move among pages of a multiple page unit. If the text font in the generated image is too small, click the Zoom In hyperlink to enlarge the image.

If the captured window is a textual editor or read-only window, an HTML representation of the entire textual object is generated.

All generated files are saved in the same system directory. The following identifying strings are appended to the specified file names:

  • htm_<system-integer>.html for the HTML file
  • image_<system-integer>.gif for the gif file
  • htm_<system-integer>.txt for the global data file (if generated)
