Command-Line Arguments

The above vista_cc and vista_c++ wrappers accept all the standard Vista command arguments. (For details, see "Invoking Vista".)

In addition, these wrappers accept the following command-line arguments pertinent to the compilation, linking, and debugging of executables:

Table 11‑1. Command-Line Arguments 

Argument

Description

-sc22

Compile for SystemC 2.2.

-sc-static,
-sc-dynamic

Forces use of either the static or dynamic version of the Vista SystemC runtime library during linking. (Static is the default.)

-gcc412,

-gcc434

Compile for gcc 4.1.2, 4.3.4 respectively. Unless otherwise set (see Note below), gcc 4.3.4 is the default for Linux/Unix. These arguments invoke the similarly- named wrappers stored in the Vista installation directory.

-tsv

Compile with support for Vista transaction recording. For details, see the "Transaction Sequence Viewer".

-osci

Linking will be performed using the standard OSCI runtime kernel provided with Vista. As indicated above, when you use this option, debugging the resulting executable will not be possible in the Vista environment.



  Note

Regarding the version of SystemC and gcc to be used, the command-line arguments described in the above table will take precedence over any other previously-set values. (For details, see "Controlling Vista Behavior".)
