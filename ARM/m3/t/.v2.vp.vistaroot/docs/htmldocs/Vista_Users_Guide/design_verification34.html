Post-Simulation Analysis

Running Vista in batch mode, and using Vista scripting capabilities, you can trace design objects in different modes and configurations. For each simulation, Vista opens a simulation directory in which it saves simulation results. It is possible to analyze these results using Vista's post-simulation analysis.

Loading Simulation Results

It is possible to load saved simulation results as soon as Vista is started up:

  1. Select the design top, and click Simulate > Load Simulation.
  1. In the Open dialog, select a simulation directory where simulation results are stored, and click OK. As a result a new simulation tab is added.
  1. You may now find the traced objects in the browser (objects with the trace sign) and acquire them to a waveform, or if you have traced and acquired objects to a specific waveform while running simulation (interactively, via the acquire dialog, via the command-line options, or from batch), you can re-open the waveform that is kept under the “Trace file” icon and view the results.

Note that at any point, you can re-run the simulation interactively, using the Re-simulate command with the same parameters.

Viewing Multiple Simulation Sessions

Vista allows opening multiple simulation history files, with each session in a separate simulation tab. Each simulation session that was saved in a specific simulation directory will keep a simulation icon under its design top. Double-clicking the simulation icon will open a new simulation tab, where traced objects from that session can be viewed.

Post-Simulate Design Executable

If you have created an executable and simulate it, it is possible to load the simulation results in the same way as above, and analyze the results. Note that in this case, the simulation icon will not belong to any project.
