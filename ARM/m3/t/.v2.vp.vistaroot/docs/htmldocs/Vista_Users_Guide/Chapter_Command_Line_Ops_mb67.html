Vista Simulation Prompt

Name

Vista simulation prompt

Description

The Vista simulation prompt is a command-line interface both to the Vista simulation Tcl interpreter, and to the GDB debugger.

The following commands are supported:

-- Vista tracing commands: trace, untrace, trace_transactions, untrace_transactions, find, wave, acquire, @. See Vista Tracing Commands for details.

-- Vista-specific debugging commands: abreak, bbreak, tibreak, pbreak, next_cycle, next_delta, next_process, switch_to_process, add_watch, save_breakpoints, load_breakpoints. See Vista Debugging Commands for details.

-- GDB commands (continue, next, step, finish, frame, up, down, print, etc.)

-- `gdb' command prefix.

-- `gdb' prompt-switching command.

-- `vista' command prefix for the GDB prompt.

-- `vista' prompt-switching command for the GDB prompt.

Notes
  • The commands are interpreted by the Tcl interpreter first. Therefore, dollar signs (`$'), braces ( `{' and `}'), and square brackets ( `[' and `]' ) have special meaning, and to be used literally must be escaped using a backslash (`\').
  • Commands which are not recognized by the Tcl interpreter are forwarded to GDB.
  • The command prefix `gdb' causes the rest of the command line to be processed as a GDB command.
  • If `gdb' is used without arguments, the GDB prompt appears.
  • On the GDB prompt, the command prefix `vista' causes the rest of the command line to be processed as a Vista command.
  • If `vista' is used without arguments (on the GDB prompt), the Vista prompt appears.

Original Tcl commands `trace', `break' and `continue' are renamed to `tcl_trace', `tcl_break' and `tcl_continue', respectively.

Examples

  • (vista) br myfunc()
  • (vista) trace *.* -kind port
  • (vista) c
  • (vista) gdb p $pc
  • (vista) gdb
  • (gdb) where
  • (gdb) vista trace -name *sig*
  • (gdb) vista
  • (vista) next process
  • (vista) next time
  • (vista) next delta
  • (vista) pbreak top.master_b.main_action
