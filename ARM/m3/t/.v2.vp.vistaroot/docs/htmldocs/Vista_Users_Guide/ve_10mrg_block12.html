Additional Tips

1. Renaming Design Objects

You can change the name of design objects in graphics.

After “save” a correct replace of the old name will follow inside the correct segments.

Warning: make sure renaming is unique and are different than existing object names.

2. Code Organization

It is recommended to place code which belongs to a specific segment within the segment, so should the segment is deleted the code belong to it, will also be deleted.

3. Check before Compiling

In general, generated code is not guaranteed to successfully compile and may require user's modifications.

4. C Preprocessor Macros

All clauses are legal (compilable) C preprocessor macros.

5. Multiple constructors

Generated code will include only one constructor. The user may add additional constructors, yet it is recommended to encapsulate long constructor code in a special constructor function avoiding multiplying long lines of code.
