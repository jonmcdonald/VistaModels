Description of Segments

The following segments are described in this section:

  • Includes Segment
  • Module Definition Segment
  • Fields Declaration Segment
  • Initialization Declaration Segment
  • Destructor Declaration Segment
  • Elaboration Declaration Segment:
Includes Segment 
              $includes_begin;
              $includes_end;

The Includes segment encompasses the list of the include files required by the components. A component's include file is the file where the module class is declared. You may add, delete or modify any include statement.

Module Definition Segment 
           $module_begin("module_name");
           ... module definitions ….
           $module_end;

A Module-definition segment contains the following sub-segments: 

              $initialization_begin
               $initialization_end
           
               $elaboration_begin;
               $elaboration_end;
           
               $destructor_begin;
               $destructor_end;
           
               $fields_begin;
               $fields_end;
Fields Declaration Segment 
           $fields_begin
           $fields_end;

A Fields segment is used to declare class data members.

Data members are grouped using the following sub-segments: 

              $port("name")
              ... port declaration
              $end
           
              $export("name")
              ... export declaration
              $end
           
              $socket("name")
              ... socket declaration
              $end
           
              $channel("name")
              ... channel declaration
              $end
           
              $component("name")
              ... component declaration
              $end
           
              $field("name")
              ... other field declaration
              $end

Where “name” is a name of the element taken from graphical block diagram.

Members declarations have the form:

ClassName fieldName;

For example: 

              $socket("S5");
              tlm::tlm_initiator_socket<> S5;
              $end;
           
              $component("MEM");
              MEMORY_pvt *MEM;
              $end;
Initialization Declaration Segment 
           $initialization_begin
           $initialization_end

An Initialization segment is used to initialize the module data members (components, ports, sockets, signals etc). It is generated at the beginning of module constructor(s).

Example of an initialization segment: 

              $init("name"),
              ... initialization code
              $end

By default, each element is initialized with one constructor parameter “name”. The user can modify the initialization section specifying additional constructor parameters.

Destructor Declaration Segment 
           $destructor_begin;
           $destructor_end;

A Destructor segment is used for deleting components. Each deleted element is enclosed in $destructor_component sub-segment 

              $destruct_component("name"),
              ... destruction code
              $end

Example: 

              $destruct_component("MEM");
              delete MEM; MEM = 0;
              $end;
Elaboration Declaration Segment: 
           $elaboration_begin;
           $elaboration_end;

The Elaboration contains the elaboration code. It is generated inside the module's constructor(s).

This segment contains two parts: component-creation and binding-definitions segments.

Example of module creation segment: 

              $create_component("CPU");
              CPU = new CPU_pvt("CPU");
              $end;

Example of binding definition segment: 

              $bind("DMA->irq","CPU->irq");
              DMA->irq.bind(CPU->irq);
              $end;

As mentioned above, you can modify any user code specifying constructor parameters, changing the order of component creation within the elaboration segment and similar.
