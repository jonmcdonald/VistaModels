Partition of Code into Segments

The source file contains Segments. Each segment typically has three parts: 

           
           $begin clause 
           ... body (including user code) clause
           $end clause

For example: 

           $component("DMA");-- Begin clause
           // user comment-- body, user code 
           DMA_pvt *DMA;
           $end;      -- End clause
  • “Begin” and “End” clauses starts with a '$' sign and may optionally contain one ore more parameters enclosed in parenthesis.
  • Creation and removal of segments is controlled from the graphical block diagram. Any new element created in the Block Diagram editor may lead to the creation of one or more segments. Deletion of a graphical element deletes its segment (including any user code defined within that segment).
  • User code defined between segment pairs will not be modified when graphical modifications are made.
  • Segments can be hierarchical: that is, one segment can contain one or more nested segments.
  • You may add/modify code before, within or after of each segment.
  • Do not modify “Begin” and “End” clauses, including their parameters. In addition, maintain pairs of “Begin” and “End” clauses: For each “Begin” clause there must be an “End” clause at the same hierarchical level of segments.
  • You may change the order of sibling segments (segments belonging to the same segment hierarchy).
