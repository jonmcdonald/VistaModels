ARM Cortex R4

Feature List
  • V7A ARM Instruction-Set
  • Thumb-2 Instruction-Set
  • Integrated level 1 D/I Caches
  • V7A Memory Management Unit (MMU)
  • VFPv3 Floating Point Unit
  • PMU (Performance monitor unit) — cycles and bus cycles counters are implemented
  • SIMD (Single Instruction Multiple Data) — integers of various widths operations and floating points
  • TCM
  • VIC Interface — Interface to VIC-PL192
  • Level 2 Cache Interface
Interface Ports
  • axi_master of BITNESS width
  • axi_peripheral_master of 32 bit width; for peripherals on AXI bus protocol
  • ahb_lite_master of 32 bit width; for legacy peripherals on AHB lite bus protocol
  • acp — Snoop ACP input port
  • n_irq — IRQ input; active low
  • n_fiq — FIQ input; active low
  • n_reset — CPU reset input; active low
  • n_pmu_irq — IRQ output for PMU; active low
  • irq_addr — 32 bit wide input from VIC that holds the ISR address to jump to when interrupt happens
  • irq_addr_v — input signal from VIC indicating the validity of the address in irq_addr
  • irq_ack — output signal to VIC indicating whether core acknowledged ISR jump address; active high
CPU Specific Parameters
  • l1_dcache_size — Indicates L1 data cache size. Default is 32K.
  • l1_icache_size — Indicates L1 instruction cache size. Default is 32K.
  • endiannes — Propagated for all components. Default is little.
  • high_vec — Specifies location of interrupt handle routines: 0 - low addresses; 1 - high addresses. Default is 0.
  • All common core parameters are available under core.*
  • All common cache parameters are available under l1_dcache.* and l1_icache.*.
  • arm_v7_poc — Contains the cache level defined as Point of Coherency (PoC) by the ARM V7 specification. Used, in most cases, by the MVA utility as a final level in which to make cache commands (invalidate and clean). Default value is 1.
  • arm_v7_pou — Contains the cache level defined as Point of Unification (PoU) by the ARM V7 specification. Used as the cache level starting point for operation commands to consider the given cache as unified. Default value is 1.
  • pmu_implemented_counters — Indicates which counters are implemented in the PMU. Each bit stands for a different counter. The pmu_implemented_counters value is assigned an AND operator with an internal PMU register. The results show which counters the PMU knows are implemented. Value range varies from 0 to 0xFFFFFFFF.
  • mpu_number_regions — Contains the number of memory regions in the MPU: 0, 8 or 12. A 0 indicates the MPU is inactive. Default value is 12.
  • atcm_size — Indicates size of the ATCM memory.
  • btcm_size — Indicates size of the BTCM memory. When there are 2, the actual size is btcm_size * 2.
  • tcm_base — Indicates the base address if the TCM is not mapped to address 0; either ATCM or BTCM are mapped to address 0 and the other is mapped to tcm_base. Default value is 0x40000000.
  • initrama — Indicates whether ATCM should be initialized and enabled at reset. Value is a Boolean (0 - false, 1- true). Default value is false (0).
  • initramb — Indicates whether BTCM should be initialized and enabled at reset. Value is a Boolean (0 - false, 1- true). Default value is false (0).
  • loczrama — Indicates whether ATCM is mapped to zero (and BTCM is mapped to tcm_base) on reset or not. Value is Boolean(0 - false, 1- true). Default value is false (ATCM is mapped to tcm_base and BTCM is mapped to 0).
  • teinit — Specifies whether the core should start working in thumb mode from reset or not. Range is Boolean (0 - false, 1- true). Default value is 0 (false).
