Functional Model (PV)


  Note

After defining ports, registers, and parameters, users can automatically generate a PV model template in which they can model the un-timed behavior. The following sections define the modeling basics used to model such functionality. For additional details, please refer to the Vista Modeling Guide.

The Vista model builder automates the creation of the communication layer through which the model sends and receives transactions. As the model builder understands the principle of bus protocols, modeling the communication becomes relatively simple, and more importantly, correct by definition. When writing a functional model the user can use “read” and “write” functions for initiating transactions through a designated master port.

In addition, a callback function is called when a transaction function is called for a specific slave port.
