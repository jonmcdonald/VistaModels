Symbol Configuration

You can create default associations for the pins of a symbol, as well as define parameters to override default values. Saving a set of reusable configurations for instantiation is helpful since when this is done it is not necessary to define a full set of ports, parameters and template parameters for each component instantiation.

To create a default configuration for a symbol:

  1. In the Symbol Editor, select a symbol.
  1. Choose Attributes > Symbol Configuration.

The Symbol Configuration dialog opens.

  1. In the Ports tab, specify expressions for one or more ports.
  1. In the Parameters tab, specify parameter values: a default configuration value, and optionally an overriding case-specific value, for each parameter.
  1. In the Template Module Parameters tab, specify values for template parameters.
Updating and Replacing Component Symbols

If you change the instantiated symbol or the instantiated component for a symbol that already appears in a block diagram, the symbol instance in the block diagram is not automatically updated. To update an instance, you need to use the Attributes > Component > Update Component command. Or to change to a different symbol (even one that instantiates a different unit), use the Attributes > Component > Replace Component command. Both commands are described in “Instantiated Components”.
