Viewing Analysis Reports

You can generate ASCII reports of the full transaction event dump and/or summary analysis results for a simulation run.

  • The Transaction Report is a file containing an ASCII list of all transactions in the TLM database. If an instance is specified, only transactions of that instance are dumped.
    Transactions and/or attributes are included, as well as start and finish time.
  • The Summary Report compiles data for the full simulation run.

Examples of the reports are provided below. This powerful tool enables the user to understand accurately the effect of different system configurations on the data traffic and on the performance of the system.


  Note

When simulating a model generated in Vista, it is recommended to use a general trace script. This includes the Ports and Power data in Analysis graphs or reports.

To create a full transaction report:

  1. In the Vista browser, highlight the simulation data that you want to analyze.
  1. From the Tools menu, select Open Analysis Reports.

The Analysis Reports dialog appears:

Figure 14‑14. Analysis Report (Transactions)

  1. In Report type, select Transaction Report.
  1. In Simulation Directory, specify the location of the simulation data.
  1. In File path, specify a destination system folder and name for the report file.
  1. In Select instance, if Selected instance is selected, specify the hierarchical instance path to be included in the report.
  1. In the Options section of the dialog box, select Transactions and/or Attributes, and optionally type the Start time and Finish time. (If not specified, the entire simulation time range is used).
  1. Click OK.

The Transaction Report is created.

To create a summary report for one or all instances:

  1. In the Vista browser, highlight the simulation data that you want to analyze.
  1. From the Tools menu, select Open Analysis Reports.

The Analysis Reports dialog appears:

Figure 14‑15. Analysis Report (Summary)

  1. In Report type, select Summary Report.
  1. In Simulation Directory, specify the location of the simulation data.
  1. In File path, specify a destination system folder and name for the report file.
  1. In Select instance, if Selected instance is selected, specify the hierarchical instance path to be included in the report.
  1. Click OK.

The Summary Report is created.
