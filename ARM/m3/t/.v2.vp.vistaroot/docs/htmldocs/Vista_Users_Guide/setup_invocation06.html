Controlling Vista Behavior

You can control how Vista behaves in a number of ways:

  • You can set environment variables in your system or Vista shell. (See "The Vista Workplace".)
  • You can set values for environment variables that will have a higher priority than the settings made using the system or Vista shell by means of the .vistarc file. This file, which is stored in the root of Vista's installation directory, is sourced by all Vista scripts. Using this means you can set values for all Vista users at your location.
  • Each Vista user can further customize Vista behavior by setting values for environment variable in the file /home/$LOGNAME/.vistarc, where $LOGNAME is the user name for this machine. Because this file is sourced after the .vistarc file stored in Vista installation directory, values set here have a higher priority than those set in the latter file.
  • Yet a higher level of priority is possible by creating a shell script, having the same format as the .vistarc file, which is stored in the /bin/sh directory. You can subsequently implement the values set for environment variables in this shell script using the - source command line option. (See "Invoking Vista".)
  • Finally, you can override any values set using environment variables (methods A through D) by invoking the appropriate command line switch (see "Environment Variables for Controlling Behavior") when running any of the Vista scripts.

For details about preventing Vista from sourcing the .vistarc files, see "Preventing Sourcing of .vistarc Files"
