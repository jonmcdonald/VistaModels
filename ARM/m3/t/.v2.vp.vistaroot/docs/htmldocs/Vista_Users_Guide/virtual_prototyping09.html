Installing Virtual Prototype Executables

In addition to running a Virtual Prototype executable, you can:

  • install the Virtual Prototype executable into a specific directory; the directory should not exist prior to the install operation
  • install and automatically run the Virtual Prototype executable
  • specify the root directory where all Virtual Prototype executables will be located.

Use one of the following options:

VP-name.exe -install dirname

Installs the Virtual Prototype in the specified directory.

VP-name.exe -install-and-run dirname

Installs the Virtual Prototype in the specified directory and launches it automatically.

VP-name.exe -vprootdir dirname

Specifies the root directory for all Virtual Prototype installations. All Virtual Prototypes will be installed under this root.

After installing a Virtual Prototype, the executable file run.exe, which runs the Virtual Prototype, can be found in the specified directory.

The following Errors can occur:

  • If you specify any of the options without specifying a directory name, the following error is displayed:
           "Missing install directory. Exiting"
  • If you specify the -install option with an existing directory, the following error is displayed:
           "Directory <dirname> already exists. Exiting."

After a Virtual Prototype is installed and run, if the Virtual Prototype is waiting to connect to a software debugger (for example: CodeBench), the software debugger will locate the Virtual Prototype and show all Virtual Prototype registered information (as it was specified by you when creating the Virtual Prototype).
