Syntax Conventions - Modeling

Most commands within the Vista model builder are available either from the GUI or the command line. A tcl script and commands can be run.

The following syntax conventions are used when describing commands;

  • The logical path to any model builder object should be specified in the following format:

root (TLM) library_name object_name

for example:

/TLM/mymodules/ahb

  • Arguments preceded by a dash '-' are optional; For example:

-verbose

  • For a list argument, surround the list with “” or {}; For example:

{file1 file2}

"file1 file2"

  • When entering an argument on the command line, unless otherwise indicated in the command usage statement, the first letter of argument can be used rather than typing the entire argument name; for example; you can type -h, instead of -help, to obtain help on a command.

This manual uses the following command usage line syntax conventions.

An ellipsis follows an argument that may appear more than once. Do not include the ellipsis when entering commands.
