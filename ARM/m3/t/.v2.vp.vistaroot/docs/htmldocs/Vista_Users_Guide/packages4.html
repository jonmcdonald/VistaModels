Package Description Format

A package description is defined by means of a Perl package that corresponds to the parallel Vista package. Such a package description serves two purposes:

  • It informs the Vista GUI about the existence of the package.
  • It specifies the gcc command line switches needed to implement the package during compilation and linking.

The following conventions apply to package descriptions:.

  • The name that you assign to a Vista package must be a valid Perl package name; i.e., it may comprise Latin letters, digits, and underscore symbols, but must not start with a digit. (For more information about Perl packages, see your Perl documentation.)
  • The name of the package description file must conform to the form: package_name.pm
  • The hierarchical name of the Perl package name must be: vista_packages::package_name
  • The first line of the Vista package description file must have the form: 
           package vista_packages::<package_name>;
  • If a Vista package requires the presence of another Vista package, the following line must be present in the header of the Vista package description file: 
           require vista_packages::<name_of_the_required_package>;
  • In order to add options to the gcc compilation command line, the following API must be appear in the topmost scope of the package description file: 
           add_package_compilation_options vista_compiler_options "<option1>" 
                [, "<option2>", ... "<optionN>"];

For example, 

           
           add_package_compilation_options vista_compiler_options "-Wall", 
               "-fpermissive";
  • In order to add options to the gcc link command line, the following API must appear in the topmost scope of the package description file): 
           add_package_link_options vista_compiler_options "<option1>" [, "<option2>", 
                ... "<optionN>"];

For example, 

           
           add_package_link_options vista_compiler_options "-Xlinker", "-E", "-lsocket";

Note that for both compilation and linking, specified options will be processed after the command line switches passed by the user, but before the options specified by required Vista packages.

  • The last line of the Vista package description file must be 
           1;

i.e., the digit one followed by a semicolon.
