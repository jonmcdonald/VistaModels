Building a Project

Building a project involves creating objects from each of the source files that are referenced by this project, then linking the resulting objects, and building an executable. Two commands are available in the Build for building an entire project on its own:

  • Build Project compiles only those sources in the project that require compilation, and then links the resulting objects and builds an executable.
  • Rebuild Project re-compiles all of the project's sources prior to the build operation. This command first deletes all the project's objects (.o files), then recompiles all sources and builds the design.

Another command, Build Executable, can be used to build a single sc_main source in a project, instead of the entire project, thereby shortening the build time.

Vista also provides additional commands for building a project together with other projects on which it depends.

To build a project:

  1. In the browser, select any object belonging to the project.
  1. Choose the appropriate command or, for Build Project, click the equivalent icon .

During the build operation Vista automatically moves the project's sc_main file from the Sources directory to the Designs directory in the browser. When the build has been successfully completed, an appropriate message is displayed in the Messages window.

To build a single sm_main source and not the entire project:

  1. In the browser, select the icon of the sc_main source.
  1. Choose the Build > Build Executable command.
