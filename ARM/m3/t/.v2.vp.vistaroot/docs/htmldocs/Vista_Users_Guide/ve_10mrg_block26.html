Instantiated Components

Within a block diagram, you can instantiate any unit defined in your Vista libraries as a component. The component is depicted by a symbol (either default or user-defined) that has predefined pins.

Components allow working in a bottom-up approach: An already-existing unit which performs some specific function in the overall design, and which itself may be hierarchical, is instantiated in a new design.

The Block Diagram Editor provides the following navigational actions related to a component instantiation:

  • Go To Text - in the text pane, Vista displays the associated code
  • Open - opens the header of the component or the Modeling dialog for the model
  • Go to Instantiation - opens the header file of the top design level, at the relevant component instantiation statement
  • Go to Declaration - opens the header file of the top design level, at the relevant instance declaration statement
