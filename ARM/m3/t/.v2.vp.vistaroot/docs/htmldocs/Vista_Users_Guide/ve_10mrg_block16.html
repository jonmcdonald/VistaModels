Adding Sockets

The Block Diagram Editor contains sidebar tools for adding initiator and target sockets to the components in the schematic.

To define sockets, create a binder that is ended dangling (unconnected); A socket connector or “none” connector may be created. (This is based on an option in Tools > Options > Setup, Default mode for new communication connectors.)


  Note

TLM sockets bind initiator sockets to target sockets or hierarchical binding, hence only the Binder object can be used to bind sockets.

Initiator/Target Socket Connector Attributes
  • Name: The name of the port/socket connector.
  • Signal Name: The name of the signal or channel to which the connector is attached.
  • Port Class: The name of the class, as defined for the port/socket
  • Data Type: The data type class name of the channel
  • Visibility: Use this check box to set whether the name of the connector will appear in the graphic diagram.

The appearance of a connector depends on the specified directionality.

SocketTarget and SocketInitiator Mode Parameters

When a binder is selected in a SystemC design, SocketTarget and SocketInitiator Modes are available and contain the following parameters:

  • Class Name: Defines the type of C++ class member of the SystemC module, that will appear in the generated SystemC module. You can do one of the following:
  • Select one of the standard TLM socket initiator/target classes.
  • Type a user-defined class, derived from a standard socket initiator/target class.
  • Template Parameters: If the option selected in the Class Name field is the C++ class/structure template, type the template parameters in this field.
  • Initialization: Type the constructor parameters, used for initialization of the C++ class member, defined for the given connector.
  • Bind-this: Selecting this option causes the selected connector to be bound (using SystemC bind functionality) to the parent SystemC module.
