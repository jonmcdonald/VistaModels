The PV Model Base Class

All the PV models are derived from the same base class that provides common utilities. The base class mb_module provides the following utilities:

  • mb_sync— Calling this function causes the calling callback/thread to wait until all the pending master_port/mb_variable/mb_register read/writes finish.
  • get_current_token— Each callback and thread that is not a part of a CPU model saves the token of the most recent action that activated it:
  • For a callback, it is the token associated with the read/write transaction that triggered it
  • For a thread, it is the token associated with the Notify call to an mb_event
  • get_clock— Returns the clock cycle time (in Picoseconds) of the port whose index it gets as a parameter. The port index is known through the port_enum enumeration type that is declared in the model's pv_base class.
  • get_port_name— Returns the name of the port whose index it gets as a parameter.
