Predefined Protocols

Vista comes with a number of pre-defined protocols. The protocols that are currently provided are:

  • ahb — Two protocols, ahb_master and ahb_slave, are provided. These protocols support communication in buses based on AMBA_AHB.
  • ahb_master — Protocol should be used for ports that initiate transactions on an AHB bus; for example, CPU and the DMA port of a DMA controller.
  • ahb_slave — Protocol should be used for ports that are targets of transactions; for example, memory and the registers port of a DMA controller. The signal HPROT is not modeled.
  • apb — Supports communication in buses based on AMBA_APB.
  • axi — Supports communication in buses based on AMBA_AXI. Protocol is implemented with five state machines; one each for the hardware -AXI channels.

Protocol parameters include WORD_SIZE (default is 32), which represents the width of the data (WDATA, RDATA...).

There are extra auto-generated parameters for a port instantiating this kind of protocol. For more details, refer to AXI parameters under Auto-Generated General Parameters.

The signals CACTIVE, CSYSREQ and CSYSACK are not modeled.

  • generic — A generic bus protocol for use if the bus protocol for the implementation is not yet known.
  • ram — Used to represent access to a ram. Protocol was developed to match a specific Mentor CatapultC interface.

Protocol parameters are ADDR_WIDTH (default is 19), which represents the width of the address, and DATA_WIDTH (default is 8), which represents the width of the data.

  • signal — Used to represent signals (for example, interrupts), and is implemented as a regular TLM2.0 socket.
  • tcm — Supports communication in buses based on ARM TCM (tightly-coupled memory) between ARM CPUs and memories.

Protocol parameters are ADDR_WIDTH (default is 15) which represents the width of the address, DATA_WIDTH (default is 72) which represents the width of the data, and CYCLE_ACCURATE (default is false) which sets whether or not the protocol will be cycle accurate.

  • usb — Supports communication in both USB1.0 and USB2.0 standard protocols.

Protocol defines two functions: read and write. The address in these functions is used to represent the USB end point number (most significant 32 word of the TLM address), and the address within the end point (least significant 32 word of the TLM address).

  • DMA_request — A point to point (stream) protocol used primarily to transfer control data between a DMA controller and a DMA. The protocol does not have an address parameter in either the Vista MB Convenience API for a master port or in a callback for a slave port.
  • SSP (Simple Serial Protocol) — used for master-slave communication on single-master serial buses.
  • four_phase — Has two channels, one address and one data, that are shared between READ and WRITE transactions. The address timing events are: BEGIN_REQ and END_REQ. The data channel timing events are: BEGIN_RESP and END_RESP.
