Component

The Component description created for a MB model contains the following elements:

  • VLNV - Vendor: defined in the Generate tab; library: specified in the save form; name: the name of the model; version: defined in the Generate tab.
  • busInterfaces - a list of bus interface elements, one for each port (socket) of the model.

Each busInterface element contains:

  • name - the name of the port (socket).
  • busType - reference to the bus definition representing the protocol specified for the port (VLNV).
  • abstractionType - reference to the TLM abstraction definition for the bus definition.
  • interfaceMode - is a master for an initiator socket, or a slave for a target socket.
  • connectionRequired - true if the port's Bind Mode is mandatory, false if optional.
  • channels - a list of channel specifications. A channel is a list of bus interfaces present in a component that belong to the same channel.
  • memoryMaps - the list of memory maps referenced by slave type bus interfaces.
  • model - describes a single TLM view for the component. The view contains the following elements:
  • name - the name of the component
  • envIdentifier - the string “systemC:mentor.com:MB.TLM2.0”, indicating that the view is a SystemC implementation, of TLM2.0 models created by the Vista model builder.
  • hierarchyRef - in case the component is hierarchical, this element references the VLNV of the hierarchical design describing the top level of the internal hierarchy.

For non hierarchical component, the following elements will be generated instead of the hierachyRef element:

  • language - “systemC”.
  • modelName - the model name (same as the sc_module class name).
  • fileSetRef - the list of files references for this model (all .h and .cpp files in the component's fileSet list.
  • parameters - the list of parameters (both user and auto-defined) for the model.
