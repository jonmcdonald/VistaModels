Setting DMI Mode for LT Simulation

You can set the DMI mode for both for QEMU and AFM models.

To enable DMI mode, set the parameter dmi_enabled to 1; to disable DMI mode, set the parameter dmi_enabled to 0. When dmi_enabled is disabled, DMI is not used even during LT simulations; all transactions (peripheral IO, memory, and interrupts) are visible in the waveform and analysis.


  Note

Simulation performance is affected when DMI mode is set to disable.

The dmi_enabled parameter can be used both as a global parameter (without prefix) and as a CPU parameter (with CPU instance-name prefix).

Limitation

In many cases, transactions are visible as spikes (no transaction latency present), but all needed values (address, data, and so on) are still visible as Recent values under the expandable port bundle.
