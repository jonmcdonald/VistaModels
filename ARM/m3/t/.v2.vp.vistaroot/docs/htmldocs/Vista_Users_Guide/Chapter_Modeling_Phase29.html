Modeling Timing and Power (T)

This key step in the modeling workflow, described in detail in the following sections, is where performance (timing and power) are incorporated to complete the comprehensive TLM design solution. To generate the model with this T component, see "Generating a Model".

Timing and power Policies are used to manually define performance in a top-down manner, when no HDL is available. Use of policies is a powerful mechanism for defining complex timing in an intuitive way, as the policies provide accurate reflections of various timing architectures: pipelining, concurrency, latencies.

The policies are defined using the concept of PV events. PV events include the following: read/write transactions on master / slave ports and read/write operations on model builder registers and PV variables.

The policies timing is defined relative to clocks. The PV or T code can change the timing and power consumed by the policies by calling the following functions: 

     void set_internal_clock(config::uint64 clock); \\ The internal clock of the model.
     void set_clock(unsigned port_index, config::uint64 clock); \\ The clock of the port.

The following is an example of the use of such a call: 

     void cb_write_clock_freq(unsigned int newValue) 
           {
             set_internal_clock ( newValue ) ;
             // Modeling an internal power controller that changes the clock frequency upon SW command
           }
