Creating a Timing (T) Model

The Vista model builder enables the modeling of complex timing behavior in one of two ways: specifying it manually, or (for advanced users) creating it automatically through its learning engine from an HDL design.

After a Model Interface list of ports and their types (protocols) has been defined, the code for the timing (T) model can be created.

As a starting point, the model builder builds:

  • _model.h and .cpp files (for example, CPU_model.h)
  • SystemC template (or “skeleton”) .h and .cpp files with all necessary definitions and callbacks (for example, CPU_t.cpp).

The template provides declarations and commented skeleton function calls. The logic for the various relevant functions can then be filled in as needed.
