ARM Cortex R5

Feature List
  • V7A ARM Instruction-Set
  • Thumb-2 Instruction-Set
  • Integrated level 1 D/I Caches
  • V7A Memory Management Unit (MMU)
  • VFPv3 Floating Point Unit
  • PMU (Performance monitor unit) — cycles and bus cycles counters are implemented
  • SIMD (Single Instruction Multiple Data) — integers of various widths operations and floating points
  • TCM
  • VIC Interface — Interface to VIC-PL192
  • Level 2 Cache Interface
  • Snoop Control Unit
  • Accelerator Coherence Port
Interface Ports
  • axi_master of BITNESS width
  • axi_peripheral_master — 32 bit wide; for peripherals using AXI3 protocol
  • ahb_lite_master — 32 bit wide, for peripherals using AHB Lite protocol.
  • acp — Snoop ACP input port; bitwidth wide
  • n_irq_<k> — Legacy IRQ input; one per CPU- Interrupt controller can be configured to pass it to the CPU core; active low
  • n_fiq_<k> — Legacy FIQ input; one per CPU- Interrupt controller can be configured to pass it to the CPU core; active low
  • n_reset_<k> — CPU reset inputs; one per CPU; active low
  • n_pmu_irq_<k> — IRQ output of CPUs for PMU; active low
  • irq_addr_<k> — 32 bit wide input from VIC of CPUs that holds the ISR address to jump to when interrupt happens
  • irq_addr_v_<k> — input signal to CPUs from VIC that indicates the validity of the address in irq_addr_<k>
  • irq_ack_<k> — output signal from CPUs to VIC indicating whether core acknowledged ISR jump address; active high
CPU Specific Parameters
  • dcache_size — Indicates L1 data cache size. Default is 32K.
  • icache_size — Indicates L1 instruction cache size. Default is 32K.
  • endianness — Propagated for all components. Default is little.
  • high_vec — Specifies location of interrupt handle routines: 0 - low addresses; 1 - high addresses. Default is 0.
  • All common core parameters are available under core.*
  • All common cache parameters are available under l1_dcache.* and l1_icache.*.
  • arm_v7_poc — Contains the cache level defined as Point of Coherency (PoC) by the ARM V7 specification. Used, in most cases, by the MVA utility as a final level in which to make cache commands (invalidate and clean). Default value is 2.
  • arm_v7_pou — Contains the cache level defined as Point of Unification (PoU) by the ARM V7 specification. Used as the cache level starting point for operation commands to consider the given cache as unified. Default value is 1.
  • pmu_implemented_counters — Indicates which counters are implemented in the PMU.

Each bit stands for a different counter. The pmu_implemented_counters value is assigned an AND operator with an internal PMU register. The results show which counters the PMU knows are implemented. Value range varies from 0 to 0xFFFFFFFF.

  • cluster_id — Specifies Cluster ID value available in the SMP Affinity register.
  • mpu_number_regions — Contains the number of memory regions in the MPU: 0, 12 or 16. A 0 indicates the MPU is inactive. Default value is 12.
  • atcm_size — Indicates size of the ATCM memory size in bytes if one is applicable. The size must be an integer that is a power of two. You can configure the TCM memory to be of zero size to disable the TCM functionality. Only one of the following values is permitted: 0, 4K, 8K, ... 512K, 1M, 2M, 4M, 8M. Default is 32K.
  • btcm_size — Indicates size of the BTCM memory in bytes if one is applicable. The size must be an integer that is a power of two. When there are 2, the actual size is btcm_size * 2. You can configure the TCM memory to be of zero size to disable the TCM functionality. Only one of the following values is permitted: 0, 4K, 8K, ... 512K, 1M, 2M, 4M, 8M. Default is 32K.
  • tcm_base — Indicates the base address if the TCM is not mapped to address 0; either ATCM or BTCM are mapped to address 0 and the other is mapped to tcm_base. Default value is 0x40000000.
  • initrama — Indicates whether ATCM should be initialized and enabled at reset. Value is a Boolean (0 - false, 1- true). Default value is false (0).
  • initramb — Indicates whether BTCM should be initialized and enabled at reset. Value is a Boolean (0 - false, 1- true). Default value is false (0).
  • loczrama — Indicates whether ATCM is mapped to zero (and BTCM is mapped to tcm_base) on reset or not. Value is Boolean(0 - false, 1- true). Default value is false (ATCM is mapped to tcm_base and BTCM is mapped to 0).
  • teinit — Specifies whether the core should start working in thumb mode from reset or not. Range is Boolean (0 - false, 1- true). Default value is 0 (false).
  • axi_periph_base — Indicates the base address required for AXI peripherals that are to be connected to the axi_peripheral_master port.
  • axi_periph_size — Indicates the memory size required for AXI peripherals that are to be connected to the axi_peripheral_master port. A 0 indicates the port is inactive. Only one of the following values is permitted: 0, 4K, 8K, ... 512M, 1G, 2G, 4G. Default is 0.
  • initppx — Enable bit for the AXI master peripheral port.
  • ahb_lite_base — Indicates the base address required for AHB Lite peripherals that are to be connected to the ahb_lite_master port.
  • ahb_lite_size - Indicates the memory size required for AHB Lite peripherals that are to be connected to the ahb_lite_master port. A 0 indicates the port is inactive. Only one of the following values is permitted: 0, 4K, 8K, ... 512M, 1G, 2G, 4G. Default is 0.
  • initpph - Enable bit for the AHB Lite peripheral port.
