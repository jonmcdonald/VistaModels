Scalable Modeling Semantics

Mentor Graphics scalable TLM model approach is based on a clear separation between the communication layer, functionality, and architectural aspects reflected by timing and power. In a scalable model, communication is defined using ports and protocols; a Programmable View (PV) layer models pure (untimed) functionality; a T layer defines timing and power information reflecting the architectural structure. The PV and T layers are combined into a single PVT model.

Once a layered model structure is achieved, architectural exploration becomes a routine modeling practice rather than non-practical theory. A layered model structure allows designers to maintain the functionality source intact during the entire design cycle, while implementation aspects are changed and refined over time.

The PV/T layered approach also allows single-source modeling for various design tasks. Software validation and virtual prototyping can “switch off” the T layer to run pure functional HW and SW simulation. Architects can `turn on' the T layer for architecture analysis, where timing and power are needed.

Policies can be augmented by C/C++ code to achieve dynamic behavior. They provide an intuitive and powerful mechanism for:

  • quickly exploring various model architecture alternatives and their effect on timing and power behavior
  • creating bus-protocol-independent models
