Modeling TLM Timing/Power

With the Vista modeling capabilities, functionality is modeled by the user, while timing and power can be defined manually using Policies. The resulting timing (T) model can then be simulated side-by-side with the functional model.

Since timing/power modeling is far more complex than functional modeling, the modeling of an entire system using this approach is greatly simplified, while nevertheless resulting in highly-accurate models.

Defining Policies

Policies are sets of attributes that allow users to define Timing and Power through a simple GUI associated with TCL scripts.
