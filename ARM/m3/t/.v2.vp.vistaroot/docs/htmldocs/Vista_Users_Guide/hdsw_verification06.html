Performing Hardware-Software Tight-Debugging

When Vista simulation is stopped at a breakpoint (break-in-hardware), and one or more Mentor/QEMU-based ISS cores are connected to a software debugger (Sourcery CodeBench or a regular GDB), click the Next Software Instruction icon (shown in Figure 16‑1) to stop the software debugger at the software instruction that occurs directly after the one that caused the hardware breakpoint to hit.

Using this feature, you can step through the hardware and software to get a clear view of the data transfer between them.


  Note

This feature needs to be enabled from the simulation dialog as it could affect simulation performance slightly. To enable, choose the Software tab in the simulation dialog and click the Enable Go-to-Next-Software-Instruction option.
