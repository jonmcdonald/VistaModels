Hardware Software Interactions

This section provides information on how to interact with a Vista Qemu ISS based virtual target platform while running your software. The Virtual target should provide a control/access API into Sourcery CodeBench to enable hardware visibility control, tight-debugging and file system interactions. Information on the following is provided:

  • Setting the VP-Plus License
  • Accessing Hardware Objects
  • Performing Hardware-Software Tight-Debugging
  • Controlling Hardware Simulation
  • GDBSTUB Port Spec Parameter
  • Manipulating Files Between Target OS and Host

For additional information on using Sourcery CodeBench to debug and bring up various software stacks on a Vista Qemu ISS based virtual target, see the Sourcery CodeBench-Vista VP Plug-in User's Manual using the Sourcery CodeBench Help menu.
