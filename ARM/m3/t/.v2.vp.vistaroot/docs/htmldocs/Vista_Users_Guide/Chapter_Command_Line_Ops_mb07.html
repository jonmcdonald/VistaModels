add_template

Adds a model builder template (also known as a Generic Model).

Syntax

add_template [-h | -help] [-usage] [{-t | -type} type] [-f | -force] [-generics] [-pure] logical_path source_directory_path

Arguments
Common (Optional)
  • [-help/-h]
Displays help.
  • [-usage]
Displays the syntax for this command.
Optional
  • [{-t/-type} type]
Specify the specific model type (for example: CPU, BUS).
  • [-f | -force]
Specify to force overwrite when the logical path already exists.
If the logical path you have specified is a non-standard location (not under the Generic_Library directory), then specify to create the logical path.
  • [-generics]
Specify a list of pairs to use as the generic defaults.
  • [-pure]
Specify to set a flag to always overwrite the PV file (pure template, no user code).
Required
  • logical_path
Logical path to the template (e.g. MyGeneric/Base1 will be located in library MyGeneric, and the generic model name is Base1).
  • source_directory_path
Source directory for the template data files.
