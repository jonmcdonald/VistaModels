Work Procedures

There are three basic work procedures that can be used to simulate your Vista designs on NC-Sim:

  • Performing Unified Compilation, Elaboration, and Simulation (Simulating with vista_ncsc_run)
  • Simulating Designs Using Dynamic Linking
  • Simulating Designs Using Static Linking
Performing Unified Compilation, Elaboration, and Simulation

In this procedure compilation, elaboration, and simulation are invoked with a single wrapper script, vista_ncsc_run, as follows: 

     vista_ncsc_run source_files [Options]

where:

  • source_files indicates the source files to be compiled (using a path, wildcards, etc. as appropriate
  • Options are any of the command line options supported by NC.
Simulating Designs Using Dynamic Linking

When you will be dynamically linking your design, the procedure involves four phases performed sequentially. In each phase you invoke the appropriate wrapper script using the commands as are specified below:

I. Compilation (vista_ncsc wrapper script) 
     vista_ncsc source_files [Options]

where:

  • source_files indicates the source files to be compiled (using a path, wildcards, etc. as appropriate).
  • Options are any of the compilation command line options supported by NC.
II. Linking (vista_nc_ld wrapper script) 
     vista_nc_ld [Options]

where:

  • Options are any of the linker options.
III. Elaboration (vista_ncelab wrapper script) 
     vista_ncelab [Options] top_level_module

where:

  • Options are any of the elaboration options supported by NC.
  • top_level_module is the name of the topmost module of the design to be elaborated.
IV. Simulation (vista_ncsim wrapper script) 
     vista_ncsim [Options] top_level_module

where:

  • Options are any of the simulation options supported by NC.
  • top_level_module is the name of the topmost module of the design to be simulated.
Simulating Designs Using Static Linking

When you will be statically linking your design, the procedure involves five phases performed sequentially. In each phase you invoke the appropriate wrapper script using the commands as are specified below:

I. Compilation (vista_ncsc wrapper script) 
     vista_ncsc source_files [Options]

where:

  • source_files indicates the source files to be compiled (using a path, wildcards, etc. as appropriate).
  • Options are any of the compilation command line options supported by NC.
II. Rebuilding ncelab (vista_nc_ld wrapper script) 
     vista_nc_ld -o new_ncelab_executable [Options]

where:

  • -o is the output flag to be followed by new_nc_elab_executable, i.e., the name of the new ncelab executable that is to be built.
  • Options are any of the supported ncelab build options.
III. Rebuilding ncsim (vista_nc_ld wrapper script) 
     vista_nc_ld -o new_ncsim_executable [Options]

where:

  • -o is the output flag to be followed by new_ncsim_executable, i.e., the name of the new ncsim simulator that is to be built.
  • Options are any of the supported ncsim build options.
IV. Elaboration (ncelab_sc) 
     ncelab_sc new_ncelab_executable top_level_module

where:

  • new_ncelab_executable is the name of the ncelab executable that was built in phase II.
  • top_level_module is the name of the topmost module of the design.
V. Simulation (vista_ncsim wrapper script) 
     vista_ncsim -ncsim new_ncsim_executable top_level_module

where:

  • new_ncsim_executable is the ncsim simulator executable built during phase III.
  • top_level_module is the name of the topmost module of the design.
