Basic Modeling Object Enable and Disable State

Each modeling object (port, register, memory range, etc) has an enable and disable state. The behavior of the modeling object is different depending on its state.

  • Enabled modeling objects:

All activity (read and write transaction, call-backs) is allowed.

  • Disabled modeling objects:
  • Read and write are disallowed.
  • SC_WARNING is generated as a reaction to read and write transactions (warnings can be ignored or forced to errors using the warning_level parameter).
  • A TLM error is optionally returned (this is controlled by another parameter).
  • Write value is ignored.
  • Read value is controlled by a disable_value parameter specified per modeling object or model wide.

The following API is used to traverse enable and disable states: 

     void [object]::enable(void);
     void [object]::disable(void);
     void [object]::set_enabled(bool value);
     bool [object]::is_enabled(void) const;

By default, the object value (the value contained in the register or memory) is not altered, so once an object is re-enabled it again holds the last stored value.
