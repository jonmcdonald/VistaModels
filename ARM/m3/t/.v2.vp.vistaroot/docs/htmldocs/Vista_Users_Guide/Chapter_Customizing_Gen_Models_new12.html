Memory

Memory size is defined using the size_in_bytes parameter. Memory endianness is defined using the endianness parameter whose value can be: host, little, or big.

The memory model stores and retrieves the data by itself; there is no need for additional code. However, two callback functions are created for incoming Read or Write transactions that can be used to further define behavior, or for debugging purposes. The address chunk or page is not allocated until the address is written, a read of an unallocated address always return zeros without allocating the chunk.

In addition the following parameters can be used:

  • disable_dmi_memory — when not zero, memory does not have a direct memory pointer; default is 0
  • mb_memory_chunk_size — By default, the memory chunk size is 0x100000 (1 M); if a transaction in the design has a data length greater than the memory chunk size then a DMI pointer is not returned.
