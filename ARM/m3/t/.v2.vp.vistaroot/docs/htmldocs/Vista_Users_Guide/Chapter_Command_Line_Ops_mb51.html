add_split_policy

Adds a 'sequential' policy. (See "Split Policy".)

Syntax

add_split_policy [-h | -help] [-usage] [-d | -disabled] [{-l | -latency} latency] [{-t | -type} type]
[{-w | -wait-states} wait_states] [{-p | -power} power] port_or_transaction burst_size

Arguments
Optional (Common)
  • [-h | -help]
Provides help.
  • [-usage]
Displays the syntax for this command.
Optional
  • -d | -disabled
A flag for not using this policy
  • {-l | -latency} latency
Latency in clocks.
  • {-t | -type} type
Split type, default is 'Generic', customizable
This option is deprecated.
  • {-w | -wait-states} wait_states
Clocks per word
(currently not supported)
  • {-p | -power} power
Specifies the power value in mW.
Required
  • port_or_transaction
Effect port or port.transaction
  • burst_size
The size of the buffer by which data is transmitted to/from the port
