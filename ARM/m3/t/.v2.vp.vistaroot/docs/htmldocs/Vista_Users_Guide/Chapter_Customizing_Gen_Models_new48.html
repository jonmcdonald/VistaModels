Generation Parameters (Generics)

Each AFM template wrapper supports the following generic parameters:

  • BITNESS — Default bitness for bus protocol ports
  • MASTER_PROTOCOL — Default master bus protocol name (for example ahb_master or axi)
  • SLAVE_PROTOCOL — Default slave bus protocol name (for example ahb_slave or axi)
  • NUM_IRQ — Number of generated interrupt lines
  • USE_EVS_MI — Boolean value used to determine a EVS generate style: 1 = MI style, 0 = old style
Interface Ports

Interface ports are automatically generated according to the AFM model description. Please refer to the AFM documentation.

CPU Specific Parameters

The following CPU specific parameters can be used:

  • instruction_latency — CPU unit delay; each instruction takes this amount of SystemC time
  • instruction_energy — When power is modeled, each instruction consumes the specified amount of energy

All other AFM specific parameters are also available as parameters in the parameter file.

Example: 

     top_inst.cpu_inst.NUM_IRQ = 32

  Note

To properly use ARM-FM models and their parameters consult the ARM-FM documentation.
