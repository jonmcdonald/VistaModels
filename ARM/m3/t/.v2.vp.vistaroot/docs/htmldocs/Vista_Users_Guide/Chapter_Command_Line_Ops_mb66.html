Simulation in Batch Mode

Vista allows simulation in batch mode, executing multiple tracing and acquiring scenarios, primarily via scripts.

The results of batch simulations can be analyzed through Vista post-simulation, where you can analyze multiple scenarios concurrently based on the tracing and acquiring commands specified in each batch simulation.

Name

Invoking Vista simulation in batch mode.

Synopsis

To run Vista simulation in batch mode:

vista -batch VISTA_INVOCATION_FLAGS SIMULATION_FLAGS TARGET_SPECIFICATION -- EXECUTION_PARAMETERS

To run with Questa:

vista_vsim -batch VISTA_INVOCATION_FLAGS SIMULATION_FLAGS
-- VSIM_PARAMETERS -c

To run with Cadence NC simulators:

vista_ncsim -batch VISTA_INVOCATION_FLAGS SIMULATION_FLAGS -ncsim NCSIM_EXECUTABLE -- NCSIM_PARAMETERS

or, alternatively,

vista_ncsc_run -batch VISTA_INVOCATION_FLAGS SIMULATION_FLAGS
-- NCSC_RUN_PARAMETERS

Description

Vista simulation can be invoked in batch mode, using a simulation script to specify tracing requests.

Simulation results are stored in the simulation directory, and can be then reviewed in Vista.

Parameters and Options

TARGET_SPECIFICATION

A simulation target can be specified in one of the following ways:

{[-exec] executable_path} {[-project] (project_name | project_file)} [-design design_name]

  • project_name refers to a project existing in a 'vista.ini' file;
  • project_file is a Vista project file.
  • design_name is the name of the design in the project.

The design executable must be built prior to invoking the simulation.

If the -design option is omitted, the first available design in the project will be used.

  • [-exec] executable_path

executable_path is a path to the simulation executable.


  Note

The [-simulate] option is an alias for the [-exec] option.
 

SIMULATION_FLAGS

[-simdir] directory

[-simulation-results-directory] directory

The directory to store simulation results and simulation-related files. This includes the simulation history file, the wave files and simulation settings.

This directory can be subsequently used to review the simulation results in Vista.

The specified directory will be created automatically as needed.

[-trace-all-signals]

Causes Vista to trace all channels of the types

sc_signal<>

sc_clock

sc_fifo<>.

[-trace-delta-cycles]

Causes Vista to trace data on each delta cycle (rather than on each time cycle).

-tsv

-with-tsv

[-trace-transactions]

Enables tracing of sequence of transactions on selected ports.

-memory-profiling

-with-memory-profiling

-enable-memory-profiling

Enables memory profiling.

[-run-for] time-specification

[-runfor] time-specification

Runs the simulation for the specified time. For example, '-runfor 10ns'.

[-simscript] script-path

[-sim-script] script-path

[-simulation-script] script-path

Specifies the tracing script to be invoked after design elaboration.

See the Vista tracing commands (file: trace.txt) for the description of simulation script syntax.

[-params] parameters-file

[-parameters-file] parameters-file

Specifies the path to the parameters file.

[-exec-input] filename

Redirect the standard input of the simulation process to the specified file.

[-exec-output] filename

Redirect the standard output of the simulation process to the specified file.

[-exec-error] filename

Redirect the standard error stream of the simulation process to the specified file.

VISTA_INVOCATION_FLAGS

Any Vista invocation flags can be used.

[--]

Prevents Vista from processing the rest of the command-line, causing it to be passed as-is to the simulation process.

EXECUTION_PARAMETERS

All parameters not recognized by Vista, which follow the TARGET_SPECIFICATION part of the command line, are passed to the simulation process.

VSIM_PARAMETERS

These parameters are passed to vsim.

NCSIM_PARAMETERS

These parameters are passed to the NC simulator.

NCSC_RUN_PARAMETERS

These parameters are passed to ncsc_run.

Examples

  • (prompt) vista [-batch] [-wd /tmp/mywd] [-simdir ./sim1] [-simscript myscript.tcl]
    [-exec] [-project Project.v2p] [-design simple_bus_main] [--] [-width 128]
  • (prompt) vista [-batch] [-tsv] [-simdir sim2] [-simulation-script myscript.tcl]
    [-exec-error err.log] [-exec my.exe] [--] [-fast]
  • (prompt) vista -batch -simdir sim1 -simscript myscript.tcl -exec -project Project.v2p
    -design simple_bus_main param1
  • (prompt) vista -batch -simdir sim2 -simulation-script myscript.tcl -exec-error err.log
    -exec my.exe param1

Running with Questa

The following are the three options for Questa simulations:

  1. (prompt) vista_vsim [-batch] [-wd ~/with_questa] [-simdir sim3] [
    -simulation-script myscript.tcl] [--] [-do run.do] [-c]
  1. Use a 'vista_run' script to pass common Vista invocation flags and simulation parameters:

(prompt) vista_run [-batch] [-wd ~/with_questa -tsv]
[-simulation-script myscript.tcl] [-runfor 100ns tcsh]

(tcsh-prompt) vista_vsim [-simdir test1] [--] [-do run1.do] [-c]

(tcsh-prompt) vista_vsim [-simdir test2] [--] [-do run2.do] [-c]

(tcsh-prompt) vista_vsim [-simdir test3] [--] [-do run3.do] [-c]

(tcsh-prompt) exit

  1. Run an existing script, which invokes Questa simulation, using 'vista_run':

(prompt) vista_run [-batch] [-wd ~/with_questa] [-tsv]
[-simulation-script myscript.tcl] [-simdir test4 -runfor 100ns run_questa.sh]

In run_questa.sh, the command invoking simulation has the regular `vsim' syntax, e.g.:

vsim [-do run3.do] [-c}

Running with Cadence NC simulators:

(prompt) vista_ncsc_run -batch [-wd ~/with_nc] [-simdir simnc] [-simulation-script myscript.tcl] [--] [-sctop sctop]

using 'vista_run':

(prompt) vista_run [-batch] [-wd ~/with_nc] [-simulation-script myscript.tcl tcsh]

(tcsh-prompt) vista_ncsc_run [-simdir simnc1] [--] [-sctop sctop ...]

(tcsh-prompt) vista_ncsc_run [-simdir simnc2] [--] [-sctop sctop ...]

(tcsh-prompt) vista_ncsc_run [-simdir simnc3] [--] [-sctop sctop ...]

(tcsh-prompt) exit

or

running using an existing script:

or

(prompt) vista_run -batch [-wd ~/with_nc] [-simulation-script myscript.tcl] [-simdir simnc4 run_nc.sh]

In run_nc.sh, the command invoking simulation has the regular NC syntax: e.g.:

ncsc_run [-sctop sctop ...]
