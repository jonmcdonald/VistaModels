Using External PV Models (TLM 2.0)

You can use external PV models, if the appropriate interface is based on standard TLM 2.0 ports, i.e., tlm_b_initiator_socket and respectively tlm_b_target_socket (see the TLM 2.0 standard for more details). Also, it is assumed that the first parameter in the user PV constructor is the sc_module_name. A set of pvt constructors is generated having 1, 2, ..., n additional parameters (n by default is 5). When the pvt wrapper is instantiated, the proper constructor is selected, based on the actual number of parameters in the user PV class.

The same recommendations for the coding style of a model generated by the Vista model builder, and the requirement to use the mb_event class for a threaded model, apply to external PV models.

When defining the model, the user must define the port names as they appear in the external PV class (i.e., using the proper name, side and width).
