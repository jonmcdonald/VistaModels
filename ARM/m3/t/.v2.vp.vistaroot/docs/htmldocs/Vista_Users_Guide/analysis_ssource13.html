Contention Level Analysis

Contention-level analysis is a Bus Socket Attribute that is provided for the Vista model builder Generic Bus. The analysis is automatically performed once a socket is traced and provides a time-weighted Average of the number of requests waiting to access the component bind to the bus socket.

There are two types of contention analysis performed: Address Phase and Data Phase.

  • Address Phase Contention is an Attribute of a bus master socket and provides a time-weighted Average of how many requests Wait to access the bus and send transactions through the specific bus master socket to the peripheral that is connected to it.
  • Data Phase Contention is an Attribute of the bus sockets and provides a time-weighted Average of how many Master or Slave components wait to send data. There are two types of Data Phase Contention: Forward and Backward. Forward Data Phase Contention reports the Average of how many Master components are waiting to forward data to a peripheral slave component; Backward Data Phase Contention reports the Average of how many peripheral Slave components are waiting to send data back to a master component.

A Contention folder, which includes the Address Phase and Data Phase contention-level analysis, is added under those bus sockets through which requests had to wait before accessing the bus.
