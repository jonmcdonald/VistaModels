Performing Atrenta SpyGlass Analysis from within Vista

Users who have a licensed copy of Atrenta's 1Team:System Lint Tool for SystemC and mixed HDL/SystemC designs, can run SpyGlass analysis directly from within Vista. When this is the case, and in your environment the root of SpyGlass' installation directory is pointed to by the SPYGLASS_HOME_1TS environment variable, in the Vista toolbar the Validate icon used to invoke SpyGlass will automatically be displayed: .

The launching of analysis from within Vista uses SpyGlass' -f option, whose argument is the location and name of the SpyGlass command file. By default, this file should be named spyglass.cmd and reside in Vista's working directory. However, you can specify a different location and filename by setting the environment variable VISTA_SPYGLASS_CMD_FILE which is defined in the .vistarc file.

Before running analysis, be sure to prepare the appropriate commands in the command file to be used. For details, see your SpyGlass documentation.

Then, to perform SpyGlass analysis on your SystemC design from Vista:

  1. From within Vista's Project browser, select the individual source files or entire project to be analyzed.
  1. In Vista's toolbar, click the Validate icon.

  Note

Vista passes the SystemC version that it is using to the Spyglass command line, thereby ensuring that both products use the same version.

As each of the source files is analyzed, a message to this effect will be displayed in Vista's message pane. After all of the selected files have been analyzed, a color-coded list of analysis-related violation messages is displayed in this same pane.

  • Fatal violations and Errors are displayed in red.
  • Warnings are displayed in blue.
  • General Information and Data messages are displayed in green.

When you double-click any of the lines displayed in this list, the source code that was its cause will be displayed in Vista's editing window.
