Vista Design Assembly

Vista provides a simple way to create SystemC structural code using an intuitive Block Diagram Editor.

Vista's schematics (block diagrams) link the various parts or units of a design in graphical representation. As part of a bottom-up design approach, individual unit models (created and successfully compiled) can be instantiated in a design unit using these schematics, as an alternative to coding in a conventional text editor.

From the block diagram window, it is possible to view and modify the generated source code (and corresponding modeling dialogs) for an instance, and to open the source for the top-level unit (schematic) at the location of the instantiation or declaration of the instance.

Each block diagram depicts a set of instantiated lower-level SystemC units under it. (For each unit created, corresponding .h and .cpp files are automatically created, and may be referenced under a project's source and public-header folders.) Whenever a unit is saved in the block diagram editor, its header file (.h) is updated accordingly.

For a more detailed review of the generated code, see "Structure & Flexibility of the Block Diagram and Its Generated Code".

For a description of the complete workflow for design and verification in Vista, see Vista Workflow.
