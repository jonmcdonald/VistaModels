Add mb_event as a Communication Channel

You can use mb_event as a communication channel with policies for annotating timing between concurrent threads.

The sequential policy supports mb_event_name.notify and mb_event_name.wait as cause and effect actions. This enables you to annotate timing between concurrent threads.

Example:

Assume two threads with the following code: 

     void thread_1() {
     ...
     port_1_read(...);
     ev_1.notify();
     ...
     }
     
     void thread_2() {
     ...
     wait(ev_1);
     port_2_write(...);
     ...
     }

Specifying the following sequential policies will enable scheduling port_2_write() from thread_2 with a delay of: delay_1 + delay_2 clocks after port_1_read() in thread_1 finishes: 

     add_sequential_policy e_1.notify -cause port_1.READ -latency delay_1 
        -sync 0 -power 0
     add_sequential_policy port_2.WRITE -cause ev_1.wait -latency delay_2 
        -sync 0 -power 0\

  Note

Only wait(mb_event& event) is supported; time_out on mb_event (wait(sc_time& t, mb_event& event)) is not supported.
