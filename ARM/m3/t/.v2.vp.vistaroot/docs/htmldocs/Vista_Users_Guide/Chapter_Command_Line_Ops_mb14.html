create_tlm_model

Creates a new TLM Model Interface with the defined ports and parameters.

Syntax

create_tlm_model [-help] [-usage] [-auto-parameters-defaults defaults]
[-class-name-external-pv class_name] [-ds | -don't_save] [-external-pv-include include]
[-generate-kind kind] [{-generics-data | -generics} generics] [-include include]
[-machine-arch-tcl-path path]
[{-max-parameters-of-pv-constructor | -mp} max_parameters]
[-overwrite overwrite] [-pv-kind kind] [-type type] [-timing-kind kind] [-power_kind kind]
[-vista-project vista_project] [-read_only] [{-other_port_list | -op} list] [-generate_ipxact]
[-ipxact-vendor vendor] [-ipxact-version version] [-hierarchical] [-force] model_name library_name port_list parameter_list

Arguments
Common (Optional)
  • -help
Displays help.
  • -usage
Displays the syntax for this command.
Optional
  • -auto-parameters-defaults defaults
List of {parameter default_value} for auto-generated parameters.
  • -class-name-external-pv class_name
For using an external PV: the external PV model class name
  • -ds | -don't_save
Flag indicating whether to save the model in an .mb file. Default is False (do save).
  • -external-pv-include include
For using an external PV: the external PV model path of the include
  • -generate-kind kind
PVT or PV.
  • {-generics-data | -generics} generics
Generic data for a model created from a template (generic model).
  • -include include
Include files
  • -machine-arch-tcl-path path
Path to the machine architecture file or script content.
  • {-max-parameters-of-pv-constructor | -mp} max_parameters
For using an external PV: maximum parameters of the PV constructor
  • -overwrite overwrite
Overwrite user code.
  • -pv-kind kind
Options include: template or external.
  • -type type
Specific model type (e.g. CPU, BUS, TLM/Generic_Library/Snoop).
  • -timing-kind kind
Options include: policies or learned.
  • -power_kind kind
Options include: policies or learned.
  • -vista-project vista_project
Add to specified Vista project name.
This option is deprecated; it was moved to the save_model command.
  • -read_only
A flag that indicates whether or not the model is read-only; incapable of performing save/generate operations. The default value is false.
  • {-other_port_list | -op} list
Ports or exports that are not TLM2.0 sockets; for example, {{port1 port sc_out<bool>} {port2 export sc_export<sc_signal_in_if<bool>>>}}
  • -generate_ipxact
If present, an IP-XACT component description (<model-name>.xml) will be generated for the model, in addition to the .h and .cpp files. The generated IP-XACT component will be placed in the same library (and optionally in the Vista Project, if specified) as the generated SystemC code.
Note: The full argument name must be used.
  • -ipxact-vendor ipxact_vendor
Specifies the vendor name for the generated IP-XACT descriptions. If not present, the vendor name defaults to “Mentor.com”. This is only considered if the -generate_ipxact option is present.
Note: The full argument name must be used.
  • -ipxact-version ipxact_version
Specifies the version string for the generated IP-XACT descriptions. If not present, the version string defaults to “1.0”. This is only considered if the -generate_ipxact option is present.
Note: The full argument name must be used.
  • -hierarchical
Specifies that the model is hierarchical.
  • -force
Flag for disable creation checks.
Required
  • model_name
Target model name.
  • library_name
Target TLM library name.
  • port_list
Port list for the model, e.g. {{port1 /TLM/protocols/ahb_master master 32} {port2 /TLM/protocols/ahb_slave slave 32}}
An additional optional field can be added to each port data in the list; -bind_mode optional. For example, {port1 /TLM/protocols/ahb_master_32 -bind_mode optional}. The default value for this field is -bind_mode mandatory.
  • parameter_list
Parameter list for the model, e.g. {{param1 1 uint64} {param2 true boolean}}
