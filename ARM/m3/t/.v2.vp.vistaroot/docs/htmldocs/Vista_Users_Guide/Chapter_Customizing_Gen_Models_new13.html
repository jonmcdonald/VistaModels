NIC301

The AMBA Network Interconnect is a highly configurable component that enables you to create a complete high performance, optimized AMBA-compliant network infrastructure. The possible configurations for the AMBA Network Interconnect can range from a single bridge component, for example an AHB to AXI protocol bridge, to a complex infrastructure that consists of up to 128 masters and 64 slaves of a combination of different AMBA protocols.

The slave port protocols can be AHB or AXI, and the master port protocols can be AHB, APB, or AXI. The model can be configured manually or imported from an IP-XACT model description, created by ARM Amba Designer tool.

NIC301 Parameters

The NUM_SLAVE_protocol_type parameters define the number of slave ports that have a particular protocol type.

The SLAVE_protocol_type_PREFIX parameters define the port names. Similar parameters exist for the master ports.

The port width is set by default to 32 bit, and can be changed manually.

Other parameters are defined automatically.

The master_port_name_base_address parameter defines the address mapping for the port.

NIC301 registers

The Nic 301 Model is hierarchical, consisting of internal models. Registers defined are created as part of an internal model; therefore, when adding a register, the register name should include the name of the internal register component and should be of the form:

internal model name_register name.

Example:

By default the generic NIC301 model defines two registers:

  • Reg0_register0
  • Reg0_register1

This means that an internal model named “Reg0” is created, containing the two registers: register0 and register1.

For each new internal register model it is necessary to define two user parameters:

  • internal model name_base_address — defines the mapping of the internal register model in the internal bus model
  • internal model name_size — the size of the register memory

The internal register model is connected to a special master port of the internal bus model (see Appendix D, Figure D‑2 “Topology of Generated NIC301 TLM Model”).
