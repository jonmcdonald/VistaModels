Editing Text on Screen

Much of the text appearing in graphical units can be edited directly in the graphical editor window, without having to bring up dialog boxes. For example, the names of all graphic elements, the width of signals, all free text, and more. When you edit such textual items on screen, you actually change their values in the database.

To edit text on screen —

  1. Select the text.
  1. Click the text again. The selected text now appears in an editable rectangle.
  1. Modify the text as required. (To cancel editing of text at this point, hit the Esc key.)
  1. Finalize the editing by clicking the mouse pointer anywhere else in the editing window. (If the edited text is single-lined, you can also finalize the editing by hitting the Return/Enter key.)
Specifying Font Style for Text

To specify a default font to be used for text in graphical and textual units —

  1. Click Tools > Options > Fonts.
  1. In the dialog box now displayed, you can specify the fonts to be used for the names of graphical elements and for free text. Click the appropriate ellipsis (...) button to display a dialog box which allows you to set the style, size, and color of the default font.

  Note

In dialog boxes that provide a Font command button, you can locally override the default font that is set here.

In addition, you can modify the font used for any selected text by clicking Attributes > Font.

Controlling Visibility of Text Accompanying Graphic Elements

In general, graphic elements in block diagrams are accompanied by text that can be toggled on or off for each such element (using the appropriate Attributes dialog box). This text can be moved independently of the element to which it belongs.

The default status for the visibility of accompanying text is set using the Tools > Options > Visibility dialog boxes accessed from the menu bar.

Finding and Replacing Text in Design Units

Use the Find dialog to find all occurrences of a string, either locally, or across all pages of an entire unit or entire hierarchical design tree.

Figure 9‑14. Find Dialog

To find a desired string in a given unit, click the Find command icon: , click Edit > Find, or select the Find command in the editor's popdown when no element is selected. In the Find dialog, specify that search results be displayed interactively or as a single report.

When you select the Interactive option, clicking the Next or Previous command button closes the dialog box and begins the search in the specified direction. If the string is found, its first occurrence is displayed. Each subsequent click on the Find Again icon ( ) displays the next or previous occurrence of the string.

If you want to change the direction of the search, click the Find icon. The Find dialog box reopens, enabling you to select the appropriate directions: Next or Previous.

When you select the Report option, after you specify a string and click the Next command button, the Search Results dialog box opens and contains a list of all found occurrences of the string. If you double-click any of the findings listed in this dialog box, the actual occurrence of the string in the design is highlighted.

To find and replace a string of text, click Edit > Replace or select Replace in the popdown. A dialog box similar to that for Find is now displayed. In the Replace text box, enter the string to be searched for and replaced, and in the With box, enter the new string. In addition, specify the mode of the search/replace operation:

  • Query. Each consecutive occurrence of the found string is highlighted in the window or dialog box in which it is located. As each occurrence is displayed, confirm whether the string should be replaced.
  • All. All occurrences of the found string are replaced at once. Changes made in closed units are saved automatically.
Adding Free Text to a Diagram

Among the elements you can place in block diagrams is free text, which is text Vista does not process but simply displays as comments among your graphics. You can attach free text (including RCS keywords) to an entire unit. In addition, free text comments can be linked specifically to components and channels.

Linking Free Text to an Entire Unit

To create and link free text to the entire unit —

  1. Click the Free Text icon: .

A dialog box appears for editing the text.

  1. Type the text that you want to place.
  1. If you want to modify the style, size, and color of fonts used, click the Font command button and use the dialog box now displayed. Also, if you want a frame drawn around the free text, select the Frame Visible check box.
  1. Click OK. The text attaches itself to your pointer.
  1. Click the spot in the diagram where you want the text. Later you can move or copy the element of free text like any other element of the diagram.

If, later, you want to revise the text, click the Free Text icon while the particular free-text element is the only element selected. Its dialog box then reappears.

Linking Free Text to a URL

Any free text integrated in block diagram units can be linked to the URL of your choice. When this hyperlinked text is clicked, the default internet browser on your system is invoked, and the specified URL is loaded.

To define a hyperlink for free text —

  1. Select this text by clicking it with your mouse pointer.
  1. Hold down the right mouse button and click the Hyperlink command.
  1. In the dialog box now displayed, enter the URL to be referenced by the hypertext.
  1. Click OK.
Using Free Graphics in Graphical Editors

In the Block Diagram editor, you can add free graphics to your design units. Free graphics do not have a logical significance on the design, but can be used for documentation purposes. To use any of the free graphics commands:

  1. Press and hold the right mouse button on any empty area of the design page, and from the now displayed, choose one of the following commands: Line, Circle, Arc, or Rectangle.
  1. To begin drawing the desired shape or line, click the left mouse button somewhere in the editing window. Conclude drawing this object by reclicking the left mouse button.

You can continue drawing additional shapes or lines of the same type. To exit the drawing mode for this shape or line, click the right mouse button.

If you select one or more of the shapes in the editing window and hold down the right mouse button, the now displays the commands Fill, Unfill, Group, or Ungroup. You can use these commands to manipulate the free graphics objects as desired.

Inserting External Bitmaps in Graphical Units

In the Block Diagram editor, you can insert external bitmap images. Such images do not have a logical significance on the design, but can be used for aesthetic and/or documentation purposes.

The bitmaps which you may want to insert may be stored in the directory specified in the Tools > Options > Setup > Bitmap directory text box.

To insert a bitmap in a design unit:

  1. Hold down the right mouse button on some empty area of the design page.
  1. From the now displayed, choose the External Bitmap command. In the displayed dialog box are listed all the bitmaps stored in the designated bitmap directory.
  1. Select the name of the bitmap file to be inserted and click OK.

  Note

You can also insert a bitmap which is stored in a different directory by clicking the Import button in the Bitmap dialog box. Using the dialog box now displayed, browse to the location of the bitmap you want to insert. When you click OK in the Bitmap dialog box, this bitmap is inserted in the unit window, and the bitmap itself is copied to the designated bitmap directory.

Entering Japanese Comments

An environment variable, visual_text_lang, is provided to indicate to Vista that it is being run under a Japanese operating system. Vista uses this information in maintaining its database.

If you assign this variable the appropriate value — 

     setenv visual_text_lang ja

where “ja” is in lowercase, Vista recognizes that it is being run under a Japanese operating system. In addition, setting such a value enables selecting the jisx0208.1983 option in the Font dialog box which is opened when you click the ellipses (...) button next to the appropriate context in the Tools > Options > Fonts dialog box. The Font dialog box enables you to select from a number of available fonts for the language in use.

(For more details about using Vista's environment variables, see "Setup and Invocation".)


  Note

As an alternative to using the VISUAL_TEXT_LANG environment variable provided by Vista, you can use the general environment variable LANG. Once again, this variable should be assigned the value “ja”.
