Supported Protocols in Vista

At the TLM level, standard protocols split between single channel and multi channels. The following protocols are implemented in Vista as single channel: AHB, APB, RAM, TCM, USB, GENERIC. There is one multi channel protocol: AMBA-AXI.

Single Channel Protocol and TLM Phases

Figure 7‑8. Single Channel — Read/Write Transaction

Protocol with single channel starting phase is BEGIN_REQ and end phase is BEGIN_RESP which complete the transaction (master return TLM_COMPLETE). This applies to both Read and Write transactions. The Delay policy spans through the start and end phases of the channel.

Multi Channel Protocol: AMBA AXI Protocol

As specified in: “AMBA AXI Protocol Specification” (provided by: ARM), the following should be enabled by the protocol:

  • Is burst-based.
  • Address information to be issued ahead of the actual data transfer.
  • Support for multiple outstanding transactions.
  • Support for out-of-order completion of transactions.
AXI Protocol Supported Features in Vista
  1. There are five channels. Two for Read (read address and read data) and three for Write (write address, write data and write response).
  1. Read and Write can execute concurrently.
  1. Read data must always occur after the read address.
  1. Read data is initiated by the slave (Pushing).
  1. Write data must always occur after the write address.

Figure 7‑9. Vista AXI Representation With TLM2.0 Base Protocol Phases

One can see that the AXI write response (acknowledge) channel is not specified with phases, hence no timing delays can be applied to it in Vista. But this does not impact the accuracy of the throughput, just the latency of a particular transaction. A future version of Vista will enable associating phases to the Write Status and Write Ack timing points and specify their timing.
