Machine Architecture Command Reference - Modeling

The Machine Architecture commands to be used in a tcl script are summarized in the following table (bold type denotes a default value).

Sample script: 

     add_pipeline_policy p1 p2 -latency 10 -buffer_size 10
     add_split_policy p1.READ 22 -type generic -latency 22 -wait_states 22
     add_split_policy p1.READ 33 -type generic -latency 0
     add_delay_policy p2 25
     add_sequential_policy p2.READ -cause p1.WRITE -latency 30
     add_power_policy p1 10 100

Table 18‑4. Machine Architecture Commands 

Command name

Description

add_delay_policy

Adds a `delay' policy

add_pipeline_policy

Adds a `pipeline' policy

add_sequential_policy

Adds a `sequential' policy

add_split_policy

Adds a `split' policy

declare_fifo_register

Declares a register of the type FIFO.

declare_memory

Declares a memory range

declare_register

Declares a register

declare_variable

Declares a PV variable (mb_variable)

set_clock_tree_power

Sets a clock power tree

set_leakage_power

Sets leakage power

set_register_value

Sets a register initialization value

set_sync_all

Sets synchronization at the model level

state_power

Adds state based power policy

set_nominal_clock

Sets the nominal clock
